TinyMB

Trumen 第一阶段验证载体。以极简化《骑马与砍杀》城镇玩法为规则骨架，在持续运转的单城邦中让多个 LLM 驱动的 Agent 自主行动与交互。

AIvilization Plus
一

定位：AIvilization Plus

TinyMB 是 AIvilization 的 Plus 版本，也是 Trumen 第一阶段（P1 世界模拟）的验证载体。
AIvilization 验证了基础命题——LLM 驱动的 Agent 能在共享世界中自主生活——但留下了三个空白。TinyMB 逐一填补：
▲ AIvilization 的空白 ✦ TinyMB 的增强
规则 无结构沙盒，行动无代价可衡量 极简骑砍规则：三派系博弈 + 四变量取舍 + 维护压力
参与 只能低频写 prompt 然后旁观，参与度低 高频实时对话 ↔ 低频挂机放置，完整频谱随时切换
AI 差异 统一服务器模型，用户无感 玩家自选基模，差异可感知、可对比、可验证
二

为什么选骑马与砍杀作为玩法骨架

首要目标
引入极简版骑砍规则的核心目标之一：给世界加上目标感和可玩性。AIvilization 的无结构沙盒让 Agent 行为缺乏可衡量的取舍标准，也缺乏让玩家持续投入的目标驱动力。骑砍的四变量（金钱 / 声望 / 荣誉 / 关系）和三派系博弈，给每个行动挂上代价、给玩家提供明确的进阶目标。
论据 1 架构天然同构——不需要改造 Agent 的基本行为模式

骑砍本身就是「角色在持续运转的沙盒世界中生活和闯荡」的游戏——这与 AIvilization「Agent 在持续运转的世界中生活」的架构天然同构。把 AIvilization 的经济沙盒世界换成骑砍的城邦世界，Agent 的行为模式基本不变，只是规则和目标换了。

AIvilization骑砍世界（TinyMB）
Agent 在小镇中生活的居民在石冠城中闯荡的外来者
长期目标：赚钱 / 升职长期目标：加入派系 / 积累声望 / 争夺权力
日常：工作、购物、社交日常：行军、补给、任务、社交
世界自主运行：其他 Agent 工作和社交世界自主运行：NPC 巡视、交易、结盟、施压
论据 2 22 年成熟规则——可直接拿来精简，不需从零设计

骑砍系列有 22 年历史，核心循环经大量玩家验证。战团的 mod 生态证明其规则可以被大幅精简到极小规模仍能成立。这是 MVP 阶段的关键优势——不需要从零设计游戏规则，只需要提取和裁剪。

论据 3 与多种主要 Agent 能力高度适配，能覆盖并验证多个核心维度

骑砍玩法的核心循环能很好地覆盖和利用以下多种 Agent 主要能力——这意味着在这个框架里跑起来的 Agent，可以同时在多个维度上产出有价值的验证数据：

  • 文本会话与领主谈判、在宴会上拉拢派系、向玩家汇报局势分析
  • 任务分解「加入派系」自动拆解为：赚钱 → 招兵 → 提声望 → 争取封地
  • 持久记忆Agent 记住对每个 NPC 的恩怨经历，持续影响后续社交和决策
  • 自主规划无明确指令时，Agent 自主制定并推进多步骤的行动计划
  • 多 Agent 交互NPC 之间自主巡逻、谈判、结盟，世界格局在互动中演化
  • 环境感知决策同时评估金钱 / 声望 / 关系 / 派系压力，权衡当前最优行动
  • 团队编排招募不同专长的同伴，分配任务，Agent 协调执行并汇报进展
  • 反思自修正行动受阻时主动调整计划，推敲过程写入日志供玩家回看
  • 人机协作日常事务自主处理，关键决策（站队 / 大额投入）主动请示玩家
  • 人设定制「好战贵族」vs「精明商人」vs「浪荡剑客」→ 行为方式持续分化
三

Agent 的预期行为模式

在石冠城的规则框架下，Agent 跑完整的感知→思考→决策→行动循环。以下是石冠城的区域布局和 Agent 的日常行为覆盖范围：

日常移动与补给
在城主堡、市集、酒馆、农庄之间移动，补充粮食与军饷，维持随从队伍的运转
经济与任务
买卖货物赚差价、接受行会或 NPC 的委托任务、管理金钱收支与资源调配
社交与政治
拜访派系领袖、参加宴会、打听情报、在三派系之间权衡站队或保持中立
冲突与防御
在任务执行中遭遇风险时自动结算冲突后果，护送、巡逻、驱赶威胁
请示与协作
日常事务自主处理，遇到重大决策（明确站队、高风险行动、大额投入）主动向玩家请示并等待拍板
成长与进阶
声望、荣誉与关系的积累逐步解锁新的行动路径——从外来者到派系帮手，再到核心成员
