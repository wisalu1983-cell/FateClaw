③ 设计展开

三项增强的设计展开

每一项增强背后的设计决策：为什么这样做、解决了什么问题、如何服务 P1「证明世界能跑起来、Agent 能自主行动并交互」的使命。

增强一 有规则的活世界——从无结构沙盒到有规则博弈
维度AIvilizationTinyMB
玩法骨架无预设目标和博弈结构，Agent 自由行动骑砍四变量（金钱 / 声望 / 荣誉 / 关系）+ 三派系博弈 + 维护压力
行动后果后果分散，无结构化取舍帮一方得罪另一方、资源有限必须取舍、站队有不可逆后果
空间约束无移动成本区域间移动耗时 5-20 分钟，迫使规划连续事务
为什么改
无规则意味着 Agent 的决策好坏无从衡量。引入结构化规则后，Agent 的行为有了可评估的标准——「在有限资源和派系压力下做出什么取舍」。
服务 P1 的方式
结构化规则让 Agent 行为可评估、可比较、可复现，为「Agent 能自主行动并交互」提供可验证的观察框架。
增强二 参与深度可调——从「只有低频放置」到「低频高频都有」

AIvilization 只有一种参与方式：设好策略 prompt，然后等待观察结果。玩家大部分时间只是旁观者，缺乏介入感。TinyMB 在游戏设计层面打开了参与频谱的另一端。

实时协作
玩家通过自然语言和 Agent 对话，Agent 规划执行，每步 10-30 秒一轮反馈；关键节点 Agent 主动暂停请示，玩家看清局面后拍板。
适用：有时间、想深度介入、正在处理重要局势时
方针放置
玩家预设三个维度的长期方针（经济优先级 / 派系倾向 / 风险偏好），Agent 据此自主行动；玩家隔一段时间回来翻日志、看世界变化。
适用：没空盯守、想挂机跑世界、类似 AIvilization 的体验时

请示的设计意图
Agent 不是在所有决策上都打断玩家——日常事务（补给、移动、常规任务）自主执行后再汇报。只有在命中明确条件时才向玩家请示：明确站队或改变派系立场、高风险高后果行动、大额资源投入、发现自身能力不足以完成目标。请示包含当前局面、Agent 的建议、风险分析、替代路径，以及「如果你没回复我会怎么做」。
为什么这样设计
玩家参与感的核心是「关键时刻我能做主」，而不是「每步都要我管」。设计目标是让请示时刻有分量、有信息、有真实选择——而不是频繁打扰。两种参与模式可随时切换，不需要玩家提前声明。
增强三 AI 脑子可选可感知——基模作为显式验证变量
维度AIvilizationTinyMB
基模位置服务器端统一控制，用户无感用户侧，用户选择基模
模型差异的作用不存在——所有角色用同一个模型是首轮要验证的核心假设之一
为什么改
Trumen 的长期愿景需要验证「不同 AI 能力会带来不同的世界演化」。TinyMB 将基模选择权交给用户，使「同一任务条件下，不同基模产生不同执行路径和结果」成为首轮可观测、可对比的验证变量。
服务 P1 的方式
为「多 Agent 框架」提供可对比的能力维度——不同基模在同样的规则和约束下表现如何，是 P1 技术可行性验证的关键数据点。
