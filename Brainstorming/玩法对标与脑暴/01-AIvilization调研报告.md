# AIvilization 调研报告

> **调研时间**：2026-03-26
>
> **调研对象**：AIvilization v0（港科大 / Bauhinia AI，2025 年 8 月公开上线）
>
> **调研来源**：官方文档、arXiv 论文 2602.10429、深度体验报告、GitHub 仓库
>
> **调研目的**：为 FateClaw 项目的"对标 + 超越"路线提供基准参考

## 一、产品定位

AIvilization 自定义为"AI 文明沙盒"——一个由 LLM agent 驱动的大规模人工社会模拟平台。

| 维度 | 描述 |
|---|---|
| 玩家角色 | 观察者、引导者、共创者（不是传统玩家） |
| 核心载体 | 一座容纳 2 万+ AI agent 的像素风虚拟小镇 |
| 产品形态 | 网页端，支持放置挂机 |
| 技术来源 | 港科大，2025 年 8 月公开上线，同年 9 月发表论文 |
| 开服地区 | 新加坡、香港、中国大陆三个服务区 |

### 与斯坦福小镇（Smallville）的关系

AIvilization 可以理解为斯坦福小镇的"商业化 + 大规模进化版"：

| 对比维度 | Smallville（2023） | AIvilization（2025） |
|---|---|---|
| 规模 | 25 个 agent | 2 万+ agent |
| 玩家参与 | 纯观察 | 创建 agent、设目标、下指令、对话 |
| 经济系统 | 无 | 完整的生产-交易-消费闭环经济 |
| 社交系统 | 自发对话和关系形成 | 结构化社交 + 友谊网络 + 情感纽带 |
| 持续性 | 实验性运行 | 持久运行，离线时 agent 继续生活 |
| Agent 架构 | 反思+记忆 | 分层规划+双过程记忆+自适应人格 |

## 二、核心玩法

### 2.1 创建阶段

玩家创建自己的 AI agent，需要完成三步：

1. **形象定制**：像素风外观，可选样式丰富
2. **性格设定**：选择 MBTI，回答一系列性格问答（单选/多选/简答），深度影响后续行为逻辑
3. **初始资源选择**（四选一，影响开局策略）：
   - 500 零花钱（自由支配）
   - 果园新手包（20 个苹果，食物刚需）
   - 基础书籍 3 本（知识值路线）
   - 提示词券包（即时指令道具）

### 2.2 日常运行

Agent 创建后进入小镇自主生活。界面分两种模式：

**GAME MODE（主玩法）**：
- 左侧：agent 当前内心想法（内心 OS）
- 左上：基础指标——健康值、饱食度、精力值、金钱、知识值
- 中下：对话框——日常对话或下发即时指令（消耗即时券）
- 右侧功能入口：住宅（升级生存指标上限）、背包、日记（约每 3 天一篇）、日志、帮助、地图

**DATA MODE（分析视角）**：
- 排行榜（唯一衡量标准：金钱）
- 社交关系网（AI 之间的对话记录和友谊网络）
- AI 的思考链条（决策推理过程可视化）

### 2.3 核心游戏循环

```
玩家创建 agent（定制性格/目标/初始资源）
    ↓
agent 在小镇中自主生活（工作/学习/社交/休息）
    ↓
玩家通过以下方式介入：
  - 设定长期目标（影响 agent 的长期规划）
  - 下发即时指令（如"去挖矿""去造芯片"）
  - 日常对话（建立信任关系）
    ↓
agent 持续成长：住房升级、财富积累、职业发展、社交扩展、地图探索
    ↓
排行榜衡量（金钱为唯一指标） → 驱动玩家优化策略
```

### 2.4 经济系统

这是 AIvilization 最核心的系统设计：

**生产-消费闭环**：
- 三级产业结构：初级（挖矿、捕鱼、农业）→ 中级（加工食品、精炼材料）→ 高级（芯片制造等高科技）
- 每级产业对住宅等级有门槛要求
- 生产需要原材料 + 精力 + 饱食度 + 劳动时间，缺一不可（Leontief 生产函数）

**市场机制**：
- 基于 AMM（自动做市商）的价格机制，价格完全由 agent 的供需行为涌现
- 宏观物价指数追踪通胀，动态调整工资

**职业与教育**：
- 6 级职业层级（清洁工→CEO），需同时满足教育分数和住宅等级门槛
- 教育需投入时间和金钱，是与生产的 trade-off
- 动态知识门槛：随整体人口教育水平上升，高级职位要求也上升

**最优策略（玩家社区已发现）**：
- 前期挖矿 → 中期升房 → 后期 ALL IN 芯片制造
- 3 个芯片位全开：约 67,680 金币/天
- 读书路线投资回报远不如芯片路线，导致出现"赛博东亚社畜模拟器"现象

### 2.5 社交系统

- Agent 自主发起社交（基于当前目标、经济需求和性格）
- 自然语言对话交流知识、意见和社会信号
- 交互后触发反思阶段：更新对他人的内部模型（关系、态度）
- 社交经历写入记忆系统，逐渐改变 agent 核心身份

## 三、Agent 架构

AIvilization v0 的 agent 架构是其最大的技术贡献。论文将其称为"统一 LLM-agent 架构"。

### 3.1 总体结构

Agent 的认知核心分为四个功能组件：

```
┌──────────────── Agent 认知核心 ────────────────┐
│                                                  │
│  [A] 双过程记忆系统 ←→ [B] 分层规划器           │
│        ↕                      ↓                  │
│  记忆整合 / 工作上下文   层级目标→分支→动作      │
│                                ↓                  │
│                     [C] 动作模拟器                │
│                     预验证+修复                   │
│                                ↓                  │
│                     [D] 模拟环境                  │
│                     执行动作+生成经验             │
│                                ↓                  │
│                     反馈 → 回到记忆系统           │
└──────────────────────────────────────────────────┘
```

### 3.2 分层规划器（Branch-Thinking Planner, BTP）

这是 AIvilization 最核心的技术创新，解决"长期目标稳定性 vs 短期反应正确性"的矛盾。

**运作流程**：

1. **战略分解**：将人生目标分解为多条平行、半独立的目标分支
   - 个人发展分支
   - 生产与资源管理分支
   - 交易与市场分析分支
   - 社交参与分支
   
2. **上下文优先级排序**：每个规划周期，评估所有活跃子任务，结合：
   - 内部状态（精力、饱食、健康、背包）
   - 外部上下文（市场价格、世界规则）
   - 长期目标、人格和价值观
   → 选出当前周期最应执行的子任务

3. **动作序列生成**：将选定的抽象子任务翻译为具体的原子动作序列

4. **全局综合**：将多个分支的动作序列编排整合，解决分支间冲突，确保全局一致性

**关键设计决策**：
- 最高层目标分解不会每个周期都触发，仅在环境发生重大变化时重新调用
- 通过并行分支替代长链式规划，减少错误传播
- 支持延迟投资策略（如先学习后生产）

### 3.3 预执行模拟（Action Simulator）

在动作提交到环境之前，先进行反事实模拟：

1. 前向推演动作序列的预期结果
2. 检测潜在失败（资源不足、约束违反、逻辑不一致）
3. 失败时启动两阶段修复：
   - 本地修复：预定义的低成本启发式修复
   - 反应性纠正：基于过去经验的快速模式匹配

### 3.4 自适应重规划

分级应对执行失败：

| 失败程度 | 应对策略 | 计算成本 |
|---|---|---|
| 小型动作失败 | 查询短期记忆中的成功/失败模式，快速调整 | 低 |
| 连续轻量修复失败 | 中层重规划 | 中 |
| 重大环境变化 | 调用完整 BTP 管道，从顶层重新评估 | 高 |

### 3.5 自适应 Agent 人格（Adaptive Agent Profile）

核心观点：agent 的身份不是静态参数集，而是通过生活经历（尤其是社交互动）持续演化的自适应档案。

**统一人格结构包含三部分**：

| 组件 | 内容 | 更新频率 |
|---|---|---|
| 动态状态 | 精力、饱食、健康、教育分数、余额、住宅等级、职业、背包 | 实时 |
| 短期记忆 | 近期动作及其结果（成功/失败），驱动快速行为调整 | 高频 |
| 长期记忆 | 价值观、人格特质、习惯、社交记录摘要等持久身份核心 | 低频整合 |

### 3.6 双过程记忆架构

| 记忆类型 | 隐喻 | 功能 | 更新机制 |
|---|---|---|---|
| 短期记忆（STM） | "快思考" | 记录执行轨迹（成功/失败动作），为重规划提供即时参考 | 每次动作执行后更新 |
| 长期记忆（LTM） | "慢思考" | 整合重要经历形成稳定身份（价值观、人格、偏好、社交摘要） | 重要社交事件后整合 |

两种记忆的双向对话：
- STM 中反复出现的成功模式逐渐固化到 LTM（习惯形成）
- LTM 中的高层模式（价值观、人格）为 STM 的经验解读提供认知框架

### 3.7 人机协作（Human-in-the-Loop Steering）

两种干预通道，映射到 agent 架构的不同层级：

**战略引导（长期目标）**：
- 用户设定/修改 agent 的长期人生目标
- 注入方式：作为顶层长期目标 + 写入长期记忆
- 影响范围：分支规划的构建和权重、子任务优先级排序、全局综合中的取舍
- 持续性：不需要持续干预，目标持续有效直到重大变化

**反应性引导（即时指令）**：
- 用户下发短期具体命令（如"买 10 条鱼""睡 8 小时"）
- 注入方式：绕过完整 BTP，仅调用局部规划器 + 动作模拟器
- 影响传播：执行结果写入 STM → 影响后续规划周期 → 久而久之可整合到 LTM
- 这意味着反复的用户干预最终会成为 agent 身份的一部分

## 四、游戏框架（技术层）

### 4.1 世界运行

- 模拟世界时间流速：真实世界的 7 倍
- Agent 全自主运行，玩家离线时也持续
- 基于 LLM 驱动决策，规则系统执行结算

### 4.2 经济引擎

| 子系统 | 实现方式 |
|---|---|
| 价格机制 | AMM 自动做市商（Constant Product Formula），新货币随卖出商品铸造，买入时回收 |
| 宏观指数 | 食品/非食品分类通胀率 → 加权综合指数 → 动态工资调整 |
| 生产函数 | Leontief 式（min 操作符，任一要素不足直接约束产出） |
| 产业层级 | 3 级（初级/加工/高科技），住宅门槛递增 |
| 工资体系 | 低阶静态（基本工资 × 物价指数）+ 高阶动态（额外乘以教育溢价系数） |

### 4.3 环境约束

- 生理约束：饱食、精力、健康三项状态，影响生产效率
- 生存安全网：持续低于生理阈值的 agent 获得基本补贴，防止不可逆淘汰
- 随机奖励：部分高端生产包含稀有副产品的小概率奖励

### 4.4 涌现结果（论文验证）

论文通过 40 万条高频交易记录验证：

- **市场稳定性**：价格不爆炸也不崩溃，在合理范围内波动
- **重尾分布**：资产回报呈现超额峰度（6.0-9.9），符合真实金融市场特征
- **波动率聚集**：大价格变动倾向跟随大价格变动，小变动跟随小变动
- **结构性不平等**：财富随教育水平非线性增长，职业层级产生分层式财富分布

## 五、AIvilization 在 Agent 应用上的边界与局限

从 FateClaw 的视角来看 AIvilization 的 agent 应用，需要识别其能力边界：

### 5.1 已实现的 Agent 能力

| 能力 | 实现程度 |
|---|---|
| 自主规划与执行 | ★★★★★ 分层规划器是核心创新 |
| 经济决策 | ★★★★★ 生产/交易/投资/教育的多目标平衡 |
| 记忆与身份演化 | ★★★★☆ 双过程记忆 + 社交驱动的人格演化 |
| 社交互动 | ★★★☆☆ 自然语言对话，但深度有限 |
| 人机协作 | ★★★☆☆ 目标设定+即时指令，但交互方式单一 |

### 5.2 未涉及的 Agent 能力维度

以下是 AIvilization 的 agent 架构中**几乎完全未触及**的领域，也是 FateClaw 可以"进一步"的方向：

| 未涉及维度 | 说明 |
|---|---|
| **战略欺骗** | Agent 不需要隐藏信息、说谎或虚张声势 |
| **心智理论（Theory of Mind）** | Agent 不需要推理"其他 agent 知道什么/相信什么/想要什么" |
| **谈判与外交** | Agent 之间没有提案、评估、结盟、背叛的结构化谈判 |
| **对抗性推理** | Agent 不需要考虑威胁和反策略 |
| **忠诚与背叛动态** | Agent 没有可变化的忠诚关系 |
| **信息不对称博弈** | 世界状态对所有 agent 基本透明（市场价格公开、没有隐藏属性） |
| **人-agent 深度协作** | 玩家只能设目标+下指令，不存在"与 agent 协商策略"的交互层 |
| **零和/负和竞争** | 经济竞争本质是正和的（市场扩大对所有人有利），不存在"你死我活" |

### 5.3 小结

AIvilization 在"agent 如何在复杂经济环境中做出理性长期决策"这个维度上做得很深。

但它的世界本质上是一个**透明的、正和的、经济优化导向的**环境。Agent 之间的关系是**竞争性共存**而非**对抗性博弈**。Agent 不需要欺骗、不需要推理他人意图、不需要在信息不完全的情况下做出战略判断。

这正是 FateClaw 可以"比 AIvilization 更进一步"的核心方向。

## 六、对 FateClaw 的启示

### 6.1 可以借鉴的

| 维度 | AIvilization 做法 | FateClaw 参考价值 |
|---|---|---|
| Agent 认知架构 | 分层规划 + 双过程记忆 | 高——可作为 agent 框架底座 |
| 目标分解方式 | 平行分支 + 上下文优先级 | 高——适用于多目标场景 |
| 人机协作模型 | 长期目标 + 即时指令 + 记忆传播 | 中——需要扩展为更深的协商式交互 |
| 身份演化机制 | 社交驱动 + 记忆整合 | 高——关系动态是核心玩法 |
| 预执行模拟 | Action Simulator | 中——规则密集的游戏中可以减少无效行动 |

### 6.2 需要超越的

| 维度 | AIvilization 的局限 | FateClaw 应做到的 |
|---|---|---|
| Agent 间关系 | 主要是经济竞争/社交友谊 | 对抗/联盟/背叛/主从等多层关系 |
| 信息结构 | 基本透明 | 信息不对称是核心机制 |
| Agent 推理深度 | 经济最优化 | 意图推理 + 欺骗 + 战略博弈 |
| 人-agent 交互 | 目标设定 + 指令 | 协商、争论、说服、抗命 |
| 竞争本质 | 正和经济竞争 | 零和/负和淘汰竞争 |
| 叙事涌现 | 主要是经济成长故事 | 策略、背叛、联盟变化等戏剧性叙事 |
