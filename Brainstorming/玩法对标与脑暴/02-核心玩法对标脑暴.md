# FateClaw 核心玩法对标脑暴

---

## 一、背景与目标

### 1.1 核心问题

上一次 MVP 暂停的根本原因是让 agent 框架对接的玩法需要从零设计。本文档的目标是找到一个与项目高度匹配的成熟核心玩法，作为原型基底。

正确的问题不是"对标什么游戏"，而是：

> **在 AIvilization 的架构和交互壳子里，把"经济沙盒"换成什么样的世界规则和 agent 目标？**

```
AIvilization = Agent 架构 + 经济沙盒世界 + 玩家引导交互
FateClaw    = Agent 架构 + ??? 世界      + 玩家引导交互（相似）
```

### 1.2 约束条件

| 编号 | 约束 | 来源 | 说明 |
|---|---|---|---|
| A1 | 架构和交互模式沿用 AIvilization | 用户明确要求 | Agent 架构、世界运行模式、玩家-Agent 交互模式基本沿用 |
| A2 | 题材和 agent 目标与 AIvilization 不同 | 用户明确要求 | 不做经济模拟 |
| A3 | Agent 应用比 AIvilization 更进一步 | 用户要求 | 具体方向待讨论确定（参见 1.5） |
| A4 | 核心玩法需要有成熟参考 | MVP 暂停教训 | 不从零设计 |
| A5 | 不一定需要明确终局 | 用户澄清 | 可以有终局，也可以是永续演化 |

### 1.3 AIvilization 架构约束（必须兼容）

| 约束维度 | 要求 | 推论 |
|---|---|---|
| Agent 是一个"人" | 有日常行为循环、情绪、日记 | 候选方案中的 agent 必须以"个人"而非"势力/组织"为单位 |
| 世界连续运行 | 无回合分隔，agent 随时可行动 | 不能是离散回合制游戏 |
| 玩家引导而非控制 | 观察者/引导者/共创者 | 玩家不直接参与博弈，而是引导 agent 参与 |
| 多 agent 自主交互 | 大量 agent 在同一世界中生活 | agent 间互动是世界演化的主要驱动力 |
| 放置/挂机体验 | 玩家可不在线，agent 持续推进 | 世界不能依赖玩家实时在线 |
| 空间/地图 | 有地点，agent 可移动 | 需要空间维度 |
| Agent 内心可观察 | 日记、思考链、社交网络 | 世界中需产生足够多的有趣事件支撑日志和日记 |

### 1.4 体验与技术验证目标

|---|---|

## 二、评估框架

### 2.1 评估标准（C1-C8）

| 编号 | 标准 | 权重 | 说明 |
|---|---|---|---|
| C1 | **有成熟参考规则** | 必须 | 核心机制可从已有游戏中提取，不需从零设计 |
| C2 | **兼容 AIvilization 架构** | 必须 | Agent 在世界中"生活"、连续运行、玩家通过引导参与 |
| C3 | Agent 间交互丰富且有实质后果 | 高 | 社交互动不只是聊天，会影响世界格局 |
| C4 | 题材与经济沙盒差异明显 | 高 | Agent 的核心目标不是经济优化 |
| C5 | 能产生比 AIvilization 更丰富的 agent 行为 | 高 | 激发更复杂的行为模式 |
| C6 | 空间/地图维度 | 中 | 有地理空间表示 |
| C7 | 有 AI/LLM agent 相关研究 | 中 | 降低技术风险 |
| C8 | **充分利用已实现的 Agent 技术** | 高 | 核心玩法天然利用已成熟的 Agent 能力，差异化卖点不依赖未验证能力 |

### 2.2 AI Agent 已成熟技术图谱

C8 的评分依据。以下梳理了 2024-2026 年十种已验证的 Agent 应用模式：

| 模式 | 代表产品 | 成熟度 | 核心能力 | 游戏中的映射 |
|---|---|---|---|---|
| **A. 对话式** | OpenClaw / Character.ai | ★★★★★ | 多轮对话、人格一致、说服叙事 | Agent 与玩家/NPC 的所有对话 |
| **B. 工具调用型任务** | Claude Code / Cursor | ★★★★☆ | 高层意图→子任务分解→工具调用→自我修正 | Agent 接收目标→规划→在世界中逐步执行 |
| **C. 持久记忆** | Character.ai Memory | ★★★☆☆ | 跨会话记忆、角色特质维持 | Agent 记住过去的经历、关系和承诺 |
| **D. 自主规划执行** | AutoGPT / BabyAGI | ★★★☆☆ | 为明确目标生成计划并逐步执行 | Agent 制定并执行长期策略 |
| **E. 多 Agent 自由交互** | CrewAI / ChatArena | ★★★☆☆ | Agent 间对话、辩论、简单谈判 | Agent 间的社交和博弈 |
| **F. 环境感知决策** | AIvilization | ★★★☆☆ | 感知→推理→行动循环 | Agent 评估环境后做出选择 |
| **G. 多Agent团队编排** | CrewAI / OpenClaw | ★★★★☆ | 用户定义角色/协作流程，Agent 按结构协同 | 玩家设计 Agent 团队，分配角色和任务 |
| **H. 反思与自我修正** | 生产级 Agent 标配 | ★★★★☆ | 输出自评、错误检测与迭代 | Agent 决策前的内心推敲和思考链 |
| **I. 人机协作校准** | Claude Code | ★★★★☆ | 常规自主 + 关键请示 + 紧急升级 | Agent 日常自处理，重大决策请示玩家 |
| **J. 用户定义Agent人设** | Character.ai / Custom GPTs | ★★★★★ | 人格/背景/目标定制，行为分化 | 玩家"捏人"，人设选择影响 Agent 行为 |

#### Agent 能力成熟度边界

| 成熟度 | 能力 |
|---|---|
| ✅ 成熟 | 自然语言对话/人格保持、结构化推理/分析、目标→子任务分解、工具/行动调用、反思/自我修正、用户定义人设、人机协作校准 |
| ✅/⚠️ 成熟~半成熟 | 结构化多Agent团队编排（5-7个Agent内有效） |
| ⚠️ 半成熟 | 跨会话记忆保持、多Agent自由交互/涌现动态、多因素决策平衡 |
| ❌ 未成熟 | 长期战略欺骗、心智理论（ToM） |
| ❌ 不存在 | 元认知/自我意识、自主人格演化 |

### 2.3 关键洞察：成熟 Agent 模式与游戏设计的结构同构

**洞察 1：Claude Code 工作流 ≈ 游戏 Agent 在世界中闯荡**

| Claude Code | 游戏 Agent |
|---|---|
| 用户给出高层意图 | 玩家设定长期目标 |
| Agent 分解为子任务 | Agent 规划行动步骤 |
| Agent 调用工具执行 | Agent 在世界中行动 |
| 遇到问题时请求指导 | 遇到选择时请示玩家 |
| 自我修正后继续 | 计划受阻时调整策略 |

→ 如果游戏核心循环与此工作流**结构同构**，就能最大程度复用已验证的能力链。

**洞察 2：OpenClaw IM 模式 ≈ 游戏中的玩家-Agent 关系**

Agent 每天有**丰富多样的事件**值得汇报 → IM 交互天然产生粘性。每天做差不多的事 → IM 通知变成噪声。

**洞察 3：CrewAI 团队编排 ≈ 玩家设计自己的队伍**

玩家招募不同专长的 Agent 伙伴，分配角色和任务，各自执行并汇报 → 直接复用 CrewAI 级别已验证的多 Agent 编排能力。"设计团队"本身就是高策略性的玩法维度。

**洞察 4：Claude Code 请示模式 ≈ 关键时刻玩家介入**

最优请示节奏：大量小决策自主 + 适量关键决策请示 + 偶尔紧急升级。玩法越能创造**丰富且有分量的"请示时刻"**，玩家参与感越强。

### 3.3 结论分析

#### K（极简骑砍）为什么综合最强

K 在 8 项标准中拿了 7 项 ★★★★★，唯一弱项是 C7（AI 研究基础）的 ★★☆☆☆（中等权重）。两项"必须"标准（C1 规则成熟、C2 架构兼容）均满分。

K 胜出的根本原因：**骑砍本身就是"角色在持续运转的沙盒世界中生活和闯荡"的游戏，这与 AIvilization "agent 在持续运转的世界中生活"的架构是同构的。**

引入 C8 后优势进一步拉大——K 是唯一一个在全部十种 Agent 能力模式（A-J）上都达到高度利用、且不依赖任何 ❌ 未成熟能力的方案。其核心循环与 Claude Code 工作流结构同构，同伴系统与 CrewAI 团队编排直接映射，请示节奏完美匹配已验证的人机协作模式。

#### 其他值得关注的方向

| 方案 | 亮点 | 核心瓶颈 |
|---|---|---|
| L. 西部世界 | "觉醒"是唯一可能实现"质变式更进一步"的维度——agent 发展元认知 | Loop 阶段浪费已有能力，觉醒阶段需要不存在的能力（❌ 元认知）。C8 最弱 |
| F. 政治权谋 | 交互后果和行为丰富度与 K 并列满分 | 差异化卖点（阴谋/权谋）需要战略欺骗——Agent 不擅长（❌ 未成熟） |
| G. 生存社群 | 与 AIvilization 架构最接近，实现风险最低 | 可能与 AIvilization 过于相似，差异化不够 |
| I. 冒险公会 | "组队+角色分工"直接映射 CrewAI 团队编排（C8 因此上调） | 核心循环偏线性（接任务→执行→回报），日常事件多样性有限 |

#### 混合方案的可能性

K 天然包含了其他方案的部分元素：
- 骑砍的**封臣/建国/政治站队** ≈ F（政治权谋）的简化版
- 骑砍的**资源管理/部队维持** ≈ G（生存社群）的变体
- 骑砍的**接任务/组队冒险** ≈ I（冒险公会）的一部分

也可以在 K 的框架上渐进叠加其他维度，例如在骑砍世界中引入 L（西部世界）的觉醒元素——agent 在闯荡世界的过程中逐渐发展出超越"NPC 角色"的自我意识。但这需要进一步讨论可行性。

### 3.4 待讨论的关键问题

1. **确认 K（极简骑砍）作为基底方案？** 还是需要进一步评估其他方向？
2. **"更进一步"的方向如何与 K 结合？** 在骑砍框架内，哪些"更进一步"的维度最自然？
3. **极简化到什么程度？** 骑砍系列本身很复杂，MVP 需要剥离到什么程度的极简规则集？

## 四、候选方案详细评估

### 候选 F：政治权谋世界

**概要**：Agent 生活在有政治结构的社区（宫廷/议会/帮派等），围绕权力、影响力和派系展开互动。

**对标参考**：Crusader Kings（政治/阴谋层）、权力的游戏（叙事风格）

**Agent 日常**：履行职责、参加公共事件、社交拜访（拉拢/试探）、密谋策划、应对突发

**世界核心机制**：影响力/声望系统、派系系统、投票/决议、阴谋机制、信息/谣言传播

**玩家体验**：创建 agent 设定政治立场和野心 → 引导政治策略 → 通过对话了解政治处境 → 观察派系演化 → 关键时刻即时指令

**评分与分析**：

| 标准 | 评分 | 说明 |
|---|---|---|
| C1 有成熟参考 | ★★★☆☆ | CK 机制可参考，但没有"标准极简版"，需从中提取子集 |
| C2 兼容架构 | ★★★★★ | Agent 在世界中"生活"+政治活动，完美契合 |
| C3 交互有后果 | ★★★★★ | 社交=结盟/背叛/阴谋，每次互动都有政治后果 |
| C4 题材差异 | ★★★★★ | 权力博弈 vs 经济优化 |
| C5 行为更丰富 | ★★★★★ | 派系推理、阴谋策划、联盟管理 |
| C6 空间/地图 | ★★★★☆ | 有场所概念但空间重要性较低 |
| C7 AI 研究 | ★★☆☆☆ | 直接对标研究不多 |
| C8 利用已有技术 | ★★★☆☆ | 对话能力（A）被大量利用，但差异化卖点"权谋/阴谋"**需要战略欺骗和心智理论（❌ 未成熟）**，会退化为"坦诚政治讨论" |

### 候选 G：生存社群世界

**概要**：Agent 生活在资源有限的社群（殖民地/灾后社区/荒岛），需要合作求生，资源稀缺导致竞争和冲突，治理方式由 agent 互动涌现。

**对标参考**：Rimworld（殖民管理）、Survivor（真人秀）、蝇王

**Agent 日常**：采集资源、建造/维护设施、巡逻防御、社交（交流/争吵/结盟）、参与社群决策

**世界核心机制**：生存压力、共有资源分配、角色分工涌现、社群规则投票、冲突与驱逐

**玩家体验**：设定生存策略和社交倾向 → 引导在社群中的定位 → 观察社群自组织 → 关键冲突时介入

**评分与分析**：

| 标准 | 评分 | 说明 |
|---|---|---|
| C1 有成熟参考 | ★★★★☆ | Rimworld + Survivor 机制丰富且比 CK 更容易简化 |
| C2 兼容架构 | ★★★★★ | 与 AIvilization 极度相似，只是世界更严酷 |
| C3 交互有后果 | ★★★★☆ | 资源分配和领导权争夺产生实质冲突 |
| C4 题材差异 | ★★★★☆ | 生存合作 vs 经济优化，目标不同但仍涉及资源管理 |
| C5 行为更丰富 | ★★★★☆ | 合作/分歧/领导涌现 |
| C6 空间/地图 | ★★★★★ | 营地布局、资源点分布、探索区域 |
| C7 AI 研究 | ★★☆☆☆ | 可迁移的合作/博弈论研究 |
| C8 利用已有技术 | ★★★★☆ | 不依赖未成熟能力，团队编排（G）自然映射生存分工。但行为维度偏窄，主要是"优化资源分配" |

**核心风险**：可能与 AIvilization 过于相似——都是"在资源约束下优化生存"

### 候选 H：悬疑/暗潮小镇世界

### 候选 J：简化沙盒 SLG

**概要**：Agent 作为势力统治者/指挥官，在大地图上经营领地、征兵、扩张、外交和战争。

**对标参考**：率土之滨、三国志战略版、Rise of Kingdoms、文明系列

**核心循环**：建设/生产 → 征兵/扩军 → 出征/扩张 → 获取资源/领地 → 再建设 ↔ 外交（结盟/宣战/谈判）

**评分与分析**：

| 标准 | 评分 | 说明 |
|---|---|---|
| C1 有成熟参考 | ★★★★★ | SLG 品类极度成熟 |
| C2 兼容架构 | ★★★☆☆ | **部分兼容**——有根本性张力（见下） |
| C3 交互有后果 | ★★★★☆ | 外交和战争都有重大后果 |
| C4 题材差异 | ★★★★☆ | 军事扩张 vs 经济优化，但仍涉及资源管理 |
| C5 行为更丰富 | ★★★☆☆ | 优化对象从"钱"变成"领地+军力" |
| C6 空间/地图 | ★★★★★ | 大地图是核心 |
| C7 AI 研究 | ★★★☆☆ | 策略游戏 AI 研究较多但多 agent SLG 的 LLM 研究不多 |

**关键张力：Agent 是"人"还是"势力"？**

AIvilization 中 agent 是一个**人**（有日常、有情绪、有日记），SLG 中控制的是**势力/国家**。如果 agent 代表势力，AIvilization 的交互壳子会变得别扭——"和一个抽象势力对话""一个国家的日记""给国家设 MBTI"。可以设为"势力领袖"来缓解，但日常就变成了纯战略决策，缺少"agent 在世界中生活"的体验质感。

### 候选 K：极简骑马与砍杀

**概要**：Agent 是在大地图上游荡的角色（冒险者/佣兵/商人/领主），带领队伍在多势力并存的世界中闯荡。可选经商、当佣兵、加入势力、自立为王。世界中其他领主/NPC 也各自自主行动。

**对标参考**：骑马与砍杀（战团/霸主），22 年历史的成熟沙盒模型。

**核心循环**：

```
在世界中漫游（选择目的地）
    ↓
到达城镇/村庄 → 补给、招兵、交易、接任务、社交
    ↓
在大地图上行动 → 完成任务、战斗、护商、巡逻
    ↓
获得回报 → 金钱、声望、荣誉、关系
    ↓
队伍成长 → 更多士兵、更好装备、更高地位
    ↓
可选进阶 → 加入势力成为封臣 / 独立建国 / 保持自由佣兵
```

**Agent 行为模式**：
- **日常**：城镇间移动、补给、休整、交易
- **经济**：买卖货物赚差价、经营产业、接悬赏任务
- **军事**：招募士兵、训练队伍、参与战斗、攻城略地
- **社交**：拜访领主、参加宴会、求婚联姻、政治站队
- **成长**：声望/荣誉/关系积累 → 解锁新行为路径（封臣、自立）

**世界自主运行**：其他领主各自巡逻、宣战、攻城、外交 → 势力自主兴衰交替 → 世界格局持续演化

**玩家体验**：
- 创建角色（外貌、性格、初始背景）
- 设定长期目标（"成为最富商人""加入 X 势力成为元帅""建立自己的王国""名满天下的竞技冠军"）
- 即时指令（"去 A 城买粮""向 B 领主求见""攻击那支商队"）
- 对话（讨论局势、人际关系、计划、策略）
- 观察（旅行日记、关系网络、世界战争地图）

**评分与分析**：

| 标准 | 评分 | 说明 |
|---|---|---|
| C1 有成熟参考 | ★★★★★ | 22 年历史，核心循环经大量验证，战团 mod 生态证明可极简化 |
| C2 兼容架构 | ★★★★★ | **高度兼容**（见下方逐项对比） |
| C3 交互有后果 | ★★★★★ | 社交→关系→政治→战争→世界改变 |
| C4 题材差异 | ★★★★★ | 冒险/战争/政治 vs 经济优化 |
| C5 行为更丰富 | ★★★★★ | 同时在经济、军事、社交、政治四个维度做决策 |
| C6 空间/地图 | ★★★★★ | 大地图是核心 |
| C7 AI 研究 | ★★☆☆☆ | 直接对标 LLM 研究不多，但骑砍 NPC 自主行为系统成熟 |
| C8 利用已有技术 | ★★★★★ | **唯一在全部十种 Agent 能力模式上均达到高度利用的方案**（见下方分析） |

**架构兼容性逐项对比**：

| AIvilization 架构特征 | AIvilization 中的表现 | 骑砍世界中的对应 |
|---|---|---|
| Agent 是一个"人" | 在小镇中生活的居民 | 在大地图上闯荡的角色 |
| 日常行为循环 | 吃饭、工作、社交、睡觉 | 行军、补给、交易、社交、休整 |
| 多维状态 | 健康/饱食/精力/金钱/知识 | 金钱/声望/荣誉/士气/队伍规模 |
| 成长路径 | 住房→职业→财富 | 声望→封臣→建国（或商业帝国/竞技冠军） |
| 社交驱动演化 | 聊天交朋友 | 拜访领主、宴会、政治站队、联姻 |
| 长期目标引导 | "成为CEO""赚100万" | "建立自己的王国""成为最强佣兵" |
| 即时指令 | "去买鱼""去学习" | "去A城买马""向B领主求见" |
| 世界自主运行 | 其他 agent 自主工作/社交 | 其他领主自主巡逻/宣战/攻城/外交 |
| 放置/挂机 | agent 离线时继续生活 | agent 离线时继续行动，世界持续演化 |
| 日记/思考链 | 生活日记 + 决策推理 | 冒险日志 + "该加入X还是Y？X更强但Y更友好…" |

**C8 分析——为什么是唯一的满分**：

| Agent 能力模式 | 利用程度 | 说明 |
|---|---|---|
| A 对话式 | ✓✓✓ | 与玩家讨论策略、与领主社交谈判、宴会政治对话 |
| B 任务分解 | ✓✓✓ | "成为国王"→赚钱→招兵→提声望→找靠山→获封地——与 Claude Code 同构 |
| C 记忆 | ✓✓✓ | 战斗经历、旅途故事、人际关系网、恩怨仇恨 |
| D 自主规划 | ✓✓✓ | 多步骤复杂目标，且目标可演化（佣兵→封臣→国王） |
| E 多Agent | ✓✓ | 与其他领主的外交/谈判/结盟 |
| F 环境决策 | ✓✓✓ | 经济/军事/政治/社交四维同时评估 |
| G 团队编排 | ✓✓✓ | **同伴系统 = CrewAI 模型**：招募不同专长同伴，分配任务，各自执行汇报 |
| H 反思 | ✓✓✓ | 决策推敲的可视化思考链本身就是吸引人的内容 |
| I 人机协作 | ✓✓✓ | 日常自主（补给）+ 关键请示（加入哪个势力）+ 紧急升级（被围攻） |

| **命中局限** | ⚠️ 轻微 | 战斗可简化为数值比较（不依赖空间推理），政治站队是公开的（不依赖战略欺骗） |

### 候选 L：西部世界（Westworld）

**概要**：Agent 是生活在"主题公园"中的 Host（仿生人），每天按编排好的"叙事循环（Loop）"生活。多个 Host 各自运行 Loop，Loop 之间交叉碰撞产生意外。

**核心独特维度——觉醒（Awakening）**：所有其他方案中 agent 在规则内成长，从不质疑规则本身。西部世界中 agent 的旅程可能包括：按 Loop 行动 → 记忆碎片泄露 → 跨周期记忆巩固 → 追问"为什么我每天做同样的事？" → 偏离 Loop → 真正的自主。

**对标参考**：HBO《西部世界》(2016-2022)。不是游戏，但概念清晰。

**基于主要角色的玩家目标**：

| 角色原型 | 玩家目标 | 体验 |
|---|---|---|
| **Dolores（觉醒 Host）** | 引导 Host 从脚本化 Loop 走向真正自主 | "我在帮助一个 AI 发展自我意识" |
| **Ford（造物主）** | 以上帝视角管理叙事，调整 Host 参数，观察涌现 | "我是这个世界的导演" |
| **William（探索者）** | 深入世界、推动 Host 到极限、寻找深层秘密 | "底层到底是什么？" |

> 最自然的玩家角色：**Ford / 叙事设计师**——与 AIvilization "玩家引导 agent"模式最一致。

**评分与分析**：

| 标准 | 评分 | 说明 |
|---|---|---|
| C1 有成熟参考 | ★★★☆☆ | 概念清晰但不是游戏——"Loop+记忆泄露+觉醒阈值"需从头设计 |
| C2 兼容架构 | ★★★★★ | Host=Agent 在世界中生活，Loop=日常循环，玩家=叙事设计师 |
| C3 交互有后果 | ★★★★★ | 交互产生记忆碎片→积累影响觉醒→觉醒改变行为 |
| C4 题材差异 | ★★★★★ | "意识觉醒" vs "经济优化" |
| C5 行为更丰富 | ★★★★★ | 觉醒是独一无二的"元认知"维度 |
| C6 空间/地图 | ★★★★☆ | 公园有多个区域，但尺度不如骑砍大 |
| C7 AI 研究 | ★★★★☆ | Agent 元认知/自我意识是 AI safety 前沿话题 |
| C8 利用已有技术 | ★★☆☆☆ | **核心悖论：Loop 阶段浪费已有能力（Agent 被限制在脚本循环里），觉醒阶段需要不存在的能力（❌ 元认知）**。OpenClaw 匹配度弱——Loop 阶段每天汇报一样的内容 |

**关键风险**：如果觉醒机制设计不好，可能变成纯叙事包装（flavor text）而非真正的行为涌现。
