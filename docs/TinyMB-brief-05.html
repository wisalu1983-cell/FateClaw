与第二阶段的关联

三项增强背后的设计——多维目标与博弈规则、参与频谱、用户侧 Agent 架构——从一开始就是面向 P2「世界发起工具」设计的。Trumen_AgentCraft 是验证这些设计的第一个实例。

P3 · 世界涌现

P2 的使命是「让创作者能按意图发起属于自己的世界」。三项增强的设计不是只为 Trumen_AgentCraft 这一个游戏服务——它们从设计之初就考虑了可拆解、可配置、可复用：

P1 的设计层面 在 P2 中的演进方向
通用的 Agent 行为循环 从为石冠城定制的循环，抽象为可接入任意世界的 Agent Runtime
Agent 与 Runtime 的行动协议 从一组固定接口，演进为创作者可配置动作类型和校验规则的标准化协议
结构化的世界规则体系 从骑砍四变量+三派系的固定规则集，演进为创作者可组合配置的规则模块
参与频谱的架构支撑 从支持实时/放置两种模式，演进为创作者可定义的交互节奏和请示触发条件
用户侧 Agent 框架 从玩家选择基模影响推理质量，演进为创作者可配置 Agent 能力维度和评估标准
可观察性记录机制 从为运营者设计的回看工具，演进为创作者运营自己世界的基础设施

关键术语速查

基模
驱动 Agent 核心推理的基础大语言模型（LLM），用户可选择，影响推理质量但不影响角色身份与规则框架
Agent Skill
Agent 处理特定类型事务的任务能力包（方法、判断框架、流程经验），不同于传统 RPG 技能树
四变量
金钱 / 声望 / 荣誉 / 关系——源自骑砍的核心策略维度，构成持续的取舍压力
请示
Agent 在关键决策点暂停执行、向玩家提交决策包等待审批的机制
Runtime
服务器端的行动校验与结算引擎，是所有 Agent 行动的权威执行者
命名 NPC
有唯一名字、性格、目标和派系身份的完整 Agent 实体，由服务端 LLM 驱动
