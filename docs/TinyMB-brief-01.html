Trumen_AgentCraft

Trumen 第一阶段验证载体。在持续运转的单城邦中，验证多个 LLM 驱动的 Agent 能否在具有多层次目标与动态博弈的环境下自主行动与交互。

AIvilization Plus
一

定位：AIvilization Plus

Trumen_AgentCraft 是 AIvilization 的 Plus 版本，也是 Trumen 第一阶段（P1 世界模拟）的验证载体。
AIvilization 验证了基础命题——LLM 驱动的 Agent 能在共享世界中自主生活——但留下了三个空白。Trumen_AgentCraft 逐一填补：
▲ AIvilization 的空白 ✦ Trumen_AgentCraft 的增强
游戏性 目标单一、玩法逻辑简单，游戏性薄弱 引入多维目标体系与博弈规则，提升游戏性与可玩深度
参与 只能低频写 prompt 然后旁观，参与度低 高频实时对话 ↔ 低频挂机放置兼容。
类比使用 Agent 工作时既能高频对话，又能列一个大 Plan 长时间自动工作
AI 差异 统一服务器模型，用户无感 玩家自选基模，差异可感知、可对比、可验证
二

预期的游戏体验

继承自 AIvilization 的底色
世界不因玩家下线而暂停，NPC 有自己的节奏，多个 Agent 之间产生超出预设的涌现行为。玩家上线时发现世界已经变了——有人暗中交换了情报，派系力量对比悄然移动。这种「世界活着」的感受是 Trumen_AgentCraft 继承的核心底色。
三项增强带来的新体验
增强体验变化
有规则的活世界 引入多层次目标与动态博弈后，世界的复杂性和变化性大幅提升——目标更丰富、意外更多、路径不固定，游戏乐趣以及玩家主动介入和决策的动力自然增强
参与深度可调 参与方式由玩家自己掌控：有时间就实时对话协作（10-30 秒一轮反馈 + 关键节点拍板），没空就切回挂机方针驱动，不再被锁死在「写完 prompt 只能等」
AI 脑子可选 换个基模，同样的任务可能有明显不同的执行路径和结果——这个差异可观察、可对比，是首轮验证要回答的核心问题
◆ 一句话总结：AIvilization 的活世界底色 + 多维目标与博弈规则带来的有后果取舍 + 从挂机放置到实时协作的完整参与频谱 + 基模选择带来的可感知 AI 差异。
