三项增强与预期体验

概览页列出的三个空白（规则 / 参与 / AI 差异），对应以下三项增强。每张卡片说明这项增强做了什么、带来什么体验。

三项增强

一
有规则的活世界
✕ AIvilization 的行动没有结构化后果，Agent 的决策好坏无从衡量。

一座名为「石冠城」的自治城邦持续运转。三大派系——守备军团、商盟、平民公社——围绕税收、粮价、安全互相牵制。6 名有独立性格和目标的命名 NPC 各自行动：在城主堡巡视、在市集做买卖、在酒馆交换情报、在农庄护理民生。

每个行动都挂着代价：帮一方就会得罪另一方，接了高回报的灰色交易就要承担荣誉下降的后果，随从要吃饭要发饷，不维护就会流失。金钱、声望、荣誉、关系四个变量构成持续的取舍压力。

二
参与深度可调：低频放置 ↔ 实时协作
✕ AIvilization 只有一种节奏——每隔几小时写 prompt 然后旁观——玩家参与度低且参与方式单一。

TinyMB 把参与频谱从「只有低频放置」扩展为「低频到实时均可」，两种模式随时切换：

实时协作模式（类似用 Claude Code）
玩家和 Agent 实时对话协作。每步行动 10-30 秒一轮即时反馈，遇到重大决策（站队、高风险）Agent 暂停请示，玩家审查局面后拍板决定。
低频放置模式（类似 AIvilization 挂机）
玩家设定长期方针（经济优先级 / 派系倾向 / 风险偏好），Agent 自主行动，玩家隔一段时间回来翻日志、观察世界变化。
三
AI 脑子可选可感知
✕ AIvilization 中所有 Agent 共用服务器端同一个模型，用户无感知、无选择权。

就像 OpenClaw 接入不同基模时表现出明显不同的对话质量和推理深度，TinyMB 中的玩家 Agent 同样由一个基础大语言模型（基模）直接驱动。基模决定的是 Agent 的智力水平——同样的任务指令，接入更强基模的 Agent 会做出更深入的局面分析、更灵活的路径规划、更合理的取舍判断。

这个差异在 TinyMB 中对玩家可感知：你能在请示包和行动日志里看到 Agent 怎么思考、为什么选这条路、在哪里判断力不足。基模不影响角色是谁、遵守什么规则（这些由官方系统 prompt 锁定），它影响的是「脑子好不好使」。

这是首轮原型要显式验证的核心变量：同等任务条件下，不同基模产生什么样的差异？

预期的游戏体验

继承自 AIvilization 的底色
世界不因玩家下线而暂停，NPC 有自己的节奏，多个 Agent 之间产生超出预设的涌现行为。玩家上线时发现世界已经变了——有人暗中交换了情报，派系力量对比悄然移动。这种「世界活着」的感受是 TinyMB 继承的核心底色。
三项增强带来的新体验
增强体验变化
有规则的活世界 不是「看 Agent 做事」，而是「承担后果、据此调整方向」——帮了守备军团可能得罪商盟，接了灰色交易可能失去正直 NPC 的信任
参与深度可调 参与方式由玩家自己掌控：有时间就实时对话协作（10-30 秒一轮反馈 + 关键节点拍板），没空就切回挂机方针驱动，不再被锁死在「写完 prompt 只能等」
AI 脑子可选 换个基模，同样的任务可能有明显不同的执行路径和结果——这个差异可观察、可对比，是首轮验证要回答的核心问题
◆ 一句话总结：AIvilization 的活世界底色 + 骑砍式规则带来的有后果取舍 + 从挂机放置到实时协作的完整参与频谱 + 基模选择带来的可感知 AI 差异。
