# 行动系统设计（MVP v1）

> **文档用途**：定义 MVP 中所有可执行行动的分类、发起方式、执行流程、时长机制、环境约束、中断规则和目标层级系统。
>
> **确认状态**：A 级（已确认基线）——经过逐项讨论确认。
>
> **设计边界**：本文档定义"行动有哪些、怎么发起、执行多久、受什么约束、怎么中断"。战斗内部结算由文档 5 定义，AI 决策逻辑由文档 6 定义，情报内容规则由文档 4 定义。

---

## 一、行动发起流程

基于已确认的实时制+暂停模型，行动遵循以下流程：

| 步骤 | 说明 |
|---|---|
| 1. 暂停 | 玩家暂停游戏时间 |
| 2. 协商 | 玩家与 Servant 自然语言对话，讨论下一步行动 |
| 3. Servant 决定 | Servant 将对话中达成的意图转化为系统可识别的行动指令，自行提交 |
| 4. 恢复时间 | 玩家恢复时间流逝，行动开始执行 |
| 5. 执行中 | 行动消耗游戏内时间，长时间行动有中间进展推送 |
| 6. 结果产出 | 达到所需时长后产出结果，仅对相关方可见的部分推送给玩家 |

### 1.1 行动意图识别

游戏中不存在"行动菜单"。玩家通过自然语言协商表达意图，**由 Servant 自行判断并转化为行动指令**。

Servant 的判断受以下因素影响：
- 对话中 Master 明确表达的意图
- Master 设定的中长期导向目标（见第七节）
- Servant 自身的性格和战术倾向
- 当前局势信息

系统层面，Servant 提交的行动指令必须映射到本文档定义的行动分类体系中。

### 1.2 执行中信息推送

| 行动时长 | 推送方式 |
|---|---|
| 短时间行动 | 仅在完成时推送最终结果 |
| 长时间行动 | 有中间节点推送（如"正在接近目标区域……""发现可疑魔力反应……"） |

中间推送仅提供方向性信息，不中断时间流逝。玩家看到推送后可选择暂停做进一步决策。

## 二、行动分类体系

按行动性质分为 7 个大类：

| 大类 | 典型行动 | 执行主体 |
|---|---|---|
| **移动** | 在节点间移动 | Servant / Master / 同行 |
| **侦察** | 调查特定区域/目标、探查结界、跟踪 | Servant / Master |
| **社交** | 与 NPC 交涉、与其他 Master 接触、外交 | Master 为主 |
| **备战** | 结界设置/强化、据点防御、道具制作 | 取决于能力 |
| **战斗发起** | 主动交战、伏击、暗杀 | Servant 为主 |
| **特殊** | 宝具释放（常时型外）、令咒使用、灵体化切换 | 各自 |
| **休整** | 魔力恢复、伤势恢复、灵体化待机 | Servant / Master |

## 三、Master 三模式与行动约束

Master 的三模式指令系统直接约束行动可选范围：

| 模式 | Master 可执行 | Servant 可执行 | 协作方式 |
|---|---|---|---|
| **留守据点** | 据点内行动（休整、结界维护、远程魔术、情报整理） | 独立执行全部行动（移动、侦察、战斗等） | 念话协商，Master 无法物理参与 |
| **随从者行动** | Master 技能参战、近距离观察 | 全部行动 | Master 能力可直接生效（如 Avalon、宝石魔术） |
| **单独外出** | 社交、调查、特定地点行动 | 独立执行全部行动 | 念话协商，双方各自独立行动 |

### 3.1 分离时的并行行动

Master 与 Servant 分离时（留守据点 / 单独外出），双方各自拥有独立的行动时间线，可同时执行不同行动。一组最多同时存在两条行动线。

同行模式下，某些行动可协作执行（如 Master 魔术支援 + Servant 战斗），算作一个联合行动。

### 3.2 模式切换

模式切换需要游戏内时间（移动时间）。从"留守据点"切换到"随从者行动"需要 Master 移动到 Servant 所在位置，反之亦然。

## 四、行动时长机制

每个行动有一个**基础时长**，受以下因素修正：

| 修正因素 | 影响的行动类型 | 示例 |
|---|---|---|
| 敏捷参数 | 移动、追击、撤退 | 高敏捷缩短移动时长 |
| 职阶技能 | 特定行动类型 | Caster 阵地建造缩短结界设置时长 |
| 地图距离 | 移动 | 节点间移动时长取决于拓扑距离 |
| 载具/坐骑 | 移动 | Rider 有坐骑时大幅缩短移动时长 |

> 各行动的具体基础时长数值为后续调参任务，不在本文档确定。

---

## 五、环境状态对行动的影响

三个环境状态（日间/黄昏/夜间）**不限制行动种类**，而是改变行动的附加后果和风险权重：

| 环境 | 对行动的影响 |
|---|---|
| **日间** | 从者现界行动有暴露风险（平民目击）；战斗会严重违反神秘隐匿原则，引发监督者/抑止力介入；社交和非魔术调查更自然 |
| **黄昏** | 过渡期，暴露风险中等；适合移动和布置 |
| **夜间** | 暴露风险最低；战斗几乎不受神秘隐匿约束；某些从者（如 Assassin）在夜间有隐蔽加成 |

---

## 六、行动中断与冲突

实时制下，行动执行过程中可能被打断：

| 中断类型 | 触发条件 | 结果 |
|---|---|---|
| 遭遇中断 | 移动或侦察途中与敌方组发生遭遇 | 行动中断，进入对峙/战斗判定 |
| 被攻击中断 | 据点被袭击 | 据点内行动中断 |
| 令咒中断 | 空间转移/强制撤退 | 当前行动立即终止 |
| 玩家主动取消 | 暂停后取消尚未完成的行动 | 行动终止，已消耗时间不可回退 |

### 6.1 同节点遭遇规则

遭遇是否发生取决于双方的**隐蔽状态**：

| 情况 | 是否自动遭遇 |
|---|---|
| 双方均为非隐蔽状态（现界行动） | 自动遭遇 |
| 一方处于隐蔽状态（灵体化/气息遮断） | 非隐蔽方无法感知隐蔽方；需主动侦察或满足特定条件才能发现 |
| 双方均处于隐蔽状态 | 互相不可见，不遭遇 |

> 遭遇后的具体判定流程（对峙→交涉/战斗→脱离）由情报系统（文档 4）和战斗结算（文档 5）共同定义。

## 七、目标层级系统

玩家可设定两个层级的目标，驱动 Servant 的行为：

### 7.1 中长期导向目标

| 属性 | 说明 |
|---|---|
| 设定方式 | 玩家通过对话向 Servant 传达战略意图 |
| 示例 | "先摸清其他组的情报再行动""优先击败 Berserker 组""保持隐蔽，等对手消耗" |
| 持续性 | 长期有效，直到玩家明确更改 |
| 作用 | 当 Servant 需要自主决策时，作为行为导向的第一参考 |

### 7.2 短期行动目标

| 属性 | 说明 |
|---|---|
| 设定方式 | 玩家在暂停中与 Servant 协商具体行动 |
| 示例 | "去柳洞寺侦察""在新都布置结界""前往教堂与监督者对话" |
| 持续性 | 执行完毕即消耗 |
| 作用 | 直接转化为系统行动指令 |

### 7.3 无短期目标时的自主行动

当玩家未下达短期行动目标时（如玩家恢复时间流逝但未给出具体指令），Servant 将自主决定下一步行动，决策依据：

1. Master 设定的中长期导向目标
2. Servant 自身的性格、战术偏好和判断
3. 当前局势（已知情报、威胁评估）

自主行动同样遵循本文档的行动分类和规则。Servant 做出的自主决定会通过推送告知玩家（如"我打算前往新都侦察，是否有异议？"），玩家可暂停干预。

> Servant 自主决策的具体 AI 逻辑由文档 6（AI 行为与驱动设计）定义。

## 八、职阶特性对行动的机制影响

| 职阶 | 行动层面效果 |
|---|---|
| **Saber** | 对魔力：对魔术类侦察/攻击的被动抗性；骑乘：有坐骑/载具时可缩短移动时长 |
| **Archer** | 单独行动：失去 Master 供魔后可独立存在更长时间；节点内战斗拥有远程距离优势（先手、保持距离）；跨节点远程攻击仅限宝具级能力 |
| **Lancer** | 高敏捷加成移动和追击/撤退时长；对魔力：被动抗性 |
| **Rider** | 骑乘：有专属坐骑时大幅缩短移动时长，并获得骑乘状态属性加成；无坐骑时无额外效果 |
| **Caster** | 阵地建造：设置结界时长更短、效果更强；道具作成：可制作消耗道具（独有行动类型） |
| **Berserker** | 狂化（常驻）：基础参数永久强化，但完全丧失自主沟通、侦察、社交、策略等需要理性的行动能力；魔力维持消耗增大；所有行动决策由 Master 全权承担 |
| **Assassin** | 气息遮断：隐蔽行动中被发现概率大幅降低；暗杀：特殊战斗发起方式，效果按条件分档（见下节） |

### 8.1 Assassin 暗杀效果分档

暗杀判定成功后，实际效果取决于双方条件差异：

| 结果档位 | 触发条件倾向 | 效果 |
|---|---|---|
| **致命** | 目标为 Master 或极低耐久从者，且完全无防备 | 目标直接重伤或死亡 |
| **重创** | 目标从者无防备但耐久/幸运较高 | 严重伤害 + Assassin 获得撤退窗口 |
| **轻伤** | 目标有部分感知（直觉等技能触发） | 中等伤害，目标进入警戒，Assassin 暴露 |
| **失败** | 目标有强感知能力或已处于警戒状态 | 暗杀被识破，无伤害，Assassin 暴露，可能被反击 |

> 暗杀判定的具体参数（气息遮断等级 vs 对方感知/直觉/幸运）由战斗结算文档（文档 5）定义。

## 九、NPC 组行动规则

NPC 组（非玩家控制的 Master-Servant 组）遵循与玩家组相同的行动框架：

- 同样受 7 大类行动分类约束
- 同样受环境状态影响
- 同样遵循一组最多两条行动线的并行规则
- 同样受遭遇隐蔽规则约束

NPC Master 同样拥有三种行动模式（留守据点/随从者行动/单独外出），其选择由 AI 决策。

> NPC 组的具体 AI 决策逻辑由文档 6 定义。
