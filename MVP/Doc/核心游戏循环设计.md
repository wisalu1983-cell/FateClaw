# 核心游戏循环设计（MVP v1）

> **文档用途**：定义 MVP 中一局游戏从开始到结束的完整时间骨架与运行模式。所有时间推进、环境状态切换、信息分层、战争推进机制和终止判定以本文档为准。
>
> **确认状态**：A 级（已确认基线）——经过逐项讨论确认。
>
> **设计边界**：本文档只定义时间结构、运行模式、信息分层、战争推进机制和终止条件。具体行动列表、数值参数、战斗结算等由后续专项文档定义。

---

## 一、体验模型：实时制 + 随时暂停

MVP 的游戏运行模式采用**实时推进 + 玩家随时暂停**的模型。

| 特征 | 说明 |
|---|---|
| 时间流逝方式 | 游戏内时间自动、连续流逝，所有参战组在同一时间轴上同时行动 |
| 玩家操作方式 | 玩家在时间流逝过程中观察大地图，可随时暂停以与从者交流、调整策略、发布指令 |
| 暂停时的时间状态 | 时间完全静止（MVP 简化：暂停对话不消耗游戏内时间） |
| 行动的执行方式 | 玩家下达行动指令后恢复时间流逝，行动在游戏内时间中持续执行，到达所需时长后产出结果 |
| 结果的可见性 | 只有对玩家组可见的结果才会推送给玩家，其余信息保存在系统层 |

### 1.1 暂停对话的前置条件（MVP 简化版）

- 从者与 Master 处于**可联络状态**（默认随时可联络，具体不可联络情形后续讨论）
- 当前处于**可正常沟通的情境**（战斗中属于不可对话状态）

### 1.2 自动暂停

系统在特定条件下自动暂停游戏，确保玩家不会错过关键决策窗口。

- 具体触发条件后续详细讨论确定
- 游戏内开放给玩家自定义配置（可关闭或调整哪些事件触发自动暂停）

## 二、宏观时间结构

### 2.1 总体参数

| 参数 | 值 |
|---|---|
| 一局游戏的游戏内跨度 | 7+ 个游戏内日 |
| 标准流速（不含暂停） | 1 真实分钟 = 1 游戏内小时 |
| 流速调节 | 支持玩家调节（具体档位后续确定） |
| 一个游戏日的标准流逝时长 | 24 真实分钟 |
| 7 天的标准流逝时长 | 约 168 真实分钟（≈ 2.8 小时） |
| 含暂停的预估总体验时长 | 约 3.5-4.5 小时（取决于暂停频率和对话深度） |
| 游戏天数上限 | 不设刚性上限，但第 3 天后启动渐进式冲突激化机制 |

> 标准流速以"1 分钟 = 1 小时"为基准建立体验框架，后续根据实际体验调整流速或天数。

### 2.2 每日环境状态

每日分为 3 个时间段。时间段的切换是**世界环境状态的刷新点**，不是回合分界线。

| 时间段 | 游戏内时间 | 环境状态变化 |
|---|---|---|
| **白天** | 清晨至下午 | 平民活动密集 → 公共区域人流高 → 从者战斗暴露风险极高 → 社交场所（学园等）开放 |
| **黄昏** | 傍晚至入夜 | 人流递减 → 暴露风险开始降低 → 隐蔽行动窗口打开 |
| **夜晚** | 深夜至凌晨 | 人流极低 → 暴露风险最低 → 从者全力行动条件最优 → 灵脉活跃度变化 |

**关键原则**：三个时间段只改变世界环境状态和行动后果的权重，**不限制玩家可以做什么**。

- 白天战斗 → 可以，但承担极高的神秘暴露代价
- 夜间只调查不战斗 → 可以
- 黄昏发起外交 → 可以

时间段切换时，系统更新对应的环境状态信息。

## 三、游戏运行模式与玩家体验流

### 3.1 连续流模型

游戏不采用离散回合制，而是连续流：

```
                      ┌─── 暂停：与从者交流 / 调整策略 ───┐
                      │                                     │
游戏时间流逝 ──→ 地图实时演进 ──→ 事件/结果出现 ──→ 玩家响应 ──→ 继续流逝
    ↑               │                    │                           │
    │               │                    ↓                           │
    │          玩家观察地图        只推送对玩家可见的结果              │
    └────────────────────────────────────────────────────────────────┘
```

### 3.2 玩家在时间流逝过程中的体验

1. **观察**：在大地图上观察各方动态（在玩家视野范围内的）
2. **接收**：收到与自身相关的事件推送（遭遇、行动完成、情报获取等）
3. **暂停**：随时暂停以与从者商议、查看系统信息、决定下一步行动
4. **下达指令**：通过与从者对话或直接操作确定行动
5. **恢复**：恢复时间流逝，行动在游戏时间中执行
6. **行动完成**：经过行动所需的游戏内时长后，结果产出并呈现给玩家

### 3.3 行动执行节奏

主从协商 → 决定行动 → 执行 → 出结果的循环依然是游戏的基本行为循环，但它以**玩家自主节奏**驱动，而非系统强制推进。

每次行动的执行时长取决于行动类型（具体行动类型和时长由行动系统设计文档定义）。

## 四、信息分层

### 4.1 系统公开信息

系统层面维护的、对所有参战者可见的公共信息。

**设计原则**：在保证游戏循环能正常运转的前提下，尽可能缩小范围。

| 信息类别 | 内容 | 更新时机 |
|---|---|---|
| 当前时间与时段 | 游戏内时刻、白天/黄昏/夜晚 | 实时 |
| 从者退场公告 | 监督者公告形式 | 有从者退场时 |
| 全城级异常事件 | 所有参战者都能感知到的大规模魔力波动、异常破坏等 | 事件发生时 |

监督者告知的信息范围参照原作设定。

**信息展示方式**：

- 随时可查看（不需要暂停）
- 随时间流逝自动更新
- 有新变更时，新信息与旧信息有明确的视觉区分，让玩家快速识别增量

### 4.2 私有信息

各组通过自身行动获得的独有情报。只有执行了相关行动（调查、遭遇、战斗、社交等）后才会获得。

以下信息均属于私有信息，不在系统公开范围内：

- 各组据点位置
- 各从者的职阶、真名、能力
- 各 Master 的身份
- 战斗结果的详细内容（除非玩家在场）
- 组间关系和同盟状态

结果产出时，只有对玩家组可见的部分才推送给玩家。其他 AI 组的情报对玩家不可见（除非通过外交、情报交易等方式获取）。

## 五、战争推进机制

### 5.1 原作战争推进驱动因素分析

基于第五次冬木圣杯战争原作，战争推进的核心驱动力有 10 项：

| # | 驱动因素 | 原作表现 | 本质 |
|---|---|---|---|
| 1 | 监督者的暗中操纵 | 绮礼派 Lancer 全城侦察试探，利用监督者身份选择性泄露情报 | 有"导演"在暗中制造冲突机会 |
| 2 | 角色性格驱动的主动攻势 | 伊莉雅主动狩猎，慎二虚张声势出击 | 部分角色性格决定了主动挑起冲突 |
| 3 | Caster 的领地扩张与魔力掠夺 | 暗中布设魔力吸收阵，持续抽取市民魔力 | "公敌"行为持续恶化，引发联合讨伐 |
| 4 | 弱势方的铤而走险 | 慎二供魔不足，孤注一掷启动鲜血神殿 | 资源劣势推动冒险行为，冒险引发连锁冲突 |
| 5 | 情报的不对称积累 | 战斗暴露真名、跟踪回据点、社交试探 | 信息差缩小过程制造冲突窗口 |
| 6 | 从者契约的脆弱性 | Rule Breaker 可强制夺取从者 | 规则层特殊威胁，迫使各组优先处理 |
| 7 | 暗线角色的条件式介入 | 吉尔伽美什前期观察，后期主动介入 | 隐藏超强势力在特定条件下登场 |
| 8 | 资源的不可逆消耗 | 魔力消耗、令咒不可恢复、从者负伤 | 时间站在所有人对立面，必须寻求决断 |
| 9 | 神秘暴露压力 | 公共场所战斗暴露魔术世界秘密 | 外部约束推动加快结束战争 |
| 10 | 个人关系与情感纠葛 | 士郎-伊莉雅、凛-樱、Archer-士郎等 | 私人关系导致行为偏离纯理性博弈 |

### 5.2 驱动因素的系统映射

#### A. 通过 NPC AI 行为模板自然产生（基线行为，始终生效）

| 原作驱动 | 系统映射 |
|---|---|
| 监督者暗中操纵 | 绮礼/Lancer 组 AI：Lancer 开局阶段执行全城侦察 |
| 角色性格驱动的主动攻势 | 各组 AI 行为倾向锚点（伊莉雅"主动狩猎"、慎二"虚张声势"等） |
| Caster 的领地扩张 | Caster 组 AI：持续扩大魔力吸收范围，制造可感知异常 |
| 弱势方铤而走险 | AI 决策中，资源低于阈值时行动倾向从保守切换为激进 |
| 个人关系纠葛 | AI 关系矩阵中预设的特殊关系权重 |

#### B. 通过系统机制产生

| 原作驱动 | 系统映射 |
|---|---|
| 情报不对称积累 | 情报与遭遇系统：遭遇、战斗、调查逐步揭示信息 |
| 资源不可逆消耗 | 魔力系统：维持费 + 行动消耗 + 战斗消耗，令咒不可恢复 |
| 从者契约脆弱性 | Rule Breaker 机制 |
| 神秘暴露压力 | 暴露度系统 |

#### C. 通过条件触发器产生

| 原作驱动 | 系统映射 |
|---|---|
| 暗线角色条件式介入 | 吉尔伽美什出场条件（存活组数 / 特定事件触发） |

### 5.3 战争阶段的典型演进

| 阶段 | 典型时段 | 特征 | 主要驱动力 |
|---|---|---|---|
| 试探期 | 第 1-2 天 | 各组以侦察和定位为主，避免正面冲突 | 驱动 1（Lancer 巡逻）、驱动 5（情报积累初期） |
| 碰撞期 | 第 3-4 天 | 第一次重大冲突爆发，可能出现首个淘汰 | 驱动 2（性格驱动攻势）、驱动 3（Caster 扩张引发公愤）、驱动 4（弱势方铤而走险） |
| 激化期 | 第 5-6 天 | 幸存组矛盾激化，背叛和连锁冲突频发 | 驱动 6（Rule Breaker 威胁）、驱动 8（资源衰减）、驱动 5（情报差缩小） |
| 决战期 | 第 7 天或更早 | 终局对决 | 驱动 7（吉尔伽美什介入）、驱动 8（资源见底） |

> 以上阶段不是刚性绑定天数，而是由驱动因素的实际触发情况决定。

## 六、渐进式冲突激化机制

### 6.1 设计原则

- 所有激化机制均为 **debug 可配置项**（可独立开关、可调整启动阶段和强度系数）
- 默认测试策略：从尽量少干预的状态开始，观察纯自然演进效果，再逐项开启
- NPC 角色行为模板（第五节 A 类）是基线行为，不属于激化机制，始终生效

### 6.2 机制清单与默认启动阶段

| 机制 ID | 名称 | 默认启动阶段 | 说明 | Debug 配置项 |
|---|---|---|---|---|
| ESC-1 | Caster 威胁加速 | 一级（第 3 天后） | Caster 魔力掠夺在基线上额外加速，更快引发公愤 | 开关 / 启动天数 / 加速系数 |
| ESC-2 | 情报暴露率提升 | 二级（第 5 天后） | 行动中被发现的概率基础值提高，遭遇频率上升 | 开关 / 启动天数 / 提升幅度 |
| ESC-3 | NPC 攻击倾向强化 | 二级（第 5 天后） | NPC AI"主动出击"阈值降低 | 开关 / 启动天数 / 阈值偏移量 |
| ESC-4 | 资源压力加大 | 三级（第 7 天后） | 全局魔力恢复效率降低 / 维持消耗加大 | 开关 / 启动天数 / 压力系数 |
| ESC-5 | 叙事压力事件 | 三级（第 7 天后） | 触发叙事性压力事件（监督者最终通牒、魔术协会干预等） | 开关 / 启动天数 / 事件列表 |

### 6.3 强度递增节奏

| 时段 | 激化行为 |
|---|---|
| 第 1-3 天 | 无系统激化，纯角色基线行为驱动 |
| 第 3 天结束 | ESC-1 启动 |
| 第 5 天 | ESC-2、ESC-3 启动 |
| 第 7 天 | ESC-4、ESC-5 启动；ESC-1/2/3 强度提升一档 |
| 第 8 天起 | 所有已启动机制每天强度递增一档 |

> 每一档的具体幅度留待实际测试后校准。初始测试建议：全部机制关闭，观察 NPC 基线行为是否足以在 7 天内自然产生至少 2-3 次重大冲突和 1-2 次淘汰。如果不够，再依次开启 ESC-1 → ESC-2/3 → ESC-4/5。

## 七、终止与结局条件

### 7.1 游戏终止触发

| 条件 | 结果 |
|---|---|
| 仅剩一组存活 | 该组获胜，游戏结束 |
| 所有幸存组丧失战斗能力或主动弃权 | 评估最终胜者 |
| 玩家组被淘汰（从者灵核毁坏 / Master 死亡） | 游戏进入结局流程 |

### 7.2 玩家被淘汰后

玩家可选择：

- **方案 A**：查看结局摘要（系统模拟剩余战争走向，生成摘要）
- **方案 B**：以旁观者模式继续观看战争实时推进至终局（无操作权）

### 7.3 玩家结局类型

| 类型 | 触发条件 |
|---|---|
| 胜利 | 玩家组是最终存活者 |
| 生还退出 | 玩家主动弃权并成功前往教会庇护 |
| 败北 | 玩家组被淘汰 |
| 特殊结局 | 预留位 |
