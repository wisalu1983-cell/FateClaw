# 最小规则表（MVP v1）

> **文档用途**：定义 MVP 中圣杯战争的完整运行规则。所有时段推进、行动判定、遭遇触发、信息暴露、冲突结算、令咒效果和胜负判定以本文档为准。
>
> **设计原则**：规则足够承载"第五次冬木市圣杯战争 if 线"的体验，同时保持系统可实现性。宁可少但清晰，不可多而模糊。

---

## 一、时间结构

### 1.1 总体节奏

| 参数 | 值 |
|------|-----|
| 一局游戏的游戏内时长 | 约 7 天（可因淘汰速度缩短） |
| 每天的时段数 | 3 个时段 |
| 总时段数 | 约 21 个时段（可因提前决出胜者而缩短） |
| 每时段的真实游戏时长 | 约 5-7 分钟 |
| 一局总体验时长 | 约 2 小时 |

### 1.2 三个时段

| 时段 | 游戏内时间 | 特征 | 典型行动 |
|------|----------|------|---------|
| **白天** | 早晨至下午 | 人流高，战斗暴露风险极高。社交和情报收集的黄金期 | 调查、社交渗透、伪装、购买补给、在学校等场所接触其他 Master、与从者协商当日计划 |
| **黄昏** | 傍晚至入夜 | 人流减少，准备阶段。接头、跟踪、设置陷阱的窗口 | 战前准备、结界布置、跟踪目标、秘密接头、最终确认行动方案 |
| **夜晚** | 深夜至凌晨 | 人流极低，战斗主要发生时段。高风险高回报 | 从者交战、伏击、暗杀、据点突入、宝具释放、大规模冲突、善后处理 |

### 1.3 战争阶段

| 阶段 | 对应天数 | 特征 |
|------|---------|------|
| 试探期 | 第 1-2 天 | 各组保持警惕，以侦察和试探性接触为主。玩家在此期间建立初步情报网络 |
| 碰撞期 | 第 3-4 天 | 第一次重大冲突爆发，有组被淘汰或重创。临时同盟可能在此期间形成 |
| 激化期 | 第 5-6 天 | 幸存组之间矛盾激化。背叛、截胡、连锁冲突频发 |
| 决战期 | 第 7 天（或更早） | 最终对决。存活组数降至 2-3 组时自动触发决战节奏 |

> 注意：以上只是典型节奏。玩家的介入可能加速或延缓战争进程。

## 二、行动系统

### 2.1 每时段行动流程

每个时段按以下顺序处理：

```
1. 信息更新：玩家收到上一时段的结果（事件日志、状态变化、情报更新）
2. 协商阶段：玩家与己方 Servant 通过自然语言协商本时段行动
3. 行动决定：玩家确认最终行动方案
4. 全局执行：所有组（包括玩家组和 AI 组）的行动同时执行
5. 结算阶段：系统根据各组行动的交集进行判定（遭遇、暴露、冲突等）
6. 结果呈现：向玩家展示本时段的事件日志和状态变化
```

### 2.2 可选行动列表

每时段，玩家组可以选择 **1 个主行动** + 最多 **1 个副行动**。

#### 主行动

| 行动 | 适用时段 | 效果 | 风险 |
|------|---------|------|------|
| **移动** | 全时段 | 移动到相邻地点 | 移动过程中可能被其他组发现或遭遇 |
| **调查** | 全时段 | 收集当前地点或指定目标的情报 | 调查行动本身可能暴露己方存在 |
| **设伏** | 黄昏/夜晚 | 在当前地点设置陷阱或埋伏等待目标 | 如果目标未出现，浪费一个时段 |
| **攻击** | 主要在夜晚 | 主动发起对指定目标的攻击 | 高暴露风险；可能遇到意料之外的第三方 |
| **防御** | 全时段 | 在当前据点加强防御，恢复状态 | 放弃主动权 |
| **外交** | 全时段 | 尝试与指定组接触、谈判、结盟或传递信息 | 接触本身可能是陷阱；对方可能拒绝 |
| **撤退** | 全时段 | 从当前地点快速撤离到安全位置 | 撤退方向有限；可能被追击 |

#### 副行动（可选，不是必须）

| 副行动 | 效果 |
|--------|------|
| 令从者灵体化侦察 | 以灵体形态在附近区域巡逻，提高发现敌人的概率 |
| 补充魔力 | 在灵脉点恢复魔力（仅限灵脉地点） |
| 情报整理 | 复盘已有信息，可能获得新的推理结论 |
| 布置结界 | 在当前位置设置简单的感知或防御结界 |

### 2.3 从者自主性

玩家通过自然语言给出行动指示后，Servant agent 会：

| 情况 | Servant 反应 |
|------|-------------|
| 指令清晰且 Servant 认同 | 执行，并在执行中做出细节层面的自主判断 |
| 指令清晰但 Servant 不认同 | 表达异议并提出替代方案，等待玩家确认或使用令咒强制 |
| 指令模糊 | 追问细节，或基于自身判断补全指令细节 |
| 玩家不给指令 | Servant 根据自身性格和当前局势自主决定行动（如 Berserker 金时可能"先干为敬"） |
| 令咒强制 | 无条件执行，但事后关系值下降 |

## 三、遭遇系统

### 3.1 遭遇触发条件

当多组在同一时段处于同一地点（或相邻地点且在移动中），系统判定是否发生遭遇。

| 条件 | 遭遇概率 |
|------|---------|
| 双方都在同一地点 + 至少一方实体化 | 高 |
| 双方在相邻地点 + 都在移动中 | 中 |
| 一方设伏 + 另一方进入该地点 | 极高（设伏方获得先手） |
| 一方灵体化侦察 + 另一方实体化 | 中低（灵体方有概率发现但不被发现） |
| 双方都灵体化 | 低（从者间可能互相感知气息但不确定身份） |

### 3.2 遭遇类型

| 类型 | 触发条件 | 结果 |
|------|---------|------|
| **擦肩** | 双方都在行动但未主动搜索 | 双方各获得对方"有人在附近活动"的模糊信息 |
| **发现** | 一方主动调查或设伏成功 | 发现方获得对方的部分信息（位置、职阶推测） |
| **对峙** | 双方都发现对方 | 进入"对峙状态"，双方决定下一步（撤退/交涉/交战） |
| **突袭** | 一方设伏成功且目标进入 | 突袭方获得先手优势，目标方被迫应对 |
| **偶遇** | 在高人流地点的随机遭遇 | 双方突然发现对方，没有先手优势 |

### 3.3 遭遇后的选择

进入对峙或偶遇后，双方在下一个判定中可选择：

| 选择 | 效果 |
|------|------|
| 撤退 | 脱离接触，但对方可选择追击 |
| 交涉 | 尝试对话、谈判或传递信息 |
| 交战 | 进入冲突结算 |
| 观察 | 不主动行动，继续收集信息 |

## 四、信息与暴露系统

### 4.1 信息类型

每组拥有以下可被其他组逐步获知的信息：

| 信息层级 | 内容 | 暴露难度 |
|----------|------|---------|
| 存在感知 | "有一组未知主从在活动" | 低——通过遭遇即可获得 |
| 职阶推测 | 从者的职阶 | 中——通过战斗风格、武器特征推断 |
| 能力画像 | 大致实力范围、战斗风格 | 中——通过交战或详细观察 |
| 据点位置 | Master 的据点在哪里 | 中高——通过跟踪、调查 |
| Master 身份 | Master 是谁 | 高——通过社交渗透、跟踪、意外暴露 |
| 从者真名 | 从者的真实身份 | 极高——核心机密。通过战斗细节推理、情报交易、或宝具暴露 |
| 宝具情报 | 宝具的效果和发动条件 | 极高——通常只有在宝具被使用后才暴露 |

### 4.2 暴露触发规则

| 行为 | 暴露结果 |
|------|---------|
| 从者使用宝具（真名解放） | 从者真名暴露给在场的所有组 |
| 从者在高人流地点实体化战斗 | 提升全局"神秘暴露度"（见 4.3） |
| Master 在公共场所使用魔术 | Master 身份可能被观察者识别 |
| 被其他组跟踪回据点 | 据点位置暴露 |
| 在学校等社交场所的试探性接触 | 对方 Master 身份可能被推断 |
| Lancer 的全城巡逻 | Lancer 组获得其他组的"存在感知"信息 |
| 贞德（Ruler）的真名看破 | 直接获得对方从者真名（Ruler 特权） |

### 4.3 神秘暴露度（全局参数）

| 参数 | 说明 |
|------|------|
| 初始值 | 0 |
| 上限 | 100 |
| 触达上限后果 | 监督者强制介入，要求所有组更加谨慎行动；严重时可能引发外部势力（魔术协会）的干预——导致整体战争加速收束 |

**暴露度增加事件**

| 事件 | 暴露度增量 |
|------|----------|
| 在高人流地点发生从者战斗（白天） | +15~20 |
| 在高人流地点发生从者战斗（夜晚） | +5~10 |
| 在低人流地点发生从者战斗 | +1~3 |
| 使用对军/对城级宝具 | +10~20（取决于地点） |
| 普通人目击从者或魔术 | +3~5 |
| 建筑物被严重破坏 | +5~10 |

**暴露度减少事件**

| 事件 | 暴露度减量 |
|------|----------|
| 善后处理（记忆消除、痕迹清理） | -3~5 |
| 监督者介入处理 | -5~10 |
| 时间自然流逝（每天自动） | -2 |

## 五、冲突结算系统

### 5.1 冲突类型

| 类型 | 触发条件 | 规模 |
|------|---------|------|
| 小规模冲突 | 偶遇后双方选择交战 | 通常为 1v1 从者对决 |
| 中规模冲突 | 有预谋的攻击或伏击 | 可能涉及 Master 参战或多方介入 |
| 大规模冲突 | 多组同时卷入 | 混战，局势复杂 |

### 5.2 日志式结算

冲突不做回合制战斗，而是由系统根据双方状态一次性结算，产出日志。

**结算考虑因素**

| 因素 | 权重 | 说明 |
|------|------|------|
| 从者能力参数对比 | 高 | 筋力、耐久、敏捷等直接影响对战结果 |
| 职阶相性 | 中 | 某些职阶对特定对手有优势/劣势 |
| 宝具发动 | 高 | 宝具是战力决定因素。但使用宝具会暴露真名并消耗大量魔力 |
| 地形因素 | 中 | 地点的灵脉、隐蔽性、空间大小影响特定从者的发挥 |
| 先手/伏击 | 中高 | 突袭方获得显著优势 |
| Master 参与 | 低~中 | 大多数 Master 无法直接影响从者对决，但特殊 Master（葛木的暗杀拳）例外 |
| 令咒强化 | 高 | 令咒加持可瞬间大幅提升从者状态 |
| 当前伤势和魔力 | 高 | 伤势未恢复或魔力不足会显著削弱从者 |
| 信息优势 | 中 | 知道对方真名和宝具的一方可以制定针对性战术 |

### 5.3 结算输出

每次冲突结算后，系统产出一份战斗日志，包含：

```
【冲突日志】
- 起因：为什么打起来
- 参与者：哪些组卷入
- 关键动作：2-4 个影响胜负的关键时刻
- 宝具使用：是否有宝具被使用（及其后果）
- 结果：胜者/败者/平手/撤退
- 伤势变化：各从者的伤势状态更新
- 魔力消耗：各组的魔力消耗
- 暴露变化：有哪些信息因此被暴露
- 后续影响：对地点状态、周边地区、神秘暴露度的影响
```

### 5.4 从者伤势等级

| 等级 | 状态 | 影响 |
|------|------|------|
| 完好 | 无显著损伤 | 全力战斗 |
| 轻伤 | 有轻微损伤 | 战力下降约 10%，1-2 天可自然恢复 |
| 中伤 | 有显著损伤 | 战力下降约 30%，需要休养或魔力治疗 |
| 重伤 | 灵核受损 | 战力下降 50% 以上，需要大量魔力恢复。此状态下再次交战有灵核毁坏的风险 |
| 灵核毁坏 | 从者消亡 | 该组被淘汰 |

### 5.5 撤退规则

交战中任一方可选择撤退：

| 条件 | 撤退成功率 | 代价 |
|------|----------|------|
| 战斗初期（试探阶段） | 高 | 低——可安全脱离 |
| 战斗中期 | 中 | 可能在撤退中受到追击伤害 |
| 战斗后期（一方明显劣势） | 低 | 需要令咒或特殊能力才能安全撤退 |
| 使用令咒强制撤退 | 极高 | 消耗 1 划令咒，但几乎必定成功 |

## 六、魔力系统

### 6.1 魔力参数

| 参数 | 说明 |
|------|------|
| 最大值 | 100（玩家组基准） |
| 初始值 | 80（开局不是满魔力，因为召唤消耗了一部分） |
| 每时段自然恢复 | +3（基础恢复） |
| 灵脉点恢复加成 | +5~10（取决于灵脉强度） |

### 6.2 魔力消耗

| 行为 | 消耗 |
|------|------|
| 从者维持现界（每时段） | -2（灵体化时 -1） |
| 从者战斗（普通交战） | -5~10 |
| 从者使用宝具 | -15~30（取决于宝具等级） |
| Master 使用魔术 | -3~5 |
| 令咒使用 | 0（令咒本身含魔力，不额外消耗） |
| 设置结界 | -5 |
| 灵体化侦察 | -2 |

### 6.3 魔力耗尽后果

| 魔力值 | 状态 |
|--------|------|
| 50 以上 | 正常运作 |
| 20~50 | 从者性能下降（约 -20% 全参数） |
| 1~20 | 从者严重虚弱（约 -50% 全参数），无法使用宝具 |
| 0 | 从者无法维持现界，除非有独立行动技能，否则在数时段内消散 |

## 七、令咒系统

### 7.1 基本规则

| 参数 | 值 |
|------|-----|
| 初始令咒数 | 3 划 |
| 补充方式 | 无（MVP v1 不支持令咒补充。3 划用完即失去令咒） |
| 令咒耗尽后果 | 失去对从者的强制控制权。从者可能继续合作（如果关系好）或脱离控制 |

### 7.2 令咒用法

| 用法 | 效果 | 消耗 |
|------|------|------|
| **绝对命令** | 对从者下达无法违抗的命令。命令越具体越有效 | 1 划 |
| **从者强化** | 将令咒魔力灌注于从者，实现瞬间大幅强化 | 1 划 |
| **空间转移** | 令从者瞬间转移到指定位置（如转移到 Master 身边） | 1 划 |
| **强制撤退** | 在不利局面下强制脱离战斗 | 1 划 |

### 7.3 令咒对主从关系的影响

| 情况 | 关系影响 |
|------|---------|
| 合理使用（紧急救援、战术强化） | 无负面影响或轻微影响 |
| 违背从者意愿的命令 | 关系值显著下降。从者事后可能变得不配合 |
| 频繁使用 | 从者对 Master 的信任持续降低 |
| 完全不使用 | 无影响。但在关键时刻不用可能导致损失 |

### 7.4 Ruler 的特殊令咒（仅限选择贞德时）

如果玩家选择贞德作为 Servant：
- 贞德对第五次战争中每一骑从者拥有 **2 划令咒**
- 可以对任何从者使用绝对命令（如"停止战斗""撤退""保护某人"）
- 这是极其强大的外交和干预手段，但一旦使用会引起对方 Master 的强烈敌意

## 八、淘汰与退场

### 8.1 淘汰条件

| 条件 | 结果 |
|------|------|
| 从者灵核毁坏 | 该组淘汰。从者灵魂被小圣杯回收 |
| Master 死亡 | 从者失去魔力供给。除非有独立行动技能，否则逐渐消散 |
| Master 令咒全部耗尽 + 从者选择离开 | Master 失去参战资格 |
| Master 主动弃权 | 可前往冬木教会寻求庇护。从者视关系状态决定是否留下 |

### 8.2 Master 死亡规则

| 情况 | 触发条件 |
|------|---------|
| 被敌方从者直接攻击 | 从者对 Master 有压倒性战力优势，几乎必杀 |
| 被敌方 Master 暗杀 | 少数 Master（葛木、绮礼）有直接战斗能力 |
| 在冲突中受波及 | 从者交战时 Master 可能被牵连 |
| Servant 叛变 | 关系极度恶化时，从者可能攻击 Master |

> **玩家 Master 的死亡处理**：在 MVP 中，玩家 Master 被杀或从者灵核毁坏即为游戏结束（败北结局）。

### 8.3 NPC 组淘汰后的连锁影响

NPC 组被淘汰后：
- 其从者灵魂被小圣杯回收
- 其据点变为"废弃据点"，可能保留残留资源或情报
- 其同盟关系解散
- 其他组可能因为该组淘汰而重新调整策略
- 如果被淘汰的是关键组（如 Caster 组），柳洞寺的结界会瓦解，大圣杯所在地暴露

## 九、胜负判定

### 9.1 胜利条件

| 条件 | 说明 |
|------|------|
| **唯一胜者** | 最终只剩下一组 Master-Servant 存活时，该组获得圣杯 |
| **事实胜利** | 虽然还有存活组，但其余组已丧失战斗能力或主动弃权 |

### 9.2 玩家的结局类型

| 结局类型 | 触发条件 | 说明 |
|----------|---------|------|
| **胜利** | 玩家组是最后的存活者 | 玩家赢得圣杯 |
| **代理胜利** | 玩家组被淘汰或弃权，但玩家扶持/影响的组赢得最终胜利 | 玩家的行动改变了战争结局 |
| **败北** | 玩家组被淘汰，且无明确的代理胜利 | 标准失败 |
| **生还退出** | 玩家主动弃权并成功到达教会 | 玩家退出战争但保全性命 |
| **特殊结局** | 特定条件触发 | 预留位——如揭露圣杯污染真相等 |

### 9.3 系统不预设的内容

- 玩家的个人目标完全自由——系统不会在开局要求选择"自己赢"或"扶持谁赢"
- 玩家可以在战争过程中随时改变自己的目标
- 系统只记录和追踪玩家的行动及其后果，不做道德判断

## 十、NPC 组行为规则

### 10.1 AI 决策基础

每个 NPC 组在每个时段的行动由以下要素共同决定：

| 要素 | 权重 | 说明 |
|------|------|------|
| 角色模板（固定） | 高 | 该角色在原作中的目标、性格、行动偏好 |
| 当前局势（动态） | 高 | 已知情报、伤势状态、魔力余量、存活组数 |
| 关系矩阵（动态） | 中 | 与其他组的敌友关系和信任度 |
| 战争阶段（动态） | 中 | 试探期、碰撞期、激化期、决战期影响行动激进度 |

### 10.2 原作角色行为锚点

以下行为倾向来自原作设定，用于锚定 AI 决策的基线方向：

| 组别 | 核心行为锚点 |
|------|-------------|
| 士郎 + Saber | 被动防守为主，不会主动攻击无威胁的组。优先保护身边的人 |
| 凛 + Archer | 主动调查，理性评估，有条件合作。Archer 可能自主行动 |
| 慎二 + Rider | 虚张声势但实力不足。可能试图在学校展开 Blood Fort Andromeda |
| 伊莉雅 + Berserker | 主动寻找并压制其他组。优先试探士郎组 |
| 葛木 + Caster | 据守柳洞寺，持续扩大影响。可能使用 Rule Breaker 夺取其他从者 |
| 绮礼 + Lancer | Lancer 全城巡逻，绮礼暗中收集情报。后期可能主动出击 |
| Assassin | 固守柳洞寺山门。任何接近者触发对峙 |
| 吉尔伽美什 | 前期观望。对 Saber 有特殊兴趣。后期可能独立出击 |

### 10.3 NPC 对玩家的反应模型

NPC 组对第八组（玩家）的态度由以下因素动态计算：

| 因素 | 影响方向 |
|------|---------|
| 玩家的主动接触方式（友善/敌对/中立） | 直接影响初始态度 |
| 玩家从者的职阶和实力评估 | 实力强者更被重视/警惕 |
| 玩家与该组的利益冲突程度 | 利益冲突高则敌意高 |
| 共同敌人的存在 | 面对共同威胁时倾向合作 |
| 历史交互记录 | 帮助过则好感提升，背叛过则永久敌意 |

## 十一、特殊事件触发器

以下事件可在特定条件下被触发，为战争增加转折：

| 事件 | 触发条件 | 效果 |
|------|---------|------|
| Caster 的魔力吸收引发公愤 | 第 2-3 天自动触发 | 多组可能联合讨伐 Caster |
| 慎二在学校展开鲜血神殿 | 特定条件下触发 | 学校成为危机现场，多组卷入 |
| 伊莉雅主动接触士郎 | 第 1-2 天自动触发 | 士郎组面临 Berserker 的试探压力 |
| Lancer 发现第八组 | 取决于玩家行动 | 绮礼获知第八组存在 |
| 吉尔伽美什现身 | 后期或特定触发 | 战争格局剧变 |
| 监督者发布公告 | 神秘暴露度过高 | 全体组被要求更加谨慎，或针对某组发布悬赏 |
| 从者被 Rule Breaker 夺取 | Caster 成功接触其他从者 | 被夺从者转为 Caster 组控制 |

## 十二、关系系统（简化版）

### 12.1 玩家与己方 Servant 的关系

| 参数 | 初始值 | 范围 |
|------|--------|------|
| 信任度 | 50 | 0~100 |
| 好感度 | 50 | 0~100 |

**信任度影响**：
- 高信任（70+）：Servant 主动提供更多信息和建议，执行命令更积极
- 中信任（30-70）：标准配合
- 低信任（30 以下）：Servant 可能隐瞒信息、消极执行、或在关键时刻拒绝配合

**好感度影响**：
- 主要影响 Servant 的对话语气和态度
- 极低好感（10 以下）+ 低信任 + 令咒耗尽 = Servant 可能脱离控制

### 12.2 组间关系

每两组之间维护一个简单的关系状态：

| 状态 | 含义 |
|------|------|
| 未知 | 互不知道对方存在 |
| 察觉 | 知道对方存在但缺乏详细信息 |
| 中立 | 互相知晓但无明确敌友关系 |
| 合作 | 达成某种形式的协议或同盟 |
| 敌对 | 明确的敌对状态 |
| 战争 | 正在或准备直接交战 |
