# 情报与遭遇系统设计（MVP v1）

> **文档用途**：定义 MVP 中信息分层、情报获取方式、遭遇判定流程（含双层感知机制和 d20 判定规则）和隐蔽体系。
>
> **确认状态**：A 级（已确认基线）——经过逐项讨论确认。
>
> **设计边界**：本文档定义"谁知道什么、怎么获取情报、遭遇怎么判定"。遭遇后的战斗结算由文档 5 定义，AI 如何利用情报做决策由文档 6 定义。

---

## 一、信息分层体系

### 1.1 三层信息结构

| 层级 | 可见性 | 内容 | 更新方式 |
|---|---|---|---|
| **系统公开层** | 全参战者自动可见 | 当前游戏日/时段/环境状态、已退场组的简要信息、监督者公告 | 系统自动推送，实时更新 |
| **组私有层** | 仅本组可见 | 自身状态、本组获取的情报、历史行动记录 | 行动结果 / 遭遇结果自动写入 |
| **隐藏层** | 无人可见 | 其他组的位置/状态/行动/据点/真名等 | 只能通过侦察、遭遇、社交等行动获取，获取后转入组私有层 |

玩家打开信息面板看到的，只有系统公开层 + 自己组私有层的内容。

### 1.2 系统公开层具体范围

| 信息类型 | 公开内容 | 不公开的部分 |
|---|---|---|
| 时间与环境 | 当前游戏日、时段、环境状态 | — |
| 退场信息 | "某组已退场" + 简短退场原因（如"从者被消灭"） | 详细情报（退场经过、交战对手等）仅亲历者和监督者知晓 |
| 据点位置 | 不公开（按原作，各组据点为私有信息） | 监督者知道所有 Master 身份；部分据点可从常识推测（如柳洞寺灵脉） |

## 二、监督者公告系统

基于原作中言峰绮礼的监督者角色设计。监督者不主动广播，更接近"信息枢纽"——参战者可主动前往教堂询问。

### 2.1 公告类型

| 公告类型 | 触发条件 | 内容 | 分发方式 |
|---|---|---|---|
| **战争开始确认** | 第 7 位从者被召唤 | "圣杯战争正式开始" | 全体参战者感知（令咒共鸣） |
| **从者退场通知** | 某位从者被消灭 | "一位从者已退场"（不透露职阶和身份） | 全体参战者感知（圣杯反应） |
| **神秘隐匿违规警告** | 战斗造成严重公众暴露 | 警告违规方，限期处理 | 定向通知违规方；未处理则升级为全体可知 |
| **庇护所提供** | Master 失去从者 | 教堂向落败 Master 提供庇护 | 私下通知落败 Master |
| **紧急事态宣言** | 战争失控（如大规模魔术威胁平民） | 号召各组暂时合作处理共同威胁 | 全体参战者 |

### 2.2 监督者的信息优势

言峰绮礼作为监督者，知道所有 Master 的身份（监督者职责要求登记）。这一信息可被他选择性利用或泄露——这属于他作为暗参战者的特殊优势，不是系统公开信息。

## 三、情报种类与获取

### 3.1 情报分类

| 情报种类 | 示例 | 获取途径 |
|---|---|---|
| **位置情报** | 某组从者在哪个节点 | 侦察、跟踪、遭遇（感知级别） |
| **身份情报** | 从者职阶、真名 | 职阶：战斗交手即暴露；真名：宝具暴露（真名解放型） |
| **能力情报** | 技能、宝具效果、属性等级 | 技能：使用/激活后暴露；属性：战斗交手获取 |
| **状态情报** | 魔力水平、伤势、令咒剩余 | 仅遭遇时可见，不持久保存 |
| **据点情报** | 某组据点位置、防御程度 | 侦察、跟踪 |
| **关系情报** | 某两组是否合作/敌对 | 社交、观察行为模式 |

### 3.2 情报获取机制

| 行动 | 可获取的情报 | 执行主体 |
|---|---|---|
| **区域侦察** | 该节点内从者活动痕迹、结界存在 | Servant 或 Master |
| **定向跟踪** | 目标的移动路径和目的地 | Servant 为主 |
| **战斗交手** | 职阶（必然暴露）、已使用技能、属性（见 3.4） | 战斗参与方 |
| **宝具暴露** | 真名（真名解放型宝具使用时自动暴露给在场者） | 在场目击 |
| **社交/打听** | 冬木市异常事件、NPC 见闻 | Master 或 Servant（需不违反神秘隐匿原则，Berserker 除外） |
| **从者知识** | 某些从者可能认识其他英灵，提供线索 | Servant 主动分享或玩家询问 |

### 3.3 情报时效性

| 情报种类 | 时效规则 |
|---|---|
| 位置情报 | 标记最后更新时间，不主动修改内容，由玩家/AI 自行判断可靠性 |
| 身份/能力情报 | 稳定，不衰减 |
| 状态情报 | **不保存**——仅在遭遇时实时可见，脱离接触后不再显示 |
| 据点/关系情报 | 标记获取时间 |

### 3.4 战斗中的情报获取

| 战斗类型 | 获取内容 |
|---|---|
| **短暂交锋**（接触后迅速脱离） | 职阶直接暴露；随机揭示 1~2 项属性等级 |
| **完整战斗**（持续对抗至一方撤退/倒下） | 职阶直接暴露；全部 6 项属性精确揭示；已使用的技能暴露 |

技能/宝具仅在实际使用或激活后暴露。未使用的技能对方无法得知。

## 四、隐蔽体系

### 4.1 灵体化与气息遮断的区别

| 维度 | 灵体化（所有从者） | 气息遮断（Assassin 职阶技能） |
|---|---|---|
| 对普通人 | 不可见 | 不可见 |
| 对其他从者 | 降低被感知概率，但可被探测 | 高等级接近不可探测 |
| 攻击时 | 需解除灵体化才能攻击 | 等级大幅下降，暴露风险急增 |
| 主要用途 | 降低魔力消耗、快速移动、对人隐身 | 专业隐匿：侦察、暗杀、潜入 |
| 对自身感知的影响 | 削弱与现世的交互能力，探测能力小幅下降 | 不影响自身感知 |
| 可否叠加 | — | 可与灵体化叠加 |

### 4.2 灵体化对自身探测能力的影响

灵体化状态下从者与现世的交互能力减弱，自身探测能力受到小幅削弱：**处于灵体化状态的探测方，roll -1**。

这意味着灵体化是一把双刃剑：更难被发现，但自己也更难发现别人。

## 五、遭遇判定系统

### 5.1 判定触发条件

当两个不同组的单位处于同一地图节点时，系统为双方各自独立进行一次探测判定。

### 5.2 双层感知判定

一次 d20 roll，对比两个门槛：

| 结果 | 条件 | 获得的信息 |
|---|---|---|
| **无感知** | roll + 探测修正 < 察觉门槛 | 什么都不知道 |
| **察觉** | 察觉门槛 ≤ roll + 探测修正 < 感知门槛 | 知道"此区域有从者存在"，不知道是谁、在哪 |
| **感知** | roll + 探测修正 ≥ 感知门槛 | 明确对象身份（至少职阶级别）和位置 |

### 5.3 判定公式

```
探测结果 = d20 + 探测方修正 + 探测方状态修正
察觉门槛 = 5 + 目标隐蔽修正 + 区域修正
感知门槛 = 12 + 目标隐蔽修正 + 区域修正

探测结果 ≥ 感知门槛 → 感知
探测结果 ≥ 察觉门槛 → 察觉
否则 → 无感知
```

### 5.4 基础门槛

| 门槛 | 基础值 | 设计标杆 |
|---|---|---|
| 察觉 | **5** | 无修正下 80% 成功率（对现界目标） |
| 感知 | **12** | 无修正下 45% 成功率（对现界目标） |

### 5.5 目标隐蔽修正（加在门槛上）

| 目标隐蔽状态 | 察觉修正 | 感知修正 |
|---|---|---|
| 现界（实体化） | +0 | +0 |
| 灵体化 | +4 | +2 |
| 气息遮断 D | +6 | +4 |
| 气息遮断 C | +8 | +5 |
| 气息遮断 B | +10 | +6 |
| 气息遮断 A | +12 | +8 |
| 气息遮断 A+ | +14 | +10 |

灵体化与气息遮断**叠加**计算。

### 5.6 完整门槛速查表

以下为零探测修正、零区域修正下的裸门槛值（括号内为 d20 成功率）：

| 目标状态 | 察觉门槛 | 感知门槛 |
|---|---|---|
| 现界 | 5（80%） | 12（45%） |
| 灵体化 | 9（60%） | 14（35%） |
| 气息遮断 D | 11（50%） | 16（25%） |
| 气息遮断 C | 13（40%） | 17（20%） |
| 气息遮断 B | 15（30%） | 18（15%） |
| 气息遮断 A | 17（20%） | 20（5%） |
| 气息遮断 A+ | 19（10%） | 22（不可能） |
| 灵体 + 气息遮断 D | 15（30%） | 18（15%） |
| 灵体 + 气息遮断 C | 17（20%） | 19（10%） |
| 灵体 + 气息遮断 B | 19（10%） | 20（5%） |
| 灵体 + 气息遮断 A | 21（不可能） | 22（不可能） |
| 灵体 + 气息遮断 A+ | 23（不可能） | 24（不可能） |

### 5.7 探测方修正（加在 roll 上）

| 修正来源 | 效果 | 示例 |
|---|---|---|
| 探测相关技能 | roll + X | 直觉 A（+2）、千里眼（+3）、对魔力感知 |
| 灵体化状态 | **roll -1** | 灵体化削弱与现世交互能力 |

> 各从者/Master 的具体探测修正值为后续定值任务。

### 5.8 区域修正（加在门槛上）

| 区域特征 | 门槛修正 | 说明 |
|---|---|---|
| 开阔/狭小区域 | -1 ~ -3 | 不易躲藏，容易发现彼此 |
| 大型/复杂区域 | +1 ~ +3 | 区域大、地形复杂，不易碰面 |
| Caster 领地结界内 | -3 ~ -5 | 结界主动探测入侵者 |

> 各节点的具体区域修正值为后续定值任务。

### 5.9 双方判定的交叉结果

双方各自独立 roll，产出六种组合：

| A 对 B | B 对 A | 场景效果 |
|---|---|---|
| 感知 | 感知 | **双方对峙**：互相发现，可选攻击/交涉/撤退 |
| 感知 | 察觉 | A 知道 B 的身份和位置；B 只知道有人在附近 |
| 感知 | 无感知 | **A 完全掌握主动权**：可选攻击（先手）/观察/撤退，B 毫无察觉 |
| 察觉 | 察觉 | 双方都隐约感到有人，但谁也不知道对方是谁 |
| 察觉 | 无感知 | A 知道有人在但不确定是谁，B 完全不知 |
| 无感知 | 无感知 | **不遭遇**：互不知情 |

### 5.10 察觉状态的游戏效果

| 效果 | 说明 |
|---|---|
| 通知推送 | "你感觉到此区域有其他从者的气息" |
| 可触发后续行动 | 玩家可暂停，下令 Servant 进一步侦察（额外 roll 尝试升级为感知） |
| 对方不知道你察觉了 | 察觉是被动感应，不暴露探测方 |
| 警戒标记 | 该区域在组私有层标记为"有未识别从者活动" |

## 六、遭遇后的流程分支

```
遭遇发生（至少一方达到感知）
  │
  ├─ 双方感知 → 进入"对峙"状态
  │    ├─ 任一方选择攻击 → 进入战斗（文档 5）
  │    ├─ 双方选择交涉 → 进入社交/外交
  │    └─ 任一方选择撤退 → 撤退判定（敏捷对抗）
  │
  └─ 单方感知
       ├─ 感知方选择攻击 → 先手优势进入战斗
       │    ├─ 满足暗杀条件（Assassin） → 按暗杀分档判定
       │    └─ 一般攻击 → 进入战斗，攻击方获得先手
       ├─ 感知方选择观察 → 获取情报但不暴露自身
       └─ 感知方选择撤退 → 安全撤离（对方未察觉）
```

仅达到"察觉"级别的一方不会自动进入遭遇流程，但可以选择主动侦察尝试升级为感知。

## 七、气息遮断的攻击衰减

Assassin 发起攻击时，气息遮断等级大幅下降：

| 原等级 | 攻击时等级 |
|---|---|
| A+ | → D |
| A | → E（接近无效） |
| B | → E |
| C | → 无 |
| D | → 无 |

这意味着 Assassin 暗杀的"窗口"是：暗杀判定发生在高等级气息遮断下，但一旦进入实际交战，隐蔽优势迅速消失。
