# 网页信息架构（MVP v1）

> **文档用途**：定义 MVP 网页界面的区域划分、信息层级、交互流程和视觉优先级。
>
> **设计原则**：默认以地图为主体（策略视角），IM 对话区可展开至与地图等分（对话沉浸视角）。玩家在任何时刻都能清楚回答三个问题：我在哪、发生了什么、我能做什么。

---

## 二、各区域详细定义

### 2.1 顶部信息栏

始终固定在顶部，不随滚动变化。显示玩家最需要持续关注的全局状态。

| 元素 | 内容 | 交互 |
|------|------|------|
| 时间显示 | 「第 X 天 · 白天/黄昏/夜晚」 | 纯展示 |
| 魔力条 | 当前魔力值 / 最大值，以进度条形式展示 | 悬浮显示详细数值和恢复预期 |
| 令咒指示 | 以三个图标形式展示剩余令咒数 | 点击可触发令咒使用面板 |
| 时段推进 | 「推进到下一时段」按钮 | 点击后确认行动方案并进入结算 |
| 战争进度 | 存活组数指示（如「存活：5/8 组」） | 悬浮显示已知淘汰组信息 |

### 2.2 冬木地图区

默认占主体空间。以简化的地点网络形式呈现冬木 11 个节点。

#### 地图元素

| 元素 | 表现形式 | 交互 |
|------|---------|------|
| 地点节点 | 带标签的圆形/方形节点，大小反映重要性 | 点击节点查看地点详情 |
| 连接线 | 地点之间的连接路径 | 悬浮显示路径描述 |
| 己方位置 | 高亮标记（如发光边框或特殊颜色） | — |
| 己方 Servant 位置 | 从者图标（如果与 Master 分开行动） | — |
| 已知敌方位置 | 半透明敌方图标（基于情报准确度显示） | 点击查看已知情报 |
| 未知活动 | 地点边缘的「?」标记（检测到异常但身份未知） | — |
| 地点状态标签 | 结界、战斗痕迹、异常等标签 | — |
| 时段指示 | 地图整体色调随时段变化（白天明亮 / 黄昏暖橙 / 夜晚深蓝） | — |

#### 地图交互

| 操作 | 效果 |
|------|------|
| 点击地点节点 | 弹出地点详情面板：地点功能、当前已知状态、可执行行动 |
| 点击地点后选择「移动至此」 | 将"移动到该地点"设为本时段主行动 |
| 点击敌方图标 | 弹出该组已知情报摘要 |
| 地图缩放/拖拽 | 基本的地图操作（如果节点数多需要的话。11 节点可能不需要） |

### 2.3 IM 对话区

承载玩家与己方 Servant 的自然语言交互。

#### 收起态（默认）

| 元素 | 内容 |
|------|------|
| Servant 头像 + 名称 | 顶部固定显示 |
| 最近消息 | 显示最近 2-3 条对话消息 |
| 输入框 | 玩家输入自然语言 |
| 展开按钮 | 点击后 IM 区扩展至 50% 宽度 |

#### 展开态

| 元素 | 内容 |
|------|------|
| Servant 头像 + 名称 + 状态摘要 | 顶部固定。状态摘要包括伤势、情绪、关系指示 |
| 完整对话历史 | 可滚动的完整对话记录 |
| Servant 状态卡 | 可折叠的面板，显示从者当前参数、伤势、技能可用状态 |
| 输入框 | 玩家输入自然语言 |
| 快捷指令 | 常用指令的快捷按钮（如「分析当前局势」「建议行动」「报告状态」） |
| 收起按钮 | 点击后 IM 区回到 35% 宽度 |

#### IM 消息类型

| 消息类型 | 视觉区分 | 内容 |
|----------|---------|------|
| 玩家输入 | 右侧气泡 | 玩家的指令、问题、协商 |
| Servant 回复 | 左侧气泡 | Servant 的分析、建议、执行反馈、情绪表达 |
| 系统通知 | 居中灰色条 | 时段推进通知、遭遇通知、战斗开始/结束通知 |
| Servant 主动消息 | 左侧气泡 + 特殊标记 | Servant 基于当前局势主动发起的对话（如发现异常、提出建议） |

### 2.4 底部面板区

以标签页形式切换，承载详细信息。

#### 标签页 1：事件日志

按时间倒序显示本局已发生的所有事件。

| 日志条目类型 | 视觉样式 | 内容示例 |
|-------------|---------|---------|
| 移动记录 | 普通文本 | 「你移动到了新都商业区」 |
| 遭遇事件 | 警告色高亮 | 「在深山町街道，你感知到一股从者的气息，但来源迅速消失了」 |
| 冲突日志 | 红色高亮，可展开 | 展开后显示完整的战斗日志（起因、关键动作、结果、后果） |
| 情报获取 | 蓝色高亮 | 「通过本次遭遇，你推断对方从者职阶为 Lancer」 |
| NPC 行动 | 灰色文本 | 「远处传来了爆炸声，似乎新都方向发生了战斗」（间接感知到的 NPC 行动） |
| 关系变化 | 紫色高亮 | 「Servant 对你的信任度提升了」 |

#### 标签页 2：情报面板

汇总玩家目前已知的所有情报。

| 分区 | 内容 |
|------|------|
| 各组情报卡 | 对每个已知组显示一张情报卡：已知职阶、已知/推测真名、已知据点、已知能力特征、敌友状态 |
| 未确认情报 | 尚未验证的推测和传闻 |
| 地点情报 | 各地点的已知状态和异常情况 |
| 全局态势摘要 | 系统自动生成的一句话当前态势描述 |

#### 标签页 3：行动面板

每时段开始时，玩家在此选择行动。

| 区域 | 内容 |
|------|------|
| 主行动选择 | 可选主行动列表（移动/调查/设伏/攻击/防御/外交/撤退），每个选项附带简要说明和风险提示 |
| 副行动选择 | 可选副行动列表（灵体侦察/补充魔力/情报整理/布置结界） |
| 行动目标指定 | 对于需要指定目标的行动（如攻击、外交），提供目标选择 |
| 行动确认 | 汇总当前选择，点击确认后锁定本时段行动 |
| Servant 意见 | 显示 Servant 对当前行动选择的态度（赞同/反对/建议） |

## 三、核心交互流程

### 3.1 单时段完整流程

```
┌─────────────────────────────────────────────┐
│ 1. 信息更新                                  │
│    - 事件日志自动更新                         │
│    - 地图状态刷新                             │
│    - Servant 可能主动发消息                   │
├─────────────────────────────────────────────┤
│ 2. 玩家浏览态势                              │
│    - 查看地图                                │
│    - 阅读日志                                │
│    - 查看情报面板                             │
├─────────────────────────────────────────────┤
│ 3. 与 Servant 协商                           │
│    - 在 IM 区进行自然语言对话                 │
│    - 讨论当前局势和行动方案                   │
│    - Servant 可能提出异议或替代方案            │
├─────────────────────────────────────────────┤
│ 4. 选择行动                                  │
│    - 在行动面板选择主行动和副行动              │
│    - 或通过 IM 直接告诉 Servant 行动计划       │
│    - Servant 确认或追问                       │
├─────────────────────────────────────────────┤
│ 5. 确认并推进                                │
│    - 点击「推进到下一时段」                   │
│    - 系统执行全局结算                         │
│    - 进入下一时段的步骤 1                     │
└─────────────────────────────────────────────┘
```

### 3.2 行动选择的两种输入方式

| 方式 | 说明 |
|------|------|
| 结构化选择 | 在行动面板中点击选项、选择目标、确认 |
| 自然语言指令 | 在 IM 中直接告诉 Servant（如"今晚我们去港口埋伏"），系统解析为对应行动 |

两种方式等价，玩家可以自由选择。自然语言指令会被系统解析后在行动面板中自动填充，玩家可以二次确认或修改。

### 3.3 令咒使用流程

| 步骤 | 操作 |
|------|------|
| 1 | 点击顶部令咒图标，或在 IM 中输入令咒相关指令 |
| 2 | 弹出令咒使用面板：选择用途（绝对命令/从者强化/空间转移/强制撤退） |
| 3 | 如果是绝对命令，输入具体命令内容 |
| 4 | 系统显示预期效果和代价提示 |
| 5 | 玩家二次确认 |
| 6 | 执行，IM 中显示 Servant 的反应 |

### 3.4 冲突发生时的特殊流程

当结算阶段检测到冲突（遭遇升级为交战）时：

| 步骤 | 内容 |
|------|------|
| 1 | 屏幕提示「冲突发生」，地图上高亮冲突地点 |
| 2 | 事件日志显示冲突起因 |
| 3 | 如果玩家组直接参与冲突，弹出紧急决策窗口 |
| 4 | 紧急决策选项：全力战斗 / 保守应对 / 使用宝具 / 使用令咒 / 撤退 |
| 5 | Servant 在 IM 中同步给出战场判断和建议 |
| 6 | 玩家做出选择 |
| 7 | 结算完成，显示完整战斗日志 |

> 如果冲突发生在玩家未直接参与的地点，玩家只会在日志中看到间接信息（如"远处传来爆炸声"）。

## 四、视觉风格方向

### 4.1 总体基调

| 维度 | 方向 |
|------|------|
| 色调 | 暗色系为主（深蓝/深灰底），随时段切换微调（白天偏亮、夜晚偏暗紫） |
| 风格 | 介于"战术地图"和"视觉小说"之间。不是 pixel art，也不是 3D 场景 |
| 字体 | 清晰的无衬线体，关键信息（战斗、令咒）用特殊字体或颜色强调 |
| 地图风格 | 简约的矢量地图 + 地点图标，不需要精细场景绘制 |
| 角色呈现 | 从者以头像/半身像 + 状态卡形式呈现，不需要全身 Live2D |

### 4.2 氛围要素

| 元素 | 说明 |
|------|------|
| 时段转换动画 | 简短的色调渐变 + 文字提示（「——夜幕降临——」） |
| 冲突发生提示 | 屏幕边缘红色闪烁 + 震动效果 |
| 令咒使用 | 令咒图标发光消失 + 特殊音效 |
| 从者淘汰 | 地图上该组标记消失 + 日志中的叙事性描述 |

## 五、响应式设计注意事项

| 设备 | 适配方案 |
|------|---------|
| 桌面（1920×1080+） | 标准布局如上 |
| 平板（横屏） | 地图和 IM 可切换全屏显示 |
| 手机 | 不作为 MVP 主要目标设备。如需适配，改为全屏切换模式（地图/IM/面板三个全屏页签） |
