# FateClaw Session Orchestration

## 何时使用

出现以下任一情况时使用本 skill：

- 继续执行已有主计划。
- 在新 session 中领取某个既有任务。
- 执行过程中出现复杂阻塞，需要开子计划。
- 一个 session 需要把任务交接给另一个 session。
- 同一项目存在多个活跃 session，需要同步状态与避免冲突。

## 上下文包

开始任何多 session 工作前，按以下顺序读取：

1. `<项目>/MVP/Management/项目状态索引.md`
2. `<项目>/MVP/Management/当前执行计划.md`
3. `<项目>/MVP/Management/活跃会话总表.md`
4. `<项目>/MVP/Management/待解决问题清单.md`
5. 相关 `<项目>/MVP/Management/子计划/*.md`
6. 相关 `Scope/`、`Design/` 文档

如果任务属于多步、跨会话或会跨多个窗口执行，而 `当前执行计划.md`、`活跃会话总表.md` 或 `子计划/` 尚不存在，先创建最小骨架，再进入执行。

## 五种操作模式

### 1. 建立或刷新主计划系统

- 确认当前项目路径。
- 确认 `当前执行计划.md` 是否存在且唯一。
- 确认 `活跃会话总表.md` 是否存在。
- 若缺少 `子计划/`，创建目录骨架。
- 不在同一项目中同时维护两份主计划。

### 2. 领取 session 任务

- 先检查 `活跃会话总表.md` 是否已有同一父任务或同一文件区域的活跃占用。
- 若存在冲突，停止并显式提示，不得默认并行推进。
- 若无冲突，在 `活跃会话总表.md` 中新增或更新当前 session 条目。
- 一个 session 任一时刻只领取一个任务或一个子计划。

### 3. 开子计划

执行中出现新问题时，先分类：

- 局部细节：并入当前任务，不单开子计划。
- 复杂阻塞：新建 `子计划/*.md`，并挂靠父计划、父任务。
- 范围或架构变化：停止实现，先回写主计划并 replan。

仅当问题需要单独验证、预计持续较长、或会独立交接给别的 session 时，才开子计划。

### 4. 同步或交接

- 把当前阶段结论写回 `当前执行计划.md`。
- 更新 `活跃会话总表.md` 中的 `最后同步时间`、`下一步`、`阻塞说明`。
- 如果问题升级为项目级待决项，更新 `待解决问题清单.md`。
- 如果形成了稳定可复用口径，按项目规则更新 `开发过程问答.md`。
- 如果当前有效状态改变，按项目规则更新 `项目状态索引.md`。

### 5. 关闭 session

- 先完成所有必要回写。
- 如果本次有实质推进，更新 `开发过程日志.md`。
- 回写完成后，再把该 session 从 `活跃会话总表.md` 中移除，或改为非活跃状态后立即清理。
- 不把 `活跃会话总表.md` 当作历史归档使用。

## 最小模板

### `活跃会话总表.md` 条目

```markdown
| Session ID | 任务标题 | 父计划 | 父任务 | 关联 Q-xx | 状态 | 最后同步时间 | 下一步 | 阻塞说明 |
| ---------- | -------- | ------ | ------ | --------- | ---- | ------------ | ------ | -------- |
| S-20260402-A | P2 观察包接口梳理 | 当前执行计划.md | Task 2 | Q-09 | 进行中 | 2026-04-02 14:30 | 回写接口边界 | 无 |
```

### `子计划/YYYY-MM-DD-<topic>.md` 头信息

```markdown
# <子计划标题>

- 父计划：`当前执行计划.md`
- 父任务：`Task N`
- 关联 Q-xx：`Q-09`
- Session ID：`S-20260402-A`
- 状态：`进行中`
- 目标：一句话说明本子计划解决什么
- 关闭条件：满足什么就算子计划完成
```

## 执行守则

- 不创建第二份主计划。
- 不把普通 session 级 todo 直接升级成 `Q-xx`。
- 不让子计划脱离父计划单独存在。
- 不在未回写的情况下直接关闭或交接 session。
- 如果用户要求跳过同步，按项目规则先提示风险并获得确认。

## 典型触发示例

用户说：

- “继续上次 TinyMB 的计划。”
- “我开个新窗口接手这个任务。”
- “这个问题太复杂了，单独开个 plan 解决。”
- “把这个 session 的进度同步一下。”
- “把这个任务交给另一个 session。”

以上场景都应先调用本 skill，再执行具体修改。
