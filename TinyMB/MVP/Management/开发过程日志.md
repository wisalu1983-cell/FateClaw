### 完成 P2：定稿结构化行动意图格式（workshop 阶段 2）

- 正式将首轮原型的行动输出契约定为**混合方案**
- 当前统一口径为：
  - `Runtime` 权威执行时只认一个 `primary_action`
  - 行动意图可同时携带 `followup_plan`、`interrupt_points`、`capability_basis`、`capability_gap` 与 `fallback`
  - `followup_plan` 只允许表达 1~2 条下一步打算，不退化为完整可执行任务队列
- 当前明确的四类主动作：
  - `move`
  - `act`
  - `ask`
  - `wait`
- 当前明确的关键边界：
  - 若当前不在目标区域，主动作必须先提交 `move`
  - 若已命中重要决策 / 高风险边界，不直接提交高风险 `act`，而改为 `ask`
  - 背景字段不由 `Runtime` 自动串行执行，主要用于请示、日志、回放、调试与后续基模 / skill 差异验证

### 调整阶段 2 的 skill 主线推进顺序（workshop 阶段 2）

- 基于对多 Agent 框架、AIvilization skill 机制和 TinyMB 当前玩法设计的复核，确认**具体 agent skill 设计高度依赖最终玩法形态、时间结构、行动粒度与首个可玩版本的真实试玩结果**
- 因此对 `Q-09` / `Q-10` 的推进口径做出调整：当前阶段**不提前定稿具体 skill 清单**，只保留候选方向；先完成 skill 相关前置任务与无具体 skill 的可玩基线
- 明确将 skill 相关开发任务拆分为三层：
  - **前置任务**：锁定玩法与接口地基（时间结构、行动粒度、观察包、结构化行动意图、日志 / 请示字段），跑通无具体 skill 的可玩基线，并固定基准模型做首轮试玩
  - **具体 skill 设计**：基于试玩结果回收高频行为、常见失败点与可感知差异，再收敛首版 skill 清单与作用边界
  - **skill 接入与验证**：在清单收敛后，再做预制库 / 槽位限制 / 配置切换、日志与请示接入，以及同模型有 / 无 skill 对照和跨模型对照验证
- `Q-10` 的当前阶段结论同步明确：skill 主验证先固定基准模型，以减少变量并优先观察 skill 是否能独立拉开行动过程与结果差异；模型切换保留为次验证开关
- 已同步更新 `MVP/Management/待解决问题清单.md` 与 `MVP/Management/项目状态索引.md`

### 完成行动-聚落层级映射（Phase 1 步骤 1.3）

- 系统梳理骑砍全部行动类型（移动/战斗/经济/军事/社交/任务/政治/信息共 30+ 种行动）
- 标注每个行动需要的最低聚落层级
- **关键发现**：
  - 城镇独占 8 项行动，是行动最丰富的聚落类型
  - 城堡没有任何完全独占的行动，所有城堡行动城镇内部都能做

- **结论**：建议世界结构从三类聚落（城镇/城堡/村庄）简化为两类（城镇 + 村庄/哨卡），城堡功能合并入城镇
