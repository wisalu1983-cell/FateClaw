## 一、旧会话纳管说明

- 本表建立前已经存在的 session，不会被自动接管，也不会自动消失。
- 如果旧 session 对应的任务仍在推进，必须先人工补登记到本表，再继续执行。
- 如果旧 session 的状态已经不清楚，应先标为 `待人工核对` 并暂停领取可能冲突的任务，直到人工确认其实际状态。
- 如果旧 session 实际已结束，只需把其有效结论回写到 `当前执行计划.md`、`开发过程日志.md` 等长期文档，不必补登记为活跃项。

## 三、状态说明

- `进行中`：session 正在执行已领取任务。
- `阻塞中`：session 因复杂问题暂停，待子计划或人工决策。
- `等待确认`：session 已形成阶段结论，等待用户或主控 session 确认。
- `待人工核对`：历史遗留 session 的真实状态尚未核清，不得默认视为可并行或已关闭。

- `设计文档 / 管理文档 / 计划文档` 默认在主开发目录 `D:/01-工作/Garena/GI/FateClaw` 执行。

- 开始新 session 前，除读取本表外，还应先判断：本次任务属于文档类还是实现类；若需进入 `worktree`，先明确原因、工作目录与分支。
- 进入 `worktree` 的 session 在收尾前必须显式选择：继续留在 `worktree`、回灌主开发目录，或通过 git 集成；在这一步未明确前，不视为真正收口。
