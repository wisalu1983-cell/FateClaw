# TinyMB 子计划目录

> **文档用途**：存放复杂阻塞、分叉调查或独立交接问题的子计划。

## 命名规则

- 文件名格式：`YYYY-MM-DD-<topic>.md`
- 一个子计划只解决一个明确问题
- 子计划必须挂靠 `当前执行计划.md` 中的父任务

## 最小头信息

每个子计划至少包含：

- `父计划`
- `父任务`
- `关联 Q-xx`
- `Session ID`
- `状态`
- `目标`
- `关闭条件`

## 回写要求

- 子计划完成后，先回写 `当前执行计划.md`
- 如问题升级为项目级待决项，再更新 `待解决问题清单.md`
- 如形成稳定口径，再更新 `开发过程问答.md`
- 如当前有效状态变化，再更新 `项目状态索引.md`
- 有实质推进时，更新 `开发过程日志.md`
