## 一、目标与约束

### 1.1 本文档的目标

从骑马与砍杀系列中提取与 AIvilization 架构高度兼容的核心机制，极简化后形成一套可被 Agent 框架驱动的世界规则。该规则集将直接用于 AIvilization Plus MVP 的设计。

### 1.2 继承的设计约束

| 约束 | 来源 | 对极简化的影响 |
|---|---|---|
| Agent 是一个"人" | AIvilization 架构 | 角色必须以个人为单位，有日常行为和内心活动 |
| 世界连续运行 | AIvilization 架构 | 不能是回合制，Agent 随时可行动 |
| 玩家引导而非控制 | AIvilization 架构 | 玩家不直接操作，而是通过对话和指令引导 Agent |
| 多 Agent 自主交互 | AIvilization 架构 | 其他 NPC 领主/角色也需要自主行动 |
| 放置/挂机体验 | AIvilization 架构 | 世界不依赖玩家实时在线 |
| 空间/地图 | AIvilization 架构 | 需要有地理空间维度 |
| Agent 内心可观察 | AIvilization 架构 | 日记、思考链、社交网络需要足够丰富的事件支撑 |
| 依赖已成熟 Agent 能力 | Agent 技术约束 | 不依赖战略欺骗、心智理论等未成熟能力 |
| 复杂度红线 | MVP v1 教训 | 先回答"最小系统是什么"，再决定保留哪些细节 |

### 1.3 核心体验假设

我相信，喜欢 AI 沙盒与轻策略体验的玩家，在通过对话引导一个有自主意志的 Agent 于持续运转的城邦中闯荡时，会因为简化版《骑马与砍杀》规则带来的资源、关系与派系取舍，以及玩家意图与 Agent 办事能力（基模、agent skill、角色数值/传统能力、权限）共同塑造行动过程与结果，而明确感知不同基模与 Agent Skill 组合会在同等任务条件下产生不同的执行规划、路径、效率与结果，并愿意据此继续调整指令与迭代 Agent Skill。

### 1.4 极简化的判断标准

| 判断 | 标准 |
|---|---|
| **保留** | 骑砍体验的灵魂机制 + Agent 天然擅长处理 |
| **简化** | 需要存在但可大幅降维的机制 |
| **移除** | MVP 不需要、或 Agent 不擅长处理的机制 |

## 二、设计全貌

### 2.1 一页纸摘要

```
世界：石冠城（Stonecrown）—— 1 座自治城邦，母稿 6 区域；首轮原型启用 4 区域
势力：3 大派系（守备军团 / 商盟 / 平民公社）三角博弈
角色：母稿 12 名命名 NPC；首轮原型启用 6 名命名 NPC + 1 名玩家 Agent
核心变量：Money / Renown / Honor / Relationships（四变量平衡）
时间：连续时钟（10 分钟基本粒度），区域间移动 5~20 分钟，每 3 天结算
交互：玩家通过对话引导 Agent；行动方案受基模、agent skill、角色状态与权限共同影响；观察输入采用“公共世界摘要 + 个人记忆召回”的最小观察包；行动输出采用“一个权威主动作 + 后续意图 / 中断点 / 回退”的混合契约；行动记录采用统一 `decision_record` 并派生 `log_card / ask_card / review_card`；首轮采用“默认自主 + 重要请示 + 安全回退”
架构：单层 Agent + 多层约束（见下方 §2.1.1）
```

#### 2.1.1 架构口径

- **单层 Agent**：每个 Agent（玩家或 NPC）运行一条 `Observe → Think → Decide → Act` 循环，输出结构化行动意图
- **多层约束**：官方系统 prompt 锁定行为框架 → Server/Runtime 校验行动合法性 → Server/Runtime 结算产生权威结果
- **部署拆分**：NPC Agent 在服务端运行（服务端统一 LLM）；玩家 Agent 在用户侧运行（用户选择基模，官方系统 prompt 不可修改）
- **接口统一**：两类 Agent 对 Runtime 的接口完全相同——接收观察包，输出结构化行动意图，由 Runtime 统一校验和结算
- **观察契约**：首轮观察包采用 `self_state / local_snapshot / world_state_summary / memory_recall / available_actions` 五块；`world_state_summary` 只放公共客观信息，`memory_recall` 承接当前问题触发的个人记忆召回
- **行动契约**：首轮原型采用“混合方案”——`Runtime` 只权威执行一个主动作，后续意图、中断点、能力依据 / 缺口与回退主要用于请示、日志、回放与差异验证
- **记录契约**：`P4` 采用“统一 `decision_record` + 展示层 `log_card / ask_card / review_card`”的双层结构；玩家可读层默认用第一人称呈现，玩家 Agent 与命名 NPC 共用核心 schema

详见 `Brainstorming/多Agent框架相关/多Agent框架讨论总结.md` §2.2.1 及 §4.2

### 2.2 世界结构示意

```
        首轮原型启用区：
        城主堡 / 市集区 / 酒馆区 / 外城农庄

        Later 母稿保留：
        练兵场 / 城墙与城门
```

### 2.3 玩家进阶路径

```
外来者 → 派系帮手 → 派系核心 → 城主
（骑砍对应：自由冒险者 → 佣兵 → 封臣 → 国王）
```

### 2.4 核心数值结构

```
角色层：Money / Renown / Honor / 派系关系×3 / NPC关系×N / 随从（数量/战力/士气）/ 粮食 / 影响力
世界层：派系势力值×3 / 繁荣度 / 外部压力标签 / 物价指数 / 城主 / 天数

核心张力：
  Money ←→ 军事（养兵烧钱）
  Renown ←→ Honor（有时需要不光彩手段获取声望）
  派系A关系 ←→ 派系B关系（帮一方得罪另一方）
  短期收益 ←→ 长期关系（贿赂赚快钱但掉荣誉）
```

## 四、裁剪决策汇总

| 骑砍原版机制 | 极简化处理 | 来源模块 |
|---|---|---|
| 多城镇大陆地图 | → 单城镇；母稿 6 区，首轮启用 4 区 | 世界结构 |
| 三类聚落（城镇/城堡/村庄） | → 合并为城镇内区域 | 世界结构 |
| 多势力（6~8 王国） | → 3 城内派系 | 世界结构 |
| 开放大地图自由移动 | → 区域间移动（5~20 分钟/次） | 世界结构 |
| 城际贸易套利 | → **移除** | 世界结构 |
| 野外遭遇 | → 移动随机事件 + 轻量外部压力 | 世界结构 |
| 商队系统 | → **移除** | 世界结构 |
| 多种地形/季节/天气 | → **移除** | 世界结构 |
| 连续自由行动 | → 连续时钟 + 同区连续办事 + 跨区移动耗时 | 核心循环 |
| 20+ 种传统数值技能 | → **移除**（Agent skill 另作任务能力包设计） | 核心循环 |
| 实时战斗操控 | → 自动结算 | 核心循环 |
| 精细装备系统 | → **移除**（战力=随从数×等级） | 数值骨架 |
| 多种食物类型 | → 单一"粮食"资源 | 数值骨架 |
| 声望/Right to Rule 分离 | → 合并为单一声望 | 数值骨架 |
| 领地税收（多领地） | → 区域管理权收入 | 数值骨架 |
| 家族（Clan）层级 | → **移除** | 势力与NPC |
| 元帅召集 | → 首轮基础站队后果；完整派系议事投票 `Later` | 势力与NPC |
| 婚姻/联姻 | → **移除** | 势力与NPC |
| 商队NPC | → **移除** | 势力与NPC |
| 玩家直接操控 | → 对话引导 + 请示机制 | 玩家引导 |
| 全知视角 | → 有限信息 + 情报系统 | 玩家引导 |

## 五、风险评估

| # | 风险 | 严重度 | 可能性 | 应对策略 |
|---|---|---|---|---|
| R1 | 区域移动无感 | 高 | 中 | 移动途中必须有随机事件 |
| R2 | 派系冲突缺乏"战争感" | 中 | 高 | 设计显性武力冲突机制 |
| R3 | NPC 行为同质化 | 高 | 中 | 性格+目标+状态组合差异化 |
| R4 | 经济崩溃/通胀 | 中 | 高 | 开发阶段模拟迭代 |
| R5 | 请示节奏失调 | 中 | 中 | 首轮最小请示基线 + 积压合并 + 可调频率 |
| R6 | 单城镇事件多样性不足 | 高 | 中 | MVP 只需支撑 2~4 周体验 |
| R7 | LLM 调用成本超预算 | 高 | 中 | 低优先级行为用规则引擎 |

### AIvilization 架构约束覆盖验证

| 约束 | 如何覆盖 |
|---|---|
| Agent 是一个"人" | 性格、日记、内心想法、社交关系（模块 4、5） |
| 世界连续运行 | 连续时钟推进 + 跨区移动耗时 + 每 3 天结算 + 事件驱动（模块 2） |
| 玩家引导而非控制 | 首轮最小请示基线 + 长期方针；完整四级自主等级保留为 Later 母稿（模块 5） |
| 多 Agent 自主交互 | 首轮 6 名命名 NPC 独立行为循环 + 自发互动（模块 4） |
| 放置/挂机体验 | 离线时按方针自主行动 + 日记回顾（模块 5） |
| 空间/地图维度 | 首轮 4 活跃区域 + 移动耗时 + 随机事件（模块 1） |
| Agent 内心可观察 | 日记系统 + NPC 日记分级可见性（模块 5） |
| 依赖已成熟 Agent 能力 | 基于"感知→判断→行动"，不依赖未成熟能力 |
| 复杂度红线 | 首轮 6 名命名 NPC、4 活跃区域、3 派系——在已验证能力范围内 |
