# 极简骑砍规则集

> **文档用途**：将骑马与砍杀（战团 + 霸主）的核心玩法提炼为极简化规则集，作为 AIvilization Plus MVP 设计的输入材料。
>
> **文档状态**：编写中

---

## 一、目标与约束

### 1.1 本文档的目标

从骑马与砍杀系列中提取与 AIvilization 架构高度兼容的核心机制，极简化后形成一套可被 Agent 框架驱动的世界规则。该规则集将直接用于 AIvilization Plus MVP 的设计。

### 1.2 继承的设计约束

以下约束来自 `02-核心玩法对标脑暴.md` 和 `多Agent框架讨论总结.md`，在本文档中作为不可变的输入：

| 约束 | 来源 | 对极简化的影响 |
|---|---|---|
| Agent 是一个"人" | AIvilization 架构 | 角色必须以个人为单位，有日常行为和内心活动 |
| 世界连续运行 | AIvilization 架构 | 不能是回合制，Agent 随时可行动 |
| 玩家引导而非控制 | AIvilization 架构 | 玩家不直接操作，而是通过对话和指令引导 Agent |
| 多 Agent 自主交互 | AIvilization 架构 | 其他 NPC 领主/角色也需要自主行动 |
| 放置/挂机体验 | AIvilization 架构 | 世界不依赖玩家实时在线 |
| 空间/地图 | AIvilization 架构 | 需要有地理空间维度 |
| Agent 内心可观察 | AIvilization 架构 | 日记、思考链、社交网络需要足够丰富的事件支撑 |
| 依赖已成熟 Agent 能力 | Agent 技术约束 | 不依赖战略欺骗、心智理论等未成熟能力 |
| 复杂度红线 | MVP v1 教训 | 先回答"最小系统是什么"，再决定保留哪些细节 |

### 1.3 极简化的判断标准

| 判断 | 标准 |
|---|---|
| **保留** | 骑砍体验的灵魂机制 + Agent 天然擅长处理 |
| **简化** | 需要存在但可大幅降维的机制 |
| **移除** | MVP 不需要、或 Agent 不擅长处理的机制 |

## 二、骑砍核心循环概要

> 本章基于骑马与砍杀：战团（2010）和骑马与砍杀 II：霸主（2022）的核心玩法提炼，聚焦与 AIvilization 架构直接相关的 6 个维度。

### 2.1 大地图与空间

骑砍的世界是一张持续运转的**大陆地图**，包含三类聚落节点：

| 聚落类型 | 功能 | 数量（战团→霸主） |
|---|---|---|
| **城镇（Town）** | 核心经济中心：交易市场、工坊、酒馆（招募同伴）、竞技场、行会 | 22 → 57 |
| **城堡（Castle）** | 军事据点：驻军、防御工事、攻城战场所 | ~48 → 更多 |
| **村庄（Village）** | 资源产出点：粮食/原材料生产，招募基础兵源，依附于城镇或城堡 | 每座城镇/城堡关联 2+ 个村庄 |

> 霸主的聚落规模约为战团的 2.5 倍。极简化设计不需要还原完整规模，只需保留三类聚落的功能分化逻辑。

聚落之间是**开放的野外区域**：
- 角色在大地图上以**队伍**为单位移动（不是单人）
- 移动速度受队伍规模、地形、马匹、技能影响
- 野外存在**强盗窝点**、流动的**商队**、巡逻的**领主队伍**
- 在大地图上遭遇其他队伍可触发战斗或对话

**与 AIvilization 的映射**：骑砍的大地图 ≈ AIvilization 的"世界空间"，聚落 ≈ 地点节点，野外移动 ≈ Agent 在地点间的位移。

### 2.2 角色日常行为循环

骑砍中不存在显式的"一天日程表"，角色的行为是**目标驱动的连续行动流**：

```
评估当前状态（钱、兵、声望、关系、队伍状况）
    ↓
选择下一个目标（去哪里？做什么？）
    ↓
在大地图上移动到目标地点
    ↓
到达后执行行动：
  - 在城镇：交易、招兵、接任务、参加比武、拜访领主、宴会社交
  - 在村庄：招募农兵、购买粮食/原材料
  - 在野外：战斗、追击、护送、巡逻
    ↓
获得结果（金钱、声望、关系变化、战利品、情报）
    ↓
回到评估 → 循环
```

**关键特征**：
- 没有固定日程，行为完全由目标和当前状态驱动
- 同一时间世界中所有领主都在独立执行自己的行为循环
- 食物会消耗、士气会波动、伤兵需要恢复——这些"维护需求"自然产生日常行为
- 任务系统是行为多样性的重要来源：村长、行会长、领主各自提供不同类型的委托（护送商队、剿灭强盗、运送物资、收税等），完成任务是获取关系值和金钱的主要途径

**骑砍策略层的核心框架**：Wiki 将骑砍的策略核心概括为**四个基础变量的平衡管理**——

| 变量 | 获取方式 | 消耗/风险 |
|---|---|---|
| **Money（金钱）** | 交易、任务、战利品、领地税 | 军队维持费、装备、招募、贿赂 |
| **Renown（声望）** | 战斗胜利（以少胜多奖励更高）、任务 | 随时间自然衰减，高声望更难获取 |
| **Honor（荣誉）** | 释放被俘领主、拒绝不义任务 | 与军事效率直接冲突（释放敌人=放虎归山） |
| **Relationships（关系）** | 完成委托、战斗协同、政治支持 | 与其他关系冲突（分封让一人满意则其他人不满） |

四者相互牵制：释放敌方领主 → 荣誉 ↑ 但军事优势 ↓；劫掠村庄 → 金钱 ↑ 但关系 ↓ 且荣誉 ↓。这个"四变量平衡"框架是骑砍策略层的灵魂，极简化时数值骨架至少需要覆盖这四个维度。

### 2.3 势力与 NPC 自主行为

骑砍世界中有多个势力并存（战团 6 个，霸主 8 个——帝国分裂为北/南/西三部分 + 5 个独立王国），每个势力包含：

| 层级 | 说明 |
|---|---|
| **国王** | 势力领袖，决定宣战/议和、分封领地 |
| **家族（Clan）** | 势力内的基本组织单位，拥有领地和军队 |
| **领主（Lord）** | 家族成员中的活跃角色，各自带领队伍行动 |
| **聚落** | 城镇/城堡/村庄，归属于具体家族 |

**NPC 领主的自主行为模式**：
- **补给**：前往聚落补充士兵和物资
- **巡逻**：在自家领地附近巡逻，驱赶强盗
- **防御**：当自家聚落受到威胁时赶去防守
- **进攻**：对敌方聚落发动劫掠或攻城
- **社交**：参加宴会、拜访其他领主、进行外交活动
- **经济**：贩卖战俘、经营产业

**领主性格影响行为倾向**（战团有 8 种性格类型，可归为以下四大倾向）：
- 正直/善良型（Upstanding/Good-natured） → 忠诚守约，响应召集，偏好巡逻防御和主动救援
- 好战型（Martial） → 偏好进攻和挑战敌军
- 狡猾型（Cunning） → 行为更灵活，追求自身利益最大化
- 卑劣型（Sadistic/Debauched/Pitiless/Bad-tempered） → 偏好劫掠，容易叛变，不守承诺

**势力兴衰机制**：
- 宣战/议和通过势力内投票决定（消耗影响力）
- 领主可以被说服叛变到其他势力（受关系、实力对比、性格影响）
- 攻城成功后领地归属变更，势力版图随之变化
- 势力过弱时领主可能纷纷叛逃，导致势力灭亡

### 2.4 社交与政治系统

骑砍的社交系统以**关系值**为核心：

**关系影响因素**：
- 完成领主委托的任务 → 关系上升
- 在战斗中帮助/损害领主 → 关系变化
- 领地分封是否公平 → 获封者 +1，其他人 -3
- 释放/俘虏领主 → 大幅影响关系和荣誉值
- 宴会邀请和赴约 → 维护社交关系

**政治层面**：
- 封臣向国王贡献影响力，用影响力提议政策/宣战/议和
- 领主之间有个人关系，会形成非正式的派系
- 联姻是建立家族联盟的重要手段
- 说服系统基于技能（魅力/口才）+ 双方性格匹配度

**宴会（Feast）系统**：骑砍中最重要的社交场景之一。领主可以在自己的城镇/城堡举办宴会，邀请其他领主参加。宴会期间：
- 大量领主聚集在同一地点，是建立和维护关系的核心机会
- 政治站队、求婚、联盟谈判等高价值社交行为集中发生
- 宴会持续数天，参与者暂时离开各自的巡逻/作战，影响军事态势
- 是骑砍中"多 Agent 在同一时空社交互动"最集中的体现

**与 AIvilization 的映射**：每次社交互动都有实质后果（关系变化 → 影响政治站队 → 影响战争走向）——正是"C3 交互有后果"的体现。宴会系统是产生大量可观察 Agent 社交事件的天然场景。

### 2.5 经济与资源

骑砍的经济系统围绕**金钱（第纳尔）** 运转：

**收入来源**：
- 交易：低买高卖（商品价格受供需影响）
- 任务报酬：领主/商人/村长的委托
- 战利品：战斗后缴获装备和俘虏
- 领地税收：拥有城镇/城堡后的稳定收入
- 产业收入：在城镇开设工坊（铁匠铺/酿酒厂等）

**支出去向**：
- 军队维持费：每个士兵每周消耗金钱
- 食物消耗：队伍需要持续补给食物
- 驻军费用：城堡/城镇的驻军
- 装备升级、招募费用

**核心经济压力**：养兵很贵——队伍越大、兵种越高级，维持费越高。玩家必须持续产生收入来维持军队，否则士气下降、士兵逃散。

**战争对经济的影响**：战争会破坏贸易路线、劫掠村庄会降低产出、攻城会造成聚落繁荣度下降。

**聚落繁荣度（Prosperity）**：每个聚落有一个繁荣度变量，影响税收、物价和招兵质量。繁荣度受贸易畅通度、巡逻安全性、战争破坏等因素影响——村庄被劫掠后繁荣度暴跌至"极穷"并需要长时间恢复，城镇被攻陷后繁荣度 -20。这是经济系统的"健康度指标"。极简化时需要决定是否保留此概念。

### 2.6 成长与进阶路径

骑砍提供了一条清晰的**身份进阶路径**：

```
自由冒险者（开局）
    ↓  积累声望和金钱
佣兵（Clan Level 1）
    ↓  获取战斗报酬，不承担政治义务
封臣（Clan Level 2）
    ↓  获得领地，参与势力政治，承担军事义务
建国者（Clan Level 4 + 占领领地）
    ↓  创建自己的王国，招纳封臣
```

**每个阶段的行为空间不同**：

| 阶段 | 可做的事 | 不能做的事 |
|---|---|---|
| 自由冒险者 | 交易、接任务、打强盗、参加比武 | 无法拥有领地、无法参与政治 |
| 佣兵 | 以上 + 为势力作战获取报酬 | 无领地，无政治投票权 |
| 封臣 | 以上 + 拥有领地、参与分封投票、提议政策 | 受国王约束，宣战/议和需投票 |
| 国王 | 所有行为 + 分封领地、管理政策、招纳封臣 | 需要维持封臣忠诚度 |

**同伴系统**：
- 在酒馆招募流浪者作为同伴（上限 4~8 人）
- 同伴可分配角色：侦察兵、军医、工程师、管家
- 同伴可被派出组建独立队伍或管理领地
- 同伴有自己的技能和性格，影响其担任角色的效果

### 2.7 概要评审结论

## 三、极简化世界结构与地图

### 3.1 关键设计决策

需要回答的核心问题：
1. 地图上有多少个地点节点？
2. 有几个势力？
3. 是否保留三类聚落分化（城镇/城堡/村庄），还是合并？
4. 地点之间的空间关系如何表示？
5. 是否保留"野外/途中"概念？

### 3.2 三种方案对比

#### 方案 A：极简图（5~8 节点，2 势力）

```
势力甲领地        争夺区          势力乙领地
[城镇A]---[城堡X]---[自由镇C]---[城堡Y]---[城镇B]
   |                                         |
 [村庄a]                                   [村庄b]
```

- **节点**：2 城镇 + 2 城堡 + 1 自由城镇 + 2 村庄 = 7 节点
- **势力**：2 个对立势力 + 1 个中立区
- **聚落分化**：保留城镇/城堡/村庄三类，但每类极少
- **空间关系**：简单图，节点之间有连接线和旅行时间
- **野外**：不单独建模，移动 = "花 N 小时从 A 到 B"，途中有概率遭遇事件

| 优势 | 劣势 |
|---|---|
| 实现最简单，Agent 数量最少 | 2 势力 = 外交退化为二元敌友，无法产生三角博弈 |
| 地图状态空间最小，易于调试 | 地点太少，Agent 行为路线单一，缺乏选择感 |
| | 无法体现骑砍"大陆闯荡"的空间感 |

#### 方案 B：小大陆（12~15 节点，3 势力）—— **推荐**

```
      [村庄1]          [村庄2]
         |                |
势力甲 [城镇A]---[关隘P]---[城镇B] 势力乙
         |                |
      [城堡α]    [自由镇F]   [城堡β]
         |        /    \      |
      [村庄3] [村庄4]  [村庄5] [村庄6]
                  |
              [城镇C]---[城堡γ]
                  |
              [村庄7]
              势力丙
```

- **节点**：3 城镇 + 3 城堡 + 1 自由城镇 + 6~7 村庄 = 13~14 节点
- **势力**：3 个势力各占一个区域 + 1 个自由城镇作为缓冲/贸易枢纽
- **聚落分化**：保留三类
  - 城镇：经济中心，有市场/酒馆/工坊，是社交和贸易的主要场所
  - 城堡：军事据点，控制区域通道，攻城战的焦点
  - 村庄：资源产出，依附于最近的城镇或城堡，招兵来源
- **空间关系**：图结构，边带旅行时间（0.5~2 天），关隘/山口等节点形成战略要冲
- **野外**：不单独建模，但移动途中有"遭遇概率"（遇到强盗/商队/敌方巡逻队）

| 优势 | 劣势 |
|---|---|
| 3 势力支撑三角外交（甲乙开战→丙可选择站队或渔利） | 比方案 A 复杂，Agent 数量更多 |
| 地点足够多，Agent 有真正的路线选择和策略空间 | 村庄数量可能仍偏多，可按需裁减 |
| 自由城镇提供中立社交场所和贸易枢纽 | |
| 关隘/要冲节点创造战略纵深 | |
| 规模可控，3 势力 × 每势力 2~3 活跃领主 ≈ 6~9 NPC Agent | |

#### 方案 C：微缩大陆（20~25 节点，4+ 势力）

- **节点**：5~6 城镇 + 5~6 城堡 + 10~12 村庄 = 20~24 节点
- **势力**：4~5 个势力
- **聚落分化**：完整保留三类，每个城镇有 2~3 个附属村庄
- **空间关系**：更复杂的图结构，有海路/山路等多种地形
- **野外**：显式建模为"路段"节点

| 优势 | 劣势 |
|---|---|
| 最接近骑砍原版的世界感 | 节点多→Agent 多→LLM 调用成本高→MVP 风险大 |
| 4+ 势力产生更复杂的多边外交 | 复杂度可能重蹈 MVP v1 覆辙 |
| | 调试和平衡难度大幅增加 |

### 3.3 推荐方案 B 的理由

**方案 B（小大陆，12~15 节点，3 势力）** 是推荐选择，核心理由：

1. **3 势力是三角博弈的最低门槛**：2 势力只有"敌/友"，3 势力才能产生"两强相争第三方渔利""结盟对抗强者""叛变投靠"等骑砍标志性的外交动态
2. **节点数量在 Agent 成本的安全区内**：3 城镇 × 2~3 领主/城镇 ≈ 6~9 NPC Agent，加上玩家的 Agent = 总计 7~10 Agent，在多 Agent 编排的已验证能力范围内（5~7 Agent 有效，10 个左右是上限）
3. **保留三类聚落分化但数量极少**：维持了骑砍"城镇做买卖、城堡打仗、村庄招兵"的角色分工感，但每类只有 3 个，不至于状态爆炸
4. **自由城镇是关键设计**：提供中立社交场所（宴会/集会/谈判的天然地点）、贸易枢纽（商品流通中心）、避难所（战败后的缓冲地带）——单独的中立区大幅增加了世界的动态性
5. **关隘/要冲节点**：用 1~2 个战略节点创造"控制通道 = 军事优势"的简单空间策略，不需要复杂地形系统

### 3.4 方案 B 的具体参数建议

| 参数 | 建议值 | 理由 |
|---|---|---|
| 城镇数 | 3（每势力 1 个） | 每个势力的经济/社交中心 |
| 城堡数 | 3（每势力 1 个） | 每个势力的军事据点 |
| 自由城镇 | 1 | 中立区、贸易枢纽、社交场所 |
| 村庄数 | 5~6（每城镇/城堡各 1 个附属） | 保留最小的资源产出层，可根据复杂度压力进一步裁减 |
| 势力数 | 3 | 三角博弈最低门槛 |
| 活跃 NPC 领主数 | 6~9（每势力 2~3 个） | 在 Agent 编排安全区内 |
| 节点间旅行时间 | 0.5~2 天 | 创造移动成本，但不至于长到无聊 |
| 途中遭遇概率 | 有/无（简单二值） | MVP 阶段不做复杂概率计算 |

### 3.5 裁剪决策记录（世界结构）

| 骑砍原版机制 | 极简化处理 | 判断 | 理由 |
|---|---|---|---|
| 22~57 城镇 | 缩减为 3+1（自由镇） | 简化 | 保留角色分化，极度缩减数量 |
| ~48 城堡 | 缩减为 3 | 简化 | 每势力 1 个军事据点足够 |
| 100+ 村庄 | 缩减为 5~6 | 简化 | 保留资源产出概念，最小数量 |
| 开放大地图 + 自由移动 | 图结构 + 节点间旅行时间 | 简化 | Agent 不需要空间坐标，只需要"在哪个节点"和"在移动中" |
| 多种地形（山地/平原/沙漠/雪地） | 仅保留"关隘"作为战略要冲概念 | 简化 | 地形细节对 Agent 决策价值低 |
| NPC 商队在城镇间流动 | MVP 移除 | 移除 | 可用简单的"贸易价格波动"替代商队系统 |
| 强盗窝点 / 多种强盗类型 | 合并为"途中遭遇" | 简化 | 强盗作为世界风险的简单表示 |
| 季节/天气影响 | MVP 移除 | 移除 | 对 Agent 决策价值低，增加不必要复杂度 |
