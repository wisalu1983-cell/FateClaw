# 势力与 NPC 自主行为

> **所属**：极简骑砍规则集 · 模块 4
>
> **设计目标**：定义三大派系的内部结构、NPC 角色名册、NPC 自主行为模式、以及派系间的动态博弈规则。首轮启用的命名势力 NPC 与关键中立 NPC 不是脚本驱动的背景板，而是与玩家使用同一套行为循环的独立 Agent；轻量同伴候选保留为传统 NPC + 专长标签方案。

---

## 0. MVP 回标说明

本文档同时保留两层口径：

- **母稿**：完整 12 名命名 NPC、完整派系内部政治与外部威胁协作设计。
- **首轮可验证原型回标稿**：只启用 6 名命名 NPC，并将派系政治压缩到最小可验证层。

### 首轮可验证原型（MVP-Must）

- 命名 NPC 总数锁定为 **6 名**
- 必须覆盖：**三派系锚点 + 中立信息节点 + 最小灰度 / 情报路线**
- 命名势力 NPC 与关键中立 NPC 使用与玩家同构的行为循环

### 首轮原型后放（Later）

- 其余 6 名命名 NPC
- 完整派系投票、弹劾、叛变与联盟流程
- 完整外部威胁合作与围城剧本

## 3. NPC 自主行为模式

首轮启用的命名势力 NPC 与关键中立 NPC 与玩家 Agent 使用**完全相同的行为循环**（见 `02-核心循环与行为空间.md`），区别仅在于：NPC 由 LLM 基于性格 + 目标 + 当前状态自主决策，有派系义务，且 NPC 之间会自发互动。`Later` 的轻量同伴候选采用传统 NPC + 专长标签方案，不要求独立运行完整 Agent 循环。

### 日常行为模板

| 行为类别 | 典型行为 | 触发条件 |
|---|---|---|
| **补给** | 去市集采购物资、去农庄补充粮食 | 粮食不足或随从需要补充 |
| **巡逻/执勤** | 在城主堡或农庄执行警戒、在本方区域维持秩序 | 轻量外部压力升高 或 日常轮值 |
| **社交** | 去酒馆交流、拜访其他NPC、举办宴会 | 关系需要维护 或 有政治议题 |
| **经济** | 交易、收取费用、处理商路异动 | 需要资金（商盟 NPC 常态） |
| **军事** | 招募兵员、组织护卫、处理小规模冲突 | 军事需求或好战性格驱动 |
| **政治** | 参加议事、拉拢其他NPC、策划行动 | 派系势力值变化 或 重大事件 |
| **情报** | 收集消息、监视对手、打听动向 | 狡猾型NPC常态行为 |

首轮原型保留：**补给 / 巡逻 / 社交 / 经济 / 情报 / 基础政治**

`Later`：完整训练、比武、正式议事投票和叛变流程

### NPC 之间的自发互动

| 互动类型 | 触发条件 | 可能结果 |
|---|---|---|
| 对话/交流 | 两个 NPC 在同一区域 | 关系变化、情报交换、达成协议 |
| 争执/冲突 | 利益冲突 + 关系 < 0 | 关系恶化、可能升级为武力对抗 |
| 合谋/密谈 | 关系 > 30 + 共同利益 | 形成临时同盟、策划联合行动 |
| 试探/对峙 | 好战型或敌对 NPC 在同一区域相遇 | 声望变化、关系变化、可能升级为冲突 |
| 交易/交换 | 双方有互补需求 | 资源转移、互惠协议 |

## 4. 派系内部机制（Later 母稿为主）

首轮可验证原型只保留**基础站队后果**：帮某派系做事会改变另外两方的态度，关键任务会逼迫玩家和 NPC 表态。以下正式提案、弹劾与叛变流程保留为 `Later` 母稿。

### 派系决策流程

1. 任一派系核心成员发起提案，消耗影响力
2. 该派系所有核心成员投票（支持/反对/弃权）
3. 多数赞成 → 通过；否则否决
4. NPC 投票倾向由性格+个人利益+与提案人的关系综合决定

### 派系内权力竞争

- 领袖不永久固定——决策持续让成员不满时可能面临弹劾
- 挑战方式：议事中发起"弹劾提案"或"领袖挑战"
- 玩家可以成为挑战者或被挑战者

### 叛变机制

触发条件（需同时满足多项）：
- 与当前派系领袖关系 < -30
- 与目标派系中某NPC关系 > 30
- 性格为狡猾型或卑劣型
- 目标派系势力值 > 当前派系势力值

后果：叛变者带走部分随从+情报 → 原派系势力值下降 → 新派系势力值上升。

## 5. 派系间博弈规则（MVP-Must 核心 + Later 母稿）

首轮可验证原型只要求：**三派系持续互相牵制、关键任务会造成站队后果、轻量外部压力会暂时改变各方优先级**。正式区域控制权争夺、联盟和完整外患协作保留为 `Later` 母稿。

### 区域争夺

完整母稿中，6 个区域里城主堡、市集区、外城农庄各有默认控制派系，酒馆区和练兵场为争夺区，城墙与城门三方共管。首轮原型只围绕四区产生关系压力，不做正式区域控制权变化。

争夺流程：议事决定"争夺 X 区域" → 派代表交涉 → 交涉失败可能升级为武力对抗 → 胜方获得控制权。

### 合作与联盟

- 两派系可达成临时联盟（持续 N 周），不争夺对方区域，外部威胁时优先协同
- 联盟可被单方面撕毁（代价：Honor↓，关系↓↓）

### 外部威胁的合作机制

外部威胁"危急"时：三方强制停战 → 贡献随从参与城防 → 胜利则声望↑↑+繁荣度↑；失败则所有派系势力值-10+繁荣度-20。

## 6. 事件驱动系统（MVP-Must 核心 + Later 母稿）

| 事件类别 | 示例 | 频率 | 影响 | 首轮回标 |
|---|---|---|---|---|
| **轻量外部压力** | 匪患传闻、农庄求援、警戒升级、外来信使 | 1~3 天一次轻量信号 | 给世界施压，驱动护民、巡逻、站队和情报行为 | **MVP-Must** |
| **经济波动** | 商路异动、物价波动、囤货消息 | 每 3 天结算或事件驱动 | 经济压力，派系利益重分配 | **MVP-Must** |
| **政治事件** | 关键任务邀约、派系施压、站队传闻 | 事件驱动 | 让玩家与 NPC 感受到立场选择的后果 | **MVP-Must** |
| **社交事件** | 密谈邀请、酒馆聚会、小型庆典 | 每 3~7 天 | 提供关系变化与信息交换机会 | **MVP-Must** |
| **个人事件** | NPC 求助、同伴消息、旧友来访 | 随机 | 丰富叙事，创造选择 | **MVP-Must** |
| **围城 / 大型政治改组 / 大型比武** | 围城、城主选举、全面叛变 | 低频高影响 | 重塑格局，放大势力冲突 | `Later` |

## 7. 裁剪决策记录

| 骑砍原版机制 | 极简化处理 | 判断 | 理由 |
|---|---|---|---|
| 6~8 个王国，多个家族 | 3 个城内派系；首轮 6 名命名 NPC，母稿 12 名 | **简化** | 先保留三派系锚点与中立信息节点，再向更完整生态扩展 |
| 领主独立队伍在大地图移动 | NPC 在城内区域间移动 | **变形** | 行为循环不变 |
| 家族（Clan）层级 | 移除 | **移除** | 单城镇内过度复杂 |
| 元帅召集 | 首轮保留基础站队后果，正式议事投票后放 | **变形** | 先验证阵营压力与立场后果，再扩展完整政治流程 |
| 婚姻/联姻 | MVP 移除 | **移除** | 不影响核心三角博弈验证 |
| 领主被俘/逃跑循环 | 简化版（关押+释放/赎回） | **简化** | 保留核心抉择 |
| 商队系统 | 移除 | **移除** | 外来商人以事件形式到访 |
| 完整外敌入侵与协同守城 | 首轮改为轻量外部压力与小事件 | **回标收边** | 先验证世界动态和短期协作压力，不直接实现完整围城系统 |
