# 玩家引导与请示节奏

> **所属**：极简骑砍规则集 · 模块 5
>
> **设计目标**：定义玩家如何与自己的 Agent 互动——不是直接操控，而是通过对话引导。同时定义 Agent 在什么场景下自主行动、什么场景下请示玩家、以及玩家离线时的行为策略。
>
> **核心约束**：玩家引导而非控制（AIvilization 架构约束）。Agent 是一个有自主判断力的"人"，玩家更像是 Agent 的"军师/顾问"而非"遥控器"。

---

## 1. 玩家与 Agent 的交互模式

| 模式 | 说明 | 何时发生 |
|---|---|---|
| **对话指令** | 玩家通过自然语言告诉 Agent 想做什么 | 玩家在线时随时可发起 |
| **请示响应** | Agent 遇到重要决策，暂停并询问玩家意见 | Agent 触发请示条件时 |
| **长期方针** | 玩家设定优先级和行为偏好，Agent 据此自主行动 | 玩家在线时设定，离线时生效 |
| **回顾观察** | 玩家查看 Agent 的日记、行动记录、世界变化 | 玩家上线时或随时查看 |

---

## 2. Agent 自主等级

### Level 0：完全自主（不通知）

Agent 自行处理，玩家事后通过日记得知。

| 行动 | 理由 |
|---|---|
| 日常移动（去市集买粮、回住所休息） | 生存性日常，无需决策 |
| 与普通NPC的日常对话 | 常规社交，影响极小 |
| 轻微的关系变化（+1/-1） | 波动太小，通知会造成信息过载 |

### Level 1：自主执行，事后通知

Agent 执行后在日记中记录，玩家下次查看时可以看到。

| 行动 | 理由 |
|---|---|
| 完成日常任务（巡逻、运货等） | 例行公事，结果有意义但决策简单 |
| 参加比武（非重大赛事） | 有风险但可接受 |
| 随从日常训练 | 长期积累型行为 |
| 应对小规模匪帮遭遇 | 被动触发、需要立即反应 |

### Level 2：请示推荐（Agent 给出建议，等待确认或修改）

Agent 暂停行动，展示当前情况和建议方案。玩家未响应时，Agent 按建议方案执行。

| 行动 | 理由 |
|---|---|
| 接受/拒绝重要任务 | 可能涉及派系站队或较大风险 |
| 参加派系议事并投票 | 政治选择影响深远 |
| 接受 NPC 的宴会/密谈邀请 | 可能改变关系网 |
| 投资工坊/大额交易 | 重要经济决策 |
| 招募新同伴 | 长期影响队伍构成 |

### Level 3：请示必须（Agent 暂停，必须等待玩家决策）

不可逆行动，Agent 不会在无玩家指示的情况下执行。

| 行动 | 理由 |
|---|---|
| 加入/退出派系 | 改变角色核心身份 |
| 背叛当前派系 | 不可逆的关系破坏 |
| 与命名 NPC 开战 | 可能导致重要角色死亡/被俘 |
| 竞选派系领袖/城主 | 角色发展的关键节点 |
| 释放/处决重要囚犯 | 荣誉和关系的重大转折 |

## 3. 请示节奏设计

| 场景 | 请示频率 | 说明 |
|---|---|---|
| 玩家在线、积极互动 | 每 1~2 个行动机会 1 次请示 | 玩家感受到参与感 |
| 玩家在线、挂机观察 | 每 3~5 个行动机会 1 次请示 | 只在重要节点打扰 |
| 玩家离线 | 0 次请示 | Agent 按长期方针自主行动 |

### 请示积压机制

多个请示事件同时触发时，合并为一个"近况汇报+待决清单"：

```
【Agent 近况汇报】
今天上午我去了市集区，完成了商盟的护送任务（+80 金币，商盟关系+5）。
下午在酒馆遇到了灰鸦·莫恩，他暗示守备军团在策划对农庄的管制。

【待决事项】
1. 铁壁·卡尔邀请我明天参加守备军团议事，讨论城防预算分配。要去吗？
2. 老橡·赫尔曼托我帮忙训练一批民兵（任务报酬 100 金币，但守备军团会不满）。接吗？
```

## 4. 长期方针系统

| 方针维度 | 选项 | 影响 |
|---|---|---|
| **经济优先级** | 攒钱 / 投资 / 不关心 | Agent 是否主动寻找赚钱机会 |
| **派系倾向** | 偏向某派系 / 保持中立 / 见机行事 | 影响任务接受和社交选择 |
| **军事态度** | 积极扩军 / 维持现状 / 精兵路线 | 影响招募和训练行为 |
| **社交风格** | 广泛交友 / 重点深耕 / 低调行事 | 影响社交行为频率和对象选择 |
| **风险偏好** | 保守 / 平衡 / 激进 | 影响是否接受高风险高回报的机会 |

玩家离线时，Agent 按方针自主行动，所有行动降级为 Level 0 或 Level 1。

## 5. Agent 日记与可观察性

### 日记内容

每个行动机会结束后，Agent 生成一条日记条目：

| 组成部分 | 内容 | 示例 |
|---|---|---|
| **行动记录** | 做了什么、在哪里、和谁 | "今天上午去了练兵场，参加了比武大赛" |
| **内心想法** | 为什么做出这个选择 | "最近手头紧，比武奖金是最快的来钱方式" |
| **观察与感受** | 对事件和NPC的主观看法 | "烈焰·塞娜的剑法越来越凌厉了，下次碰到她得小心" |
| **对未来的打算** | 接下来想做什么 | "明天该去找老橡·赫尔曼谈谈" |

### NPC 日记（可观察部分）

- 与玩家直接互动的部分 → 完全可见
- NPC 的公开行为 → 可见
- NPC 的内心想法 → 默认不可见，除非：
  - 玩家与该NPC关系 ≥ 50（可看到部分内心想法）
  - 通过情报手段获取（雇探子、酒馆打听）

## 6. 裁剪决策记录

| 骑砍原版机制 | 极简化处理 | 判断 | 理由 |
|---|---|---|---|
| 玩家直接操控角色 | 对话式引导 + 请示机制 | **重构** | AIvilization 核心约束 |
| 实时战斗操控 | Agent 自动执行战斗 | **重构** | 玩家不操控实时动作 |
| 暂停菜单 | 请示机制替代 | **变形** | 功能等价 |
| 队伍管理界面 | 通过对话指令管理 | **变形** | 自然语言交互替代 UI 操作 |
| 全知视角 | 有限信息 + 情报系统 | **简化** | 信息获取本身成为有意义的行为 |
