# 数值骨架

> **所属**：极简骑砍规则集 · 模块 3
>
> **设计目标**：定义游戏世界中所有可量化的状态变量、变化规则和平衡关系。以骑砍的"四变量平衡框架"为核心，适配单城镇+三派系的世界结构。
>
> **设计原则**：变量够用即可，绝不引入"可有可无"的维度。每个变量必须满足：①Agent 能理解其含义 ②影响至少一个有意义的决策 ③有清晰的获取和消耗途径。

---

## 0. MVP 回标说明

本文档同时保留两层口径：

- **母稿**：完整单城镇数值骨架，支持后续扩到完整 6 区、12 名 NPC 与更长线经营。
- **首轮可验证原型回标稿**：只启用验证当前体验假设所需的最小数值层。

### 首轮可验证原型（MVP-Must）

- 首轮必须跑通：**Money / Renown / Honor / 三派系关系 / NPC 个人关系 / 随从数量与战力 / 士气 / 粮食 / 派系势力值 / 物价指数 / 当前天数**
- 资源与维护压力按**每 3 天**做一次关键结算
- 数值层负责**门槛、成本、收益、概率和结果结算**

### 数值层与能力层分工

- **数值层**：决定能不能做、代价多大、做完以后数值怎么结算。
- **基模**：决定对目标和上下文的理解深度、规划质量、表达与变通能力。
- **Agent Skill**：决定在特定任务里是否有更稳定的工作流、专门知识、工具调用顺序和校验方式。
- **权限**：决定是否合法、是否有资格访问某区域 / 资源 / 行动。

换句话说：**数值层不负责“怎么想”，只负责“能否做、要付出什么、做完怎么算”**。

## 1. 核心变量总览

### 角色变量

| 变量 | 类型 | 范围 | 说明 |
|---|---|---|---|
| **金钱（Money）** | 整数 | 0 ~ 无上限 | 通用货币。收支不平衡是核心压力源 |
| **声望（Renown）** | 整数 | 0 ~ 100 | 在城邦中的知名度。影响准入门槛和NPC态度 |
| **荣誉（Honor）** | 整数 | -50 ~ 50 | 道德声誉。正值=守信正义，负值=狡诈卑劣 |
| **派系关系** | 整数 × 3 | -100 ~ 100 | 与三大派系各自的关系值 |
| **NPC 个人关系** | 整数 × N | -100 ~ 100 | 与每个NPC的个人关系值 |
| **随从数量** | 整数 | 0 ~ 上限（由声望决定） | 当前带领的随从人数 |
| **随从战力** | 整数 | 由随从数量×平均等级决定 | 部队整体战斗力 |
| **随从士气** | 整数 | 0 ~ 100 | 低士气→逃兵、战斗力下降 |
| **粮食** | 整数 | 0 ~ 储备上限 | 按天消耗，断粮→士气暴跌 |
| **派系身份** | 枚举 | 无/帮手/核心 | 在所属派系中的地位等级 |
| **影响力** | 整数 | 0 ~ 无上限 | 派系内的政治资本，投票和提案消耗 |

### 世界变量

| 变量 | 类型 | 范围 | 说明 |
|---|---|---|---|
| **派系势力值** × 3 | 整数 | 各 0~100，三者之和 = 100 | 三大派系在城邦中的影响力占比 |
| **城邦繁荣度** | 整数 | 0 ~ 100 | 整体经济健康度。影响市场物价、招兵质量 |
| **外部压力标签** | 枚举 | 平静/警戒/紧张 | 首轮以轻量外部信号为主，影响任务、物价和站队压力 |
| **市场物价指数** | 浮点 | 0.5 ~ 2.0 | 基于繁荣度和供需的物价波动系数 |
| **当前天数** | 整数 | 从第 1 天开始 | 游戏内时间，驱动每 3 天结算和事件周期 |
| **城主** | 角色ID / 空缺 | — | 完整母稿变量；首轮通常为空缺或背景常量 |

## 2. 金钱系统（经济循环）

### 收入途径

| 途径 | 场景 | 预期收入（每次） | 频率 |
|---|---|---|---|
| 完成任务 | 各区域接任务 | 50~200 | 每天可接 |
| 交易差价 | 市集区买卖 | 10~100 | 取决于行情判断 |
| 巡逻 / 护民报酬 | 外城农庄、临时任务 | 30~80 | 事件驱动 |
| 比武奖金 | 练兵场参加比武 | 50~150（胜利时） | `Later` |
| 工坊收入 | 投资后每 3 天或长期结算 | 100~300 / 结算周期 | `Later` |
| 贩卖战俘 | 市集区 | 50~200/人 | `Later` |

### 支出途径

| 途径 | 场景 | 预期支出 | 频率 |
|---|---|---|---|
| 随从军饷 | 每 3 天结算 | 10~30/人/结算周期（按等级） | 每 3 天 |
| 粮食采购 | 外城农庄 | 2/人/天 × 储备天数 | 按需 |
| 招募费 | 各区域 | 20~100/人（按兵种） | 按需 |
| 投资工坊 | 市集区 | 500~1500（一次性） | `Later` |
| 宴饮/社交 | 酒馆区 | 20~50/次 | 按需 |
| 贿赂 | 各区域 | 50~500 | 按需 |

首轮原型保留的主收入：**完成任务、交易差价、巡逻 / 护民报酬**

`Later`：比武奖金、工坊收入、战俘经济

首轮原型保留的主支出：**军饷、粮食、招募、社交、贿赂**

`Later`：工坊投资

### 经济平衡目标

- 开局：身无分文或接近于此，必须立即找活干
- 首轮前期（第 1~3 天）：任务与交易是主要收入来源，目标是先养活 2~4 名随从
- 首轮中段（第 4~9 天）：开始承受 3 天一次的军饷与补给压力，玩家需要在赚钱、补给、站队之间取舍
- 首轮验证目标：不依赖工坊和完整领地税收，也能形成稳定但紧张的现金流压力
- `Later`：工坊、区域管理权收入和更大规模队伍维持

## 3. 声望系统

| 声望范围 | 等级 | 解锁内容 |
|---|---|---|
| 0~10 | 无名之辈 | 只能做最基础的行动 |
| 11~25 | 小有名气 | 可觐见权贵、接受派系邀请 |
| 26~50 | 城邦名人 | 可申请加入派系核心、参与议事 |
| 51~75 | 风云人物 | 可竞争区域管理权、影响派系决策 |
| 76~100 | 传奇人物 | 可竞争城主之位 |

首轮原型重点验证：**0~50** 的声望门槛变化

`Later`：51+ 的区域管理权与城主竞争

**获取方式**：战斗胜利（+3~8，以少胜多额外奖励）、任务完成（+1~3）、比武获胜（+2~5）、重大事件参与（+5~15）

**衰减机制**：首轮原型默认**不启用自然衰减**；`Later` 如需加强长线经营压力，可恢复低速衰减。

## 4. 荣誉系统

| 行为 | 荣誉变化 |
|---|---|
| 释放被俘的角色 | +3~5 |
| 信守承诺 | +1 |
| 拒绝不义之举 | +2 |
| 保护平民免受匪帮侵害 | +1~2 |
| 食言/违约 | -3 |
| 接受贿赂 | -2 |
| 无故攻击/劫掠 | -5 |
| 背叛派系 | -5~10 |

**荣誉对 NPC 反应的影响**：正直型重视荣誉，卑劣型无感，狡猾型看情境。

## 5. 关系系统

### 派系关系

| 范围 | 状态 | 影响 |
|---|---|---|
| -100 ~ -51 | 敌对 | 被该派系控制区域拒绝进入，该派系NPC主动敌视 |
| -50 ~ -1 | 不满 | 部分行动受限，任务选择减少 |
| 0 ~ 25 | 中立 | 正常互动 |
| 26 ~ 50 | 友好 | 可以"帮手"身份为派系做事 |
| 51 ~ 75 | 信赖 | 可申请成为派系核心成员 |
| 76 ~ 100 | 核心盟友 | 获得派系全面支持，可竞争领导权 |

**关键约束**：帮助一个派系通常会降低与竞争派系的关系（"关系零和"）。

### NPC 个人关系

与派系关系独立计算。受任务完成（+5~15）、战斗协同（+3~5）、议事投票（±5）、竞争管理权（-10~-20）、性格匹配（±1~3）等影响。

## 6. 随从与军事

| 参数 | 设计值 | 说明 |
|---|---|---|
| 随从上限 | 声望 / 5（向上取整） | 声望 10 = 最多带 2 人，声望 50 = 最多带 10 人 |
| 随从等级 | 1~3 级 | 1=农兵，2=正规军，3=精锐 |
| 战力计算 | Σ(每人等级) | 3 个 1 级 = 战力 3，2 个 3 级 = 战力 6 |
| 士气影响 | 士气 < 30 时每 3 天结算时有逃兵概率 | 每人 10% 概率逃跑 |
| 士气恢复 | 有食物+按时发饷+最近胜利 → 每天+5 | 断粮/欠薪/败仗 → 每天-10 |

### 战斗结算（极简模型）

```
进攻方战力 = 随从战力 × 士气系数（0.5~1.2） × 随机因子（0.8~1.2）
防守方战力 = 同上（防守方可能有地形加成 ×1.2）

胜方 = 战力更高的一方
双方损失 = 按战力比计算：
  - 碾压胜（战力比 > 2:1）：胜方损失 10%，败方损失 60%
  - 优势胜（战力比 1.5~2:1）：胜方损失 20%，败方损失 50%
  - 险胜（战力比 1~1.5:1）：胜方损失 30%，败方损失 40%
败方被俘概率：30%
```

## 7. 派系势力值

三派系势力值之和恒为 100。影响因素：控制区域数量、军事实力、经济产出、民心向背。

首轮原型中，派系势力值在**每 3 天结算**或重大事件后重新计算。

势力值 > 60 代表该派系进入明显优势状态；正式推举城主候选人与强行任命流程保留到 `Later`。

## 8. 繁荣度（Later 为主，首轮只保留简化物价影响）

| 繁荣度范围 | 状态 | 效果 |
|---|---|---|
| 0~20 | 萧条 | 物价×1.5，任务报酬减半 |
| 21~40 | 低迷 | 物价×1.2，商盟更强势，赚钱空间变窄 |
| 41~60 | 正常 | 基准状态 |
| 61~80 | 繁荣 | 物价×0.9，来访机会增多 |
| 81~100 | 鼎盛 | 物价×0.8，高价值任务和外来者更常见 |

首轮原型主要用繁荣度驱动物价与任务生态，不要求完整产业系统。

**升降因素**：贸易活跃 +2 / 每 3 天、护民成功 +3、来访活跃 +1；匪患未处理 -3、内部冲突 -5、商路受阻 -2。

`Later`：攻城胜负、工坊被破坏、精锐招兵质量等更重的联动。

## 9. 数值骨架总结

```
角色层：Money / Renown / Honor / 派系关系×3 / NPC关系×N / 随从（数量/战力/士气）/ 粮食 / 影响力
世界层：派系势力值×3 / 繁荣度 / 外部压力标签 / 物价指数 / 城主 / 天数

核心张力：
  Money ←→ 军事（养兵烧钱）
  Renown ←→ Honor（有时需要不光彩手段获取声望）
  派系A关系 ←→ 派系B关系（帮一方得罪另一方）
  短期收益 ←→ 长期关系（贿赂赚快钱但掉荣誉）
```

首轮原型里，**基模与 Agent Skill 不直接改写这些数值公式**；它们影响的是：

- 先做什么、后做什么
- 选哪条路径去达成目标
- 是否能在信息不足时仍找到更优方案
- 在同样数值状态下，执行效率和结果质量是否更高

数值层负责结算，能力层负责规划与方法。

## 10. 裁剪决策记录

| 骑砍原版机制 | 极简化处理 | 判断 | 理由 |
|---|---|---|---|
| 20+ 种传统数值技能 | 移除（agent skill 另作任务能力包设计） | **移除** | 原版技能系统增加大量状态维度，但与 agent skill 不是同一概念 |
| 精细装备系统 | 移除（战力=随从数×等级） | **移除** | Agent 无法感知装备"手感"差异 |
| 多种食物类型 | 合并为单一"粮食" | **简化** | 种类分化对 Agent 决策无额外价值 |
| 声望/Right to Rule 分离 | 合并为单一"声望" | **简化** | 单城镇下功能重叠 |
| 原版 Trade Skill 影响价格 | 移除（物价由繁荣度+供需决定） | **移除** | 交易获利更依赖判断与局势，而非传统技能等级 |
| 周结算经营循环 | 首轮改为每 3 天结算的轻量维护压力 | **回标收边** | 更适合首轮体验周期，也更容易观察资源压力 |
| 工坊 / 领地税收 / 城主经营 | 保留为 `Later` 母稿，不作为首轮默认收入 | **回标收边** | 先验证基础资源压力与站队后果，再扩展长期经营层 |
| 领地税收（多领地） | 区域管理权收入 | **简化** | 单城镇下无多领地概念 |
