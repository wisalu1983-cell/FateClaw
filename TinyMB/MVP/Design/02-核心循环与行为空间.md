# 核心循环与行为空间

> **所属**：极简骑砍规则集 · 模块 2
>
> **设计目标**：将骑砍的"目标驱动连续行动流"映射到单城镇（石冠城）框架下，定义 Agent 的行为循环、各区域的可执行行动、以及驱动行为多样性的维护压力。

---

## 1. Agent 行为循环

```
感知当前状态（Observe）
  ├─ 自身：金钱、随从规模/状态、声望、荣誉、派系关系、当前位置
  ├─ 环境：所在区域的NPC、事件、机会
  └─ 全局：派系势力对比、外部威胁等级、近期发生的重大事件
    ↓
评估与决策（Think → Decide）
  ├─ 我最紧迫的需求是什么？（钱不够？随从受伤？关系恶化？）
  ├─ 当前有什么机会？（宴会邀请？新任务？比武大赛？）
  └─ 我的长期目标推进到哪一步了？（加入派系？竞争管理权？）
    ↓
执行行动（Act）
  ├─ 移动到目标区域（耗半天，途中可能触发随机事件）
  └─ 在区域内执行具体行动
    ↓
获得结果
  ├─ 数值变化：金钱±、声望±、荣誉±、关系±
  ├─ 状态变化：随从伤亡、获得物品、解锁新区域/NPC
  └─ 叙事事件：产生可被观察的日记/思考内容
    ↓
回到感知 → 循环
```

### 与 Trumen 框架的映射

| Trumen 概念 | 在 TinyMB 中的具体表现 |
|---|---|
| Identity（身份定义） | 角色背景（外来者/派系帮手/派系核心/城主）+ 性格倾向 + 长期目标 |
| Environment（环境感知） | 当前区域 + 区域内可见NPC和事件 + 全局状态摘要 |
| Actions（可执行行动） | 由当前区域 × 角色身份 × 角色状态 三者交叉决定（见第 2 节） |
| Decision Constraints（决策约束） | 时间限制（每天 2~3 个行动）、资源门槛、关系门槛、派系准入 |

## 2. 各区域行动清单

### 城主堡（Citadel）

| 行动 | 前置条件 | 效果 | 骑砍对应 |
|---|---|---|---|
| 觐见权贵 | 声望 ≥ 门槛 或 派系关系 ≥ 门槛 | 对话→可能获得任务、情报、或关系变化 | 拜访领主 |
| 参加议事 | 派系核心身份 | 对派系政策投票，消耗影响力 | 势力投票 |
| 提出议案 | 派系核心身份 + 影响力 ≥ 阈值 | 发起政策/人事/军事提案 | 提议宣战/议和 |
| 探监 | 无特殊限制 | 与被关押的角色对话，可能释放 | 释放被俘领主 |
| 请求裁决 | 有纠纷事件待解决 | 由城主/议事厅裁定争端 | — |

### 市集区（Market）

| 行动 | 前置条件 | 效果 | 骑砍对应 |
|---|---|---|---|
| 买卖商品 | 有金钱 | 交易获利或采购物资，价格受供需波动 | 市场交易 |
| 投资/经营工坊 | 金钱 ≥ 投资额 | 获得持续产出（周结算收入） | 开设工坊 |
| 接行会任务 | 与商盟关系 ≥ 0 | 获得商业/民生类委托 | 接行会长任务 |
| 雇佣服务 | 有金钱 | 雇探子（情报）、雇护卫（临时战力） | — |
| 贩卖战俘 | 持有俘虏 | 获得赎金 | 贩卖战俘 |

### 酒馆区（Tavern）

| 行动 | 前置条件 | 效果 | 骑砍对应 |
|---|---|---|---|
| 招募同伴 | 酒馆内有可招募同伴 | 获得有技能和性格的同伴角色 | 招募 Companion |
| 招募佣兵 | 有金钱 | 获得即战力，维持费较高 | 招募佣兵 |
| 打听消息 | 无 | 获得NPC动向、派系内幕、外部威胁情报 | 酒馆打听 |
| 社交/宴饮 | 无 | 与在场NPC互动，关系±，可能发现机会 | 宴会社交 |
| 地下交易 | 与特定NPC有关系 | 贿赂、黑市交易、秘密协议 | — |

### 练兵场（Training Grounds）

| 行动 | 前置条件 | 效果 | 骑砍对应 |
|---|---|---|---|
| 训练随从 | 有随从 | 随从经验增长，可升级兵种 | 部队训练 |
| 参加比武 | 无 | 胜利→金钱+声望；失败→受伤 | 竞技场比武 |
| 招募军士 | 有金钱，与守备军团关系 ≥ 0 | 获得有一定训练水平的兵员 | 城镇招兵 |
| 切磋/挑战 | 目标NPC在场 | 与特定NPC比武→关系变化+声望变化 | — |

### 外城农庄（Farmlands）

| 行动 | 前置条件 | 效果 | 骑砍对应 |
|---|---|---|---|
| 招募农兵 | 与平民公社关系 ≥ 0 | 获得基础兵源，成本最低但战力弱 | 村庄招兵 |
| 购买粮食/物资 | 有金钱 | 补充队伍补给 | 村庄采购 |
| 接平民任务 | 与平民公社关系 ≥ 0 | 获得民生类委托（剿匪、护送、修缮等） | 接村长任务 |
| 巡逻保护 | 有随从 | 驱赶匪帮→平民关系↑，声望↑；可能触发战斗 | 巡逻护民 |
| 拜访公社领袖 | 无 | 了解平民诉求，可能影响后续任务 | 拜访村长 |

### 城墙与城门（Walls & Gates）

| 行动 | 前置条件 | 效果 | 骑砍对应 |
|---|---|---|---|
| 执行巡逻 | 有随从 | 发现外部威胁→预警；声望↑，守备军团关系↑ | 领地巡逻 |
| 参与城防 | 外敌攻城事件发生中 | 协助防御→战斗结算，胜利→大量声望和关系 | 防守攻城战 |
| 观察外部动态 | 无 | 获得外部势力、商路、威胁等级的信息 | 查看大地图 |
| 迎接/拦截来访者 | 值守时 | 与外来者互动→可能获得机会/任务 | 野外遭遇对话 |

## 3. 时间系统

| 参数 | 设计值 | 说明 |
|---|---|---|
| 基本时间单位 | **半天**（上午/下午） | 每次区域移动+行动 = 消耗 1 个半天 |
| 每天可用时间 | 2 个半天（上午+下午） | 夜晚为休息时间，不可行动 |
| 夜晚事件 | 无主动行动，但可能发生被动事件 | 夜袭、偷盗、紧急召集等 |
| 周结算 | 每 7 天 | 军饷支出、工坊收入、领地税收、食物消耗、派系势力更新 |
| 外部威胁周期 | 约 20~30 天 | 触发一次重大外部事件 |

每天只有 2 个行动机会 → Agent 必须在"经济/军事/社交/情报"之间取舍，这是骑砍核心循环中"总有做不完的事"的压力来源。

## 4. 维护压力（驱动日常行为）

| 维护项 | 机制 | 不处理的后果 | 骑砍对应 |
|---|---|---|---|
| **军饷** | 每周结算，随从数量×等级决定支出 | 欠薪→士气下降→逃兵 | 军队维持费 |
| **粮食** | 随从每天消耗粮食，需定期采购 | 断粮→士气暴跌→战斗力下降 | 队伍食物消耗 |
| **伤员恢复** | 战斗后随从受伤，需要时间恢复 | 带伤出战→减员风险 | 伤兵恢复 |
| **关系维护** | 长期不互动的NPC关系会缓慢衰减 | 关系降低→失去支持/准入权 | 关系自然衰减 |
| **派系义务** | 加入派系后需响应召集、执行任务 | 不响应→派系关系恶化→可能被开除 | 封臣军事义务 |

## 5. 行动分类与四变量影响

| 行动类型 | 代表行动 | Money | Renown | Honor | Relationships |
|---|---|---|---|---|---|
| **经济活动** | 交易、经营工坊、接商业任务 | ↑↑ | — | — | 商盟↑ |
| **军事活动** | 训练、比武、巡逻、城防 | ↓（军饷） | ↑↑ | — | 守备军团↑ |
| **社交活动** | 宴饮、拜访、对话 | ↓（花费） | — | — | 目标NPC↑↑ |
| **政治活动** | 议事、投票、提案 | — | ↑ | ±（取决于决定） | ±（多方影响） |
| **民生活动** | 剿匪、护送、保护农庄 | ↑（报酬） | ↑ | ↑ | 平民公社↑↑ |
| **灰色活动** | 贿赂、地下交易、偷袭 | ↑↑ | — | ↓↓ | ±（看对象） |

### 四变量冲突实例

| 情境 | 选择 A | 选择 B | 冲突的变量 |
|---|---|---|---|
| 抓到了商盟的走私犯 | 上交守备军团：Honor↑ 守备关系↑ | 私放（收取贿赂）：Money↑ 商盟关系↑ | Honor vs Money |
| 平民暴动要求减税 | 支持平民：平民关系↑↑ 商盟关系↓ | 支持商盟：Money↑ 平民关系↓↓ | Relationships冲突 |
| 敌方斥候被俘 | 释放：Honor↑ 但泄露城防信息风险 | 关押/处决：军事安全↑ 但Honor↓ | Honor vs 军事安全 |
| 比武大赛中对手是盟友 | 全力出手赢：Renown↑↑ 但关系↓ | 放水让对方赢：关系↑ 但Renown↓ | Renown vs Relationships |

## 6. 裁剪决策记录

| 骑砍原版机制 | 极简化处理 | 判断 | 理由 |
|---|---|---|---|
| 连续自由行动 | 半天制（每天 2 个行动机会） | **简化** | Agent 需要离散的决策点 |
| 大地图+聚落双层行动 | 单层：区域内行动 | **合并** | 单城镇无需双层结构 |
| 20+ 种技能 | MVP 移除 | **移除** | 用角色身份+同伴效果替代 |
| 实时战斗操控 | 自动结算 | **简化** | Agent 只做战略决策 |
| 食物种类分化 | 统一为"粮食" | **简化** | 种类分化对 Agent 决策无额外价值 |
| 装备系统 | MVP 移除 | **移除** | 用"随从整体战力"替代 |
| 同伴技能分配 | 保留简化版（专长标签） | **简化** | 同伴增加叙事丰富度，但专长简化 |
