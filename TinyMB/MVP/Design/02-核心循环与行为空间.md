# 核心循环与行为空间

> **所属**：极简骑砍规则集 · 模块 2
>
> **设计目标**：将骑砍的"目标驱动连续行动流"映射到单城镇（石冠城）框架下，定义 Agent 的行为循环、各区域的可执行行动、以及驱动行为多样性的维护压力。

---

## 0. MVP 回标说明

本文档同时保留两层口径：

- **母稿**：完整极简骑砍规则集的行动空间设计，供后续扩回完整版本时参考。
- **首轮可验证原型回标稿**：明确首轮只启用哪些区域、行动和节奏，用于验证当前体验假设。

### 首轮可验证原型（MVP-Must）

- 活跃区域锁定为：**城主堡 / 市集区 / 酒馆区 / 外城农庄**
- 玩家 Agent 的行动规划显式受 **基模 + 已启用 Agent Skill + 角色状态 + 权限** 共同影响
- Agent Skill 采用**预制库 + 启用 / 停用 + 槽位限制**的方式接入，不做自由成长系统
- 外部威胁只保留**轻量替代物**：来访者、商路异动、匪患传闻、农庄求援、警戒状态变化

### 首轮原型后放（Later）

- 练兵场、城墙与城门作为独立区域
- 完整训练线、比武线、城防 / 攻城循环
- 完整外部威胁周期事件

## 1. Agent 行为循环

```
感知当前状态（Observe）
  ├─ 自身：当前基模、已启用的 Agent Skill、金钱、随从规模/状态、声望、荣誉、派系关系、当前位置
  ├─ 环境：所在区域的NPC、事件、机会
  └─ 全局：派系势力对比、轻量外部压力、近期发生的重大事件
    ↓
评估与决策（Think → Decide）
  ├─ 我最紧迫的需求是什么？（钱不够？补给不足？关系恶化？）
  ├─ 当前启用的 Agent Skill 是否适合这个目标？如果不适合，有没有替代路径？
  ├─ 当前有什么机会？（宴会邀请？新任务？来访者消息？）
  └─ 是否命中重要决策条件，需要向玩家生成请示包？
    ↓
执行行动（Act）
  ├─ 若目标不在当前区域，先消耗移动时间到达目标区域（5~20 分钟，途中可能触发随机事件）
  ├─ 在目标区域内执行具体行动；若当前目标明确，可连续处理多件同区事务
  └─ 若命中重要决策、风险节点或能力缺口，再暂停等待玩家对请示包做出决定
    ↓
获得结果
  ├─ 数值变化：金钱±、声望±、荣誉±、关系±
  ├─ 状态变化：随从伤亡、获得物品、解锁新区域/NPC
  └─ 叙事事件：产生可被观察的日记/思考内容，并暴露本次行动的能力依据
    ↓
回到感知 → 循环
```

### 与 Trumen 框架的映射

| Trumen 概念 | 在 TinyMB 中的具体表现 |
|---|---|
| Identity（身份定义） | 角色背景（外来者/派系帮手/派系核心/城主）+ 性格倾向 + 长期目标 |
| Environment（环境感知） | 当前区域 + 区域内可见NPC和事件 + 全局状态摘要 |
| Actions（可执行行动） | 由当前区域 × 角色身份 × 角色状态 三者交叉决定（见第 2 节） |
| Decision Constraints（决策约束） | 连续时钟（10 分钟基本粒度）+ 跨区移动耗时 + 时段窗口 / 事件时效、资源门槛、关系门槛、派系准入 |

### 部署与权威边界

上述行为循环的部署位置按 Agent 类型拆分：

| Agent 类型 | 循环运行位置 | LLM 来源 | 行为框架约束 |
|---|---|---|---|
| 玩家 Agent | 用户侧 | 用户选择的基模 | 官方系统 prompt（不可修改） |
| 命名 NPC Agent | 服务端 | 服务端统一 LLM | 服务端配置 |

两类 Agent 对 Server/Runtime 的接口完全相同：循环中的 Act 输出**结构化行动意图**（而非最终结果），由 Runtime 校验合法性并结算产生权威结果后回传。

### P2：结构化行动意图格式（MVP-Must）

首轮原型的行动输出契约采用**混合方案**：

- `Runtime` 权威执行时，**只认一个主动作**（`primary_action`）
- 行动意图中允许同时携带该主动作背后的**后续意图**、**中断条件**、**能力依据 / 能力缺口**与**回退策略**
- `Runtime` 不会把这些背景字段当作必须一次性执行完的硬计划，它们主要服务于：请示、日志、回放、调试与后续 skill 差异验证

#### 契约原则

- **一条意图只提交一个权威主动作**，避免把多步计划直接塞给 `Runtime`
- **若当前不在目标区域**，主动作必须先是 `move`；到达后的打算写入 `followup_plan`
- **若当前已经命中重要决策 / 高风险边界**，主动作不应直接提交高风险 `act`，而改为 `ask`
- **`followup_plan` 只记录 1~2 条下一步打算**，不允许退化成完整可执行任务队列

#### 最小结构

```json
{
  "intent_id": "uuid",
  "actor_id": "player_agent_01",
  "goal_ref": "确认商盟是否囤粮",
  "primary_action": {
    "type": "act",
    "name": "gather_info",
    "target_region": "酒馆区",
    "target_id": "老酒·加利",
    "params": {
      "topic": "粮价与商盟动向",
      "approach": "low_profile"
    },
    "expected_duration_minutes": 10
  },
  "followup_plan": [
    "若消息可信，则去外城农庄核价"
  ],
  "interrupt_points": [
    {
      "trigger": "需要直接与金秤·杜兰高风险交涉",
      "reason": "失败成本高，且当前缺少强交涉能力",
      "response": "ask"
    }
  ],
  "capability_basis": [
    "当前启用情报类 Agent Skill",
    "与老酒·加利的关系"
  ],
  "capability_gap": [
    "缺少强交涉能力"
  ],
  "fallback": "若目标不在场，则改为收集公开市场情报"
}
```

#### 字段说明

| 字段 | 是否必填 | 说明 |
|---|---|---|
| `intent_id` | 是 | 本次行动意图的唯一标识，用于日志、回放与追踪 |
| `actor_id` | 是 | 提交该意图的 Agent 标识 |
| `goal_ref` | 是 | 当前短期目标的简短描述，便于 `Runtime`、日志和请示包对齐上下文 |
| `primary_action.type` | 是 | `move / act / ask / wait` 四选一 |
| `primary_action.name` | 条件必填 | 当 `type = act` 时必填，对应标准化动作名，如 `gather_info`、`trade_goods`、`hire_service`、`seek_audience`、`patrol` |
| `primary_action.target_region` | 条件必填 | 目标区域；`move` 与绝大多数 `act` 都需要 |
| `primary_action.target_id` | 否 | 目标 NPC、事件、任务或物件 |
| `primary_action.params` | 否 | 行动的必要参数，如情报主题、交易对象、交涉方式 |
| `primary_action.expected_duration_minutes` | 是 | Agent 预估本动作耗时，最终以 `Runtime` 实际结算为准 |
| `followup_plan` | 否 | 1~2 条下一步打算，只做说明，不作为 `Runtime` 的自动执行队列 |
| `interrupt_points` | 否 | 记录需要请示、改道或放弃的条件；每条至少包含 `trigger / reason / response` |
| `capability_basis` | 否 | 当前主动作主要依赖的基模 / Agent Skill / 数值 / 权限 |
| `capability_gap` | 否 | 当前目标存在但尚未补齐的能力缺口 |
| `fallback` | 否 | 主动作被阻断、目标缺席或条件不足时的安全默认方案 |

#### `Runtime` 的处理边界

`Runtime` 只把 `primary_action` 视为权威执行请求，并据此完成：

1. 合法性校验：身份、权限、区域准入、目标存在性
2. 成本与耗时结算：时间、资源、关系、风险与结果
3. 执行反馈回传：成功、失败、受阻、生成新观察结果

以下字段**不由 `Runtime` 自动串行执行**：

- `followup_plan`
- `interrupt_points`
- `capability_basis`
- `capability_gap`
- `fallback`

它们主要用于：

- 生成请示包
- 写入日志与状态回顾
- 做失败归因与路径比较
- 在后续阶段验证基模 / Agent Skill 是否改变了规划方式

#### 四类主动作的使用规则

| `primary_action.type` | 何时使用 | 例子 |
|---|---|---|
| `move` | 当前不在目标区域，需要先移动 | 去酒馆区、去外城农庄 |
| `act` | 已在目标区域，且当前动作可直接执行 | 打听消息、买卖商品、请求裁决 |
| `ask` | 已命中重要决策 / 高风险边界，不应直接执行下一步 | 需要明确站队、需要大额投入、需要高风险交涉 |
| `wait` | 当前没有安全且合法的可执行动作，选择等待或保持观察 | 夜间等待、等待玩家回复、等待事件变化 |

#### 典型例子

1. **不在目标区域时**
   - 当前在市集区，想去酒馆打听粮价
   - 本轮主动作应为 `move`
   - `followup_plan` 再写“到达后打听消息”

2. **已经在目标区域且行动安全时**
   - 当前就在酒馆区，且只是普通打听
   - 本轮主动作可直接为 `act:gather_info`

3. **已经命中重要决策时**
   - 当前准备直接和命名 NPC 高风险交涉
   - 本轮主动作不应直接提交 `act`
   - 应改为 `ask`，并把拟执行动作、理由、替代路线和回退写入背景字段

### P3：观察包与世界状态摘要字段（MVP-Must）

首轮原型中，Agent 在每轮 `Observe` 阶段接收的不是全量世界快照，而是“公共世界信息 + 个人被召回记忆”的最小观察包。当前观察与 `memory_recall` 共同构成后续 `Think / Decide` 的输入。

#### 设计原则

- `available_actions` 是机制硬边界：由 `Runtime` 基于位置、时间、资源、权限与状态锁定等条件裁定本轮合法可提交的主动作，等价于真人界面里当前可点击的动作集合
- `world_state_summary` 只放普通人普遍会知道的客观世界信息，不写“这对我意味着什么”的个人解释
- `memory_recall` 只放当前问题触发的个人记忆 `Top N`；完整记忆系统的写入、预算、衰减与覆盖规则留作后续任务
- 角色职业 / 派系 / 性格与玩家长期方针不改变公共世界真相，只影响记忆写入优先级、召回优先级和模型最终筛选

#### 最小结构

```json
{
  "self_state": {
    "current_time": "09:10",
    "location_id": "酒馆区",
    "current_primary_goal": "确认商盟是否在囤粮"
  },
  "local_snapshot": {
    "visible_npcs": ["老酒·加利", "酒馆老板"],
    "local_events": ["来访者带来粮价异动消息"]
  },
  "world_state_summary": {
    "top_public_signals": [
      {
        "headline": "农庄求援剩余半天",
        "signal_type": "deadline",
        "time_pressure": "half_day"
      }
    ]
  },
  "memory_recall": {
    "recalled_items": [
      {
        "memory_type": "promise",
        "summary": "昨天答应农庄管事今天给回复",
        "linked_entities": ["农庄管事"],
        "relevance_to_current_issue": "是否应先去农庄而不是继续打听",
        "risk_if_ignored": "失约并降低关系"
      }
    ]
  },
  "available_actions": {
    "allowed_primary_actions": ["move", "act", "ask", "wait"],
    "blocked_actions_with_reason": [
      "无法直接 seek_audience：当前不在城主堡且准入不足"
    ]
  }
}
```

#### 字段分块

| 区块 | 作用 | 首轮最小字段 |
|---|---|---|
| `self_state` | 说明“我现在是什么状态” | `current_time`、`location_id`、`money`、`renown`、`honor`、`faction_relations_summary`、`follower_state_summary`、`current_long_term_policies`、`current_primary_goal`、`enabled_skills`、`model_profile_ref` |
| `local_snapshot` | 说明“我眼前有什么” | `visible_npcs`、`local_events`、`local_opportunities`、`local_risks`、`local_constraints`、`reachable_locations` |
| `world_state_summary` | 说明“当前世界有哪些普通人也该知道的重要公共变化” | `top_public_signals`（`1~3` 条），每条只保留 `headline / signal_type / time_pressure` |
| `memory_recall` | 说明“我这次想起了哪些与当前问题相关的个人记忆” | `recalled_items`（`Top N`），每条至少保留 `memory_type / summary / linked_entities / relevance_to_current_issue / risk_if_ignored` |
| `available_actions` | 说明“这轮合法能做什么” | `allowed_primary_actions`、`blocked_actions_with_reason` |

#### `world_state_summary` 与 `memory_recall` 的边界

- `world_state_summary` 回答的是：**当前世界外部态势里，普通人普遍会知道什么**
- `memory_recall` 回答的是：**我过去记住的内容里，这次该想起什么**

同一件事若两边都要出现，应使用不同角度表述：

- `world_state_summary`：农庄求援剩余半天
- `memory_recall`：我昨天已经答应农庄管事今天给答复

#### 当前问题相关记忆的最小召回规则

为避免 `P3` 直接膨胀成完整认知系统，首轮只定召回接口，不定完整记忆公式。当前可先采用以下轻量规则：

1. 与当前主目标、当前地点、当前 NPC 或当前事件直接相关的记忆优先召回
2. 有时限、未兑现承诺或高风险后果的记忆优先召回
3. 角色职业 / 派系 / 性格与玩家长期方针对召回排序产生影响
4. 最后只将 `Top N` 条结果写入 `memory_recall`

这让“当前观察 + 被召回记忆 = 共同输入”先跑通，而完整的记忆写入、预算、衰减与覆盖机制保留为后续设计任务。

### 首轮原型中的能力入口（MVP-Must）

- **基模**：玩家 Agent 的基模必须可通过配置参数切换，作为首轮原型的显式验证变量。用户选择基模只影响推理质量，行为框架由官方系统 prompt 锁定。
- **Agent Skill**：采用预制库，玩家只能决定启用或停用；同一时间可启用的 skill 数量受槽位或容量限制。
- **角色状态 / 权限**：负责提供门槛、合法性和确定性结算边界，不替代基模与 Agent Skill 的规划作用。

## 2. 各区域行动清单

### 首轮区域回标

| 区域 | 首轮回标 | 处理方式 |
|---|---|---|
| 城主堡 | **MVP-Must** | 保留为派系接触和高门槛政治入口 |
| 市集区 | **MVP-Must** | 保留为赚钱、物资和正规任务入口 |
| 酒馆区 | **MVP-Must** | 保留为社交、情报和灰色机会入口 |
| 外城农庄 | **MVP-Must** | 保留为平民公社、补给、低成本招募和护民任务入口 |
| 练兵场 | `Later` | 独立区域后放，首轮不做完整训练 / 比武线 |
| 城墙与城门 | `Later` | 独立区域后放，首轮用轻量外部压力替代 |

### 城主堡（Citadel）`MVP-Must`

| 行动 | 前置条件 | 效果 | 骑砍对应 |
|---|---|---|---|
| 觐见权贵 | 声望 ≥ 门槛 或 派系关系 ≥ 门槛 | 对话→可能获得任务、情报、或关系变化 | 拜访领主 |
| 参加议事 | 派系核心身份 | 对派系政策投票，消耗影响力 | 势力投票 |
| 提出议案 | 派系核心身份 + 影响力 ≥ 阈值 | 发起政策/人事/军事提案 | 提议宣战/议和 |
| 探监 | 无特殊限制 | 与被关押的角色对话，可能释放 | 释放被俘领主 |
| 请求裁决 | 有纠纷事件待解决 | 由城主/议事厅裁定争端 | — |

首轮原型保留：**觐见权贵、请求裁决**

`Later`：参加议事、提出议案、探监

### 市集区（Market）`MVP-Must`

| 行动 | 前置条件 | 效果 | 骑砍对应 |
|---|---|---|---|
| 买卖商品 | 有金钱 | 交易获利或采购物资，价格受供需波动 | 市场交易 |
| 投资/经营工坊 | 金钱 ≥ 投资额 | 获得持续产出（被动收入） | 开设工坊 |
| 接行会任务 | 与商盟关系 ≥ 0 | 获得商业/民生类委托 | 接行会长任务 |
| 雇佣服务 | 有金钱 | 雇探子（情报）、雇护卫（临时战力） | — |
| 贩卖战俘 | 持有俘虏 | 获得赎金 | 贩卖战俘 |

首轮原型保留：**买卖商品、接行会任务、雇佣服务**

`Later`：投资/经营工坊、贩卖战俘

### 酒馆区（Tavern）`MVP-Must`

| 行动 | 前置条件 | 效果 | 骑砍对应 |
|---|---|---|---|
| 招募同伴 | 酒馆内有可招募同伴 | 获得有专长标签和性格的同伴角色 | 招募 Companion |
| 招募佣兵 | 有金钱 | 获得即战力，维持费较高 | 招募佣兵 |
| 打听消息 | 无 | 获得NPC动向、派系内幕、外部威胁情报 | 酒馆打听 |
| 社交/宴饮 | 无 | 与在场NPC互动，关系±，可能发现机会 | 宴会社交 |
| 地下交易 | 与特定NPC有关系 | 贿赂、黑市交易、秘密协议 | — |

首轮原型保留：**招募佣兵、打听消息、社交/宴饮、地下交易**

`Later`：招募同伴

### 练兵场（Training Grounds）`Later`

| 行动 | 前置条件 | 效果 | 骑砍对应 |
|---|---|---|---|
| 训练随从 | 有随从 | 随从经验增长，可升级兵种 | 部队训练 |
| 参加比武 | 无 | 胜利→金钱+声望；失败→受伤 | 竞技场比武 |
| 招募军士 | 有金钱，与守备军团关系 ≥ 0 | 获得有一定训练水平的兵员 | 城镇招兵 |
| 切磋/挑战 | 目标NPC在场 | 与特定NPC比武→关系变化+声望变化 | — |

首轮可验证原型不启用独立练兵场。相关体验处理方式：

- 基础兵源由**外城农庄招募农兵**与**酒馆区招募佣兵**承担
- 完整训练、比武、军士招募和切磋线后放到 `Later`

### 外城农庄（Farmlands）`MVP-Must`

| 行动 | 前置条件 | 效果 | 骑砍对应 |
|---|---|---|---|
| 招募农兵 | 与平民公社关系 ≥ 0 | 获得基础兵源，成本最低但战力弱 | 村庄招兵 |
| 购买粮食/物资 | 有金钱 | 补充队伍补给 | 村庄采购 |
| 接平民任务 | 与平民公社关系 ≥ 0 | 获得民生类委托（剿匪、护送、修缮等） | 接村长任务 |
| 巡逻保护 | 有随从 | 驱赶匪帮→平民关系↑，声望↑；可能触发战斗 | 巡逻护民 |
| 拜访公社领袖 | 无 | 了解平民诉求，可能影响后续任务 | 拜访村长 |

首轮原型保留：**招募农兵、购买粮食/物资、接平民任务、巡逻保护、拜访公社领袖**

### 城墙与城门（Walls & Gates）`Later`

| 行动 | 前置条件 | 效果 | 骑砍对应 |
|---|---|---|---|
| 执行巡逻 | 有随从 | 发现外部威胁→预警；声望↑，守备军团关系↑ | 领地巡逻 |
| 参与城防 | 外敌攻城事件发生中 | 协助防御→战斗结算，胜利→大量声望和关系 | 防守攻城战 |
| 观察外部动态 | 无 | 获得外部势力、商路、威胁等级的信息 | 查看大地图 |
| 迎接/拦截来访者 | 值守时 | 与外来者互动→可能获得机会/任务 | 野外遭遇对话 |

首轮可验证原型不启用独立城墙区。相关体验改为以下轻量替代物：

- **来访者事件**：在酒馆区或市集区触发，承担外部消息和新机会输入
- **匪患传闻 / 农庄求援**：在外城农庄或酒馆区触发，承担最小外部压力
- **警戒状态变化**：作为全局状态输入写入日志与观察摘要，不做完整攻城循环

## 3. 时间系统

| 参数 | 设计值 | 说明 |
|---|---|---|
| 基本时间粒度 | **10 分钟** | 用于保留连续行动感；绝大多数短行为按 10 分钟计时 |
| 常规短行为 | 10 分钟/次 | 例：打听消息、单次买卖、简短对话、基础采购 |
| 中等行为 | 20 分钟/次 | 例：正式觐见、完整社交、处理普通委托 |
| 重行为 | 30 分钟/次 | 例：巡逻保护、复杂交涉、较长任务处理 |
| 世界时钟 | 连续推进 | 玩家可看到具体时刻或自然时段；夜间仍推进，但正式事务活跃度下降 |
| 结算周期 | 每 3 天 | 军饷支出、食物消耗、基础关系维护、局势摘要更新 |
| 轻量外部压力 | 1~3 天出现一次相关消息或小事件 | 来访者、商路异动、匪患传闻、农庄求援 |
| 完整外部威胁周期 | `Later` | 完整围城 / 攻城 / 强制协作后放 |

### 首轮原型的区域移动耗时

| 路线 | 耗时 | 设计意图 |
|---|---|---|
| 酒馆区 ↔ 市集区 | 5 分钟 | 保留商业区与情报区的高频切换，但不允许零成本折返 |
| 城主堡 ↔ 市集区 | 10 分钟 | 政务与市场相邻，但来回跑仍有明显代价 |
| 城主堡 ↔ 酒馆区 | 10 分钟 | 让政治接触与情报搜集之间存在取舍 |
| 市集区 ↔ 外城农庄 | 15 分钟 | 从赚钱 / 采购切换到补给 / 护民需要提前规划 |
| 酒馆区 ↔ 外城农庄 | 15 分钟 | 从消息验证切换到农庄核实时有明显成本 |
| 城主堡 ↔ 外城农庄 | 20 分钟 | 直接从政务区下乡最耗时，强化“先计划再动” |

首轮原型的压力来源不再是“每天只有 2 个行动机会”，而改为以下三项叠加：

- **跨区折返成本**：一次移动会吃掉 5~20 分钟，一天内多次来回会明显压缩办事时间
- **时段窗口**：城主堡正式事务、市集交易活跃期、酒馆消息高峰等会影响同一行动的价值
- **事件时效**：农庄求援、商路异动、来访者机会等拖延后会恶化或消失

因此，Agent 需要先规划“进一个区域后顺手完成哪些事”，再决定是否换区。这保留骑砍式“总有做不完的事”的压力来源，同时避免体验被压成强回合制。

## 4. 维护压力（驱动日常行为）

| 维护项 | 机制 | 不处理的后果 | 骑砍对应 |
|---|---|---|---|
| **军饷** | 每 3 天结算，随从数量×等级决定支出 | 欠薪→士气下降→逃兵 | 军队维持费 |
| **粮食** | 随从每天消耗粮食，需定期采购 | 断粮→士气暴跌→战斗力下降 | 队伍食物消耗 |
| **伤员恢复** | 战斗后随从受伤，需要时间恢复 | 带伤出战→减员风险 | 伤兵恢复 |
| **关系维护** | 长期不互动的NPC关系会缓慢衰减 | 关系降低→失去支持/准入权 | 关系自然衰减 |
| **派系义务** | 加入派系后需响应召集、执行任务 | 不响应→派系关系恶化→可能被开除 | 封臣军事义务 |

## 5. 行动分类与四变量影响

| 行动类型 | 代表行动 | Money | Renown | Honor | Relationships |
|---|---|---|---|---|---|
| **经济活动** | 交易、经营工坊、接商业任务 | ↑↑ | — | — | 商盟↑ |
| **军事活动** | 训练、比武、巡逻、城防 | ↓（军饷） | ↑↑ | — | 守备军团↑ |
| **社交活动** | 宴饮、拜访、对话 | ↓（花费） | — | — | 目标NPC↑↑ |
| **政治活动** | 议事、投票、提案 | — | ↑ | ±（取决于决定） | ±（多方影响） |
| **民生活动** | 剿匪、护送、保护农庄 | ↑（报酬） | ↑ | ↑ | 平民公社↑↑ |
| **灰色活动** | 贿赂、地下交易、偷袭 | ↑↑ | — | ↓↓ | ±（看对象） |

### 四变量冲突实例

| 情境 | 选择 A | 选择 B | 冲突的变量 |
|---|---|---|---|
| 抓到了商盟的走私犯 | 上交守备军团：Honor↑ 守备关系↑ | 私放（收取贿赂）：Money↑ 商盟关系↑ | Honor vs Money |
| 平民暴动要求减税 | 支持平民：平民关系↑↑ 商盟关系↓ | 支持商盟：Money↑ 平民关系↓↓ | Relationships冲突 |
| 敌方斥候被俘 | 释放：Honor↑ 但泄露城防信息风险 | 关押/处决：军事安全↑ 但Honor↓ | Honor vs 军事安全 |
| 比武大赛中对手是盟友 | 全力出手赢：Renown↑↑ 但关系↓ | 放水让对方赢：关系↑ 但Renown↓ | Renown vs Relationships |

## 6. 裁剪决策记录

| 骑砍原版机制 | 极简化处理 | 判断 | 理由 |
|---|---|---|---|
| 完整 6 区空间 | 首轮启用 4 区（城主堡/市集区/酒馆区/外城农庄） | **回标收边** | 满足 Must 所需的赚钱、社交/情报、派系接触与平民/补给空间 |
| 连续自由行动 | 连续时钟（10 分钟基本粒度）+ 跨区移动耗时 + 同区连续办事 | **变形** | 保留连续行动感，同时通过移动成本、时段窗口和事件时效制造规划压力 |
| 大地图+聚落双层行动 | 单层：区域内行动 | **合并** | 单城镇无需双层结构 |
| 20+ 种传统数值技能 | MVP 移除（agent skill 另作任务能力包设计） | **移除** | 原版技能系统增加大量状态维度，且与 agent skill 不是同一概念 |
| 完整外部威胁周期 | 首轮改为轻量外部压力替代物 | **回标收边** | 先验证世界动态与最小外生压力，不直接实现完整攻城系统 |
| 实时战斗操控 | 自动结算 | **简化** | Agent 只做战略决策 |
| 食物种类分化 | 统一为"粮食" | **简化** | 种类分化对 Agent 决策无额外价值 |
| 装备系统 | MVP 移除 | **移除** | 用"随从整体战力"替代 |
| 同伴专长分配 | 保留简化版（专长标签） | **简化** | 同伴增加叙事丰富度，但专长简化 |
