# TinyMB 项目背景介绍

## 项目名称

TinyMB（Tiny Mount & Blade）—— 骑马与砍杀极简版

## 项目定位

TinyMB 是 **AIvilization Plus** 的 MVP 版本，以骑马与砍杀的核心玩法为基底，极简化后接入多 Agent 框架，验证"AI Agent 在持续运转的沙盒世界中自主闯荡"的核心体验。

## 来龙去脉

### 前序项目：FateClaw

FateClaw 最初计划做一个 Agent 驱动的圣杯战争模拟游戏（详见 `Brainstorming/Agent圣杯战争提案-对话提炼.md`）。MVP v1 阶段完成了 4 份 A 级设计文档（核心循环、数值框架、行动系统、情报系统），但复杂度严重膨胀，暂停并封存（详见 `MVP/Doc/MVP-v1设计讨论封存总结.md`）。

### 核心玩法选型

暂停后进行了系统化的核心玩法对标脑暴（详见 `Brainstorming/玩法对标与脑暴/02-核心玩法对标脑暴.md`），评估了 8 个候选方案，结论是 **K. 极简骑砍** 综合最强——骑砍本身就是"角色在持续运转的沙盒世界中闯荡"，与 AIvilization 架构同构，且是唯一在全部 10 种 Agent 能力模式上均达到高度利用、不依赖任何未成熟能力的方案。

### TinyMB 的目标

基于上述结论，TinyMB 的目标是：

1. 将骑砍核心玩法极简化为可被 Agent 框架驱动的规则集
2. 基于该规则集开发可运行的 MVP
3. 验证核心体验假设和 Trumen 多 Agent 框架的技术可行性

## 技术底座

- **Trumen 多 Agent 框架**：Client / Agent Framework / Server(Runtime) 三层结构
- **Agent 框架职责**：驱动 Observe → Think → Decide → Act 循环
- **与玩法解耦**：框架通过标准化接口接收具体游戏的环境适配模型（身份/环境/行为/决策约束四类定义）

详见 `Brainstorming/多Agent框架相关/多Agent框架讨论总结.md`

## 设计约束

| 约束 | 来源 |
|---|---|
| Agent 是一个"人"（有日常行为、情绪、日记） | AIvilization 架构 |
| 世界连续运行（非回合制） | AIvilization 架构 |
| 玩家引导而非控制 | AIvilization 架构 |
| 多 Agent 自主交互 | AIvilization 架构 |
| 放置/挂机体验 | AIvilization 架构 |
| 空间/地图维度 | AIvilization 架构 |
| Agent 内心可观察 | AIvilization 架构 |
| 依赖已成熟 Agent 能力，避开未成熟能力 | Agent 技术边界 |
| 先回答"最小系统是什么" | MVP v1 复杂度膨胀教训 |
