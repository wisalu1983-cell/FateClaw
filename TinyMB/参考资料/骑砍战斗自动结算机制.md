# 骑砍战斗自动结算机制

> **文档用途**：详细记录骑马与砍杀（特别是霸主）的战斗自动结算（Auto-Resolve / Combat Simulation）系统的完整机制和公式，作为 TinyMB 战斗结算模型设计（Q-05）的输入材料。
>
> **数据来源**：霸主 v1.3.2 反编译源码分析（[jzebedee/Bannerlord Combat Simulation](https://gist.github.com/jzebedee/be076d28f162c8d05fd7d2d72109a46f)）、Steam 社区讨论、Nexus Mods 社区。

---

## 一、系统概述

霸主的战斗自动结算不是简单的"比数字"，而是一个**多轮模拟系统**：

```
设置战场（双方部队按优先级排序）
    ↓
计算双方的回合数（基于场景类型）
    ↓
执行回合循环：
    每回合：随机决定进攻方/防守方 → 执行模拟战斗
    ↓
直到一方耗尽部队或士气 → 溃败
    ↓
宣布胜者
```

### 关键结论

- **战术技能（Tactics）** 是影响模拟结果最重要的技能
- **防守方天然占优**（聚落防御优势 2:1 到 7:1）
- **士气极其重要**——触发溃败是大型战斗的胜负关键
- **步兵**是模拟战中最全能的兵种，从不被惩罚
- **骑兵**在模拟战中表现最差
- **弓箭手和骑射手**在守城模拟中表现最好

## 二、战斗轮次计算

### 2.1 四种场景类型

#### 大型攻城战

当城镇防守兵力 > 100 人 或 城堡防守兵力 > 30 人时触发。

- 聚落防御优势**减少 30%**
- 进攻方回合 = `round(1.5 + 防守方人数^0.3) × 2`
  - 例：50 名守军 → 10 回合
- 防守方回合 = `round(0.5 + max(1 + 防守方人数^0.3 × 聚落优势, 防守方/进攻方)) × 2`
  - 例：一级城墙城镇，500 守军 vs 250 攻方 → 48 回合

#### 守军被压制

当防守方 ≤ 10 人时触发，防止"人海战术"中低级攻方无限消耗高级守军。

- 防守方回合 = `max(round(min(进攻方×3, 防守方×0.3)), 1)`（1~3 回合）
- 进攻方回合 = `max(round(min(防守方×3, 进攻方×0.3)), 1)`（1~3 回合）

#### 常规战斗

- 防守方回合 = `round(min(进攻方×2, 防守方^0.6))`
- 进攻方回合 = `round(min(防守方×2, 进攻方^0.6))`

#### 撤退

双方均无回合（立即结束）。

### 2.2 回合执行机制

每回合通过概率决定哪方行动：

- 进攻方行动概率 = `进攻方剩余回合 ÷ (进攻方剩余回合 + 防守方剩余回合)`
- 投骰子 → 低于概率则进攻方行动，否则防守方行动
- 行动方消耗 1 个己方回合，执行一次"模拟战斗"

## 三、模拟战斗（单次 Tick）

### 3.1 执行流程

1. 从双方队列中**随机各选一个兵种**
2. 基于兵力值、战场优势和士气计算伤害
3. 随机伤害类型：30% 钝击、70% 割伤
4. 对防守方兵种造成伤害（可能杀死）
5. 给予进攻方经验和士气奖励

### 3.2 伤害公式

```
伤害 = (0.5 + 0.5 × random) × 40 × (进攻方兵力值 ÷ 防守方兵力值)^0.7 × 进攻方优势
```

其中 `random` 为 0~1 之间的随机数。

伤害之后依次应用：
1. 技能/天赋修正
2. 士气修正

### 3.3 士气对伤害的修正

**进攻方**（低士气时被惩罚，高士气无加成）：
- 士气修正 = `min(进攻方士气 - 50, 0)`
- 100 士气 → 修正 0；25 士气 → 修正 -25

**防守方**（高士气时获得加成，低士气无惩罚）：
- 士气修正 = `max(防守方士气 - 50, 0)`
- 100 士气 → 修正 +50；25 士气 → 修正 0

**综合**：伤害最终乘以 `(进攻方修正 - 防守方修正) × 0.005`
- 低士气进攻 vs 高士气防守 → 伤害降低 37.5%
- 高士气进攻 vs 低士气防守 → 无额外降低

## 四、兵力值计算（Troop Power）

### 4.1 总公式

```
兵力值 = 默认兵力值 × (1 + 领袖修正 + 地形修正)
```

### 4.2 默认兵力值

**基础战力**：
- 英雄 = `英雄等级 ÷ 4 + 1`
- 普通兵 = 兵种等级（Tier）

**缩放公式**：
```
缩放值 = (2 + 基础战力) × (10 + 基础战力) × 0.02
```

**兵种类型修正**：
| 类型 | 修正 |
|---|---|
| 英雄 | ×1.5 |
| 骑乘部队 | ×1.2 |
| 普通部队 | ×1.0 |

**示例**：
- Tier 3 步兵：`(2+3) × (10+3) × 0.02 = 1.3`
- Tier 5 骑兵：`(2+5) × (10+5) × 0.02 × 1.2 = 2.52`
- 12 级英雄：`(2+4) × (10+4) × 0.02 × 1.5 = 2.52`

### 4.3 地形/环境修正

#### 步兵（最全能，从不被惩罚）

| 地形 | 进攻方 | 防守方 |
|---|---|---|
| 攻城 | 0.00 | 0.00 |
| 村庄 | +0.05 | +0.05 |
| 渡河 | 0.00 | +0.05 |
| 森林 | +0.05 | +0.05 |
| 平原 | 0.00 | 0.00 |

#### 弓箭手（守城最佳）

| 地形 | 进攻方 | 防守方 |
|---|---|---|
| 攻城 | **-0.20** | **+0.30** |
| 村庄 | -0.10 | +0.05 |
| 渡河 | 0.00 | +0.10 |
| 森林 | -0.10 | **-0.50** |
| 平原 | 0.00 | 0.00 |

#### 骑兵（模拟战中最差）

| 地形 | 进攻方 | 防守方 |
|---|---|---|
| 攻城 | -0.10 | -0.10 |
| 村庄 | 0.00 | -0.10 |
| 渡河 | -0.15 | -0.05 |
| 森林 | **-0.20** | -0.15 |
| 平原 | **+0.25** | +0.10 |

#### 骑射手

| 地形 | 进攻方 | 防守方 |
|---|---|---|
| 攻城 | -0.20 | **+0.30** |
| 村庄 | +0.10 | 0.00 |
| 渡河 | -0.10 | 0.00 |

## 五、聚落防御优势

### 5.1 防御方优势计算

| 聚落类型 | 基础优势 |
|---|---|
| 村庄 | 125%（+25%） |
| 城镇/城堡 | 100% + 城墙加成 |

**城墙加成**：

| 城墙等级 | 加成 | 城墙被破坏后 |
|---|---|---|
| 基础城墙 | +400% | +100%（-75%） |
| 一级升级 | +500% | +125% |
| 满级城墙 | +600% | +150% |

**防御优势范围**：200% ~ 700%（2:1 到 7:1）

### 5.2 攻城方优势计算

| 攻城设备 | 加成 |
|---|---|
| 任意攻城锤或攻城塔 | +25% |
| 改良攻城锤 | +24%（普通 +16%） |
| 多架攻城塔 | +24%（单架 +16%） |
| 每台火焰攻城器械 | +12%（普通 +8%） |

**攻城方优势范围**：100% ~ 221%

### 5.3 优势比

优势比 = 防御方优势 ÷ 攻城方优势

| 场景 | 比值 |
|---|---|
| 无特殊加成 | 2:1 |
| 最强防御 | 7:1 |
| 最弱防御（城墙全破 vs 满装备攻方） | 0.905:1 |

## 六、战术技能的影响

- 每点 Tactics 技能 → +0.1% 战场优势
  - 100 级 → +10%
  - 330 级 → +33%
- "行军中"天赋 → 攻城时 +20% 优势
- "巡逻"天赋 → 对强盗 +10% 优势
- "战前机动"天赋 → 每 100 点 Tactics 差值 +0.1%

## 七、溃败与胜利

### 7.1 胜利条件

一方耗尽部队**或**士气归零即判负。

### 7.2 溃败机制

若因士气归零而失败（而非全军阵亡）：
- 所有英雄负伤
- 所有剩余部队立即损失
- 可能生成逃兵队伍

## 八、对极简化战斗设计的启示

### 8.1 可以直接借鉴的机制

| 机制 | 极简化价值 |
|---|---|
| 兵力值 = f(兵种等级) | 用 Tier 直接代表战力，一个数字概括兵种实力 |
| 伤害 ∝ (攻/守兵力比)^0.7 | 非线性——以少胜多不是不可能，但代价巨大 |
| 士气影响战斗结果 | 保留，作为战前状态的重要输入 |
| 防守方天然优势 | 保留，攻城代价高于野战 |
| 随机性 | 保留，避免确定性结果（0.5 + 0.5×random 的基础随机） |

### 8.2 可以大幅简化的部分

| 原版机制 | 极简化方向 |
|---|---|
| 4 种兵种类型的地形修正矩阵 | 合并为"兵种战力值"，不区分兵种类型 |
| 攻城设备系统 | 移除，用简单的"攻城方惩罚"替代 |
| 技能/天赋修正系统 | 移除或合并为"指挥能力"单一修正值 |
| 多轮回合制 | 简化为单次结算：基于双方总兵力值 + 士气 + 随机因素 |

### 8.3 推荐的极简战斗公式原型

```
进攻方总兵力 = Σ(每个兵种的Tier) × 士气修正
防守方总兵力 = Σ(每个兵种的Tier) × 士气修正 × 防御加成

胜率 = 进攻方总兵力 ÷ (进攻方总兵力 + 防守方总兵力)
结果 = 基于胜率的概率判定 + 随机偏移

伤亡 = f(双方兵力比, 随机因素)
```

防御加成：
- 野外：1.0
- 聚落防守：1.5~3.0（视聚落等级）
