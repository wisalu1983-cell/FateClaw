# 骑砍行动与聚落层级映射

> **文档用途**：系统梳理骑马与砍杀中玩家（角色）可执行的所有行动类型，标注每个行动需要的最低聚落层级。用于指导极简化世界结构设计中的聚落类型取舍决策。
>
> **数据来源**：Mount & Blade Wiki (mountandblade.fandom.com)、StrategyWiki、TaleWorlds 官方开发日志。以战团为主，标注霸主差异。

---

## 一、聚落层级定义

骑砍世界由 4 个层级构成：

| 层级 | 说明 | 功能定位 |
|---|---|---|
| **大陆（World Map）** | 聚落之间的开放野外区域 | 移动、遭遇、追踪、攻城发起 |
| **城镇（Town）** | 最大的聚落类型，内部包含城堡区（城主大厅） | 经济中心 + 社交枢纽 + 军事管理 |
| **城堡（Castle）** | 独立的军事据点 | 军事防御 + 有限社交（领主驻扎时） |
| **村庄（Village）** | 最小的聚落类型，依附于城镇或城堡 | 资源产出 + 基础兵源 |

关键结构：**城镇内部包含了城堡的功能区**（城主大厅、地牢），因此城镇可以承载城堡的几乎所有行动。

## 二、完整行动清单与最低层级映射

### 移动与探索

| 行动 | 最低层级 | 说明 |
|---|---|---|
| 在地图上移动（选择目的地） | 大陆 | 以队伍为单位，受移动速度影响 |
| 追踪其他队伍（查看足迹） | 大陆 | 需要 Tracking 技能 |
| 观察/发现其他队伍 | 大陆 | 受 Spotting 技能和视野范围影响 |
| 进入聚落 | 大陆 | 友好/中立聚落可进入，敌对需攻城 |

### 战斗

| 行动 | 最低层级 | 说明 |
|---|---|---|
| 野外遭遇战 | 大陆 | 与敌方队伍/强盗/商队在野外遭遇时触发 |
| 攻城战（发起） | 大陆 | 从城镇/城堡外部发起，需要负关系+至少10兵 |
| 攻城战（防守） | 城镇/城堡 | 被攻击时在城镇或城堡内防守 |
| 劫掠村庄 | 村庄 | 对敌方村庄的掠夺行动 |
| 参加比武/竞技 | **城镇** | 竞技场仅存在于城镇中 |
| 加入AI战斗（支援友军） | 大陆 | 在野外观察到友军交战时可选择加入 |

### 经济

| 行动 | 最低层级 | 说明 |
|---|---|---|
| 交易商品（市场买卖） | **城镇** | 完整市场仅存在于城镇 |
| 购买食物/原材料 | 村庄 | 村庄可购买基础物资，城镇市场更丰富 |
| 开设工坊/产业 | **城镇** | 通过行会长购买产业用地 |
| 贩卖战俘 | **城镇** | 酒馆中的赎金经纪人 |
| 收取领地税 | 城镇/城堡/村庄 | 拥有领地后自动周结算 |

### 军事（招募与管理）

| 行动 | 最低层级 | 说明 |
|---|---|---|
| 招募农兵（基础兵源） | **村庄** | 村庄是基础兵源的唯一来源 |
| 招募佣兵 | **城镇** | 酒馆中招募 |
| 招募同伴（Companion） | **城镇** | 同伴仅在酒馆中出现 |
| 驻军（留兵驻守） | 城镇/城堡 | 村庄不能驻军 |
| 管理驻军 | 城镇/城堡 | 调整驻军编制 |
| 建设改良（信使站/瞭望塔等） | 城镇/城堡/村庄 | 三种聚落各有可建设项 |

### 社交

| 行动 | 最低层级 | 说明 |
|---|---|---|
| 拜访领主（对话/接任务） | 城镇/城堡 | 领主驻扎在城镇的城堡区或独立城堡中 |
| 举办/参加宴会 | 城镇/城堡 | 战团中城镇和城堡均可举办宴会 |
| 求婚/联姻 | 城镇/城堡 | 需要拜访贵族女性，她们在城镇/城堡中 |
| 说服领主叛变 | 大陆/城镇/城堡 | 任何能与领主对话的场合 |
| 与村长交流 | **村庄** | 村长只在村庄中 |
| 酒馆社交（酒客/旅行者） | **城镇** | 酒馆仅在城镇中 |

### 任务

| 行动 | 最低层级 | 说明 |
|---|---|---|
| 接领主任务（军事/政治类） | 城镇/城堡 | 领主在场时可接 |
| 接行会长任务（商业/民生类） | **城镇** | 行会长仅在城镇中 |
| 接村长任务（乡村生存类） | **村庄** | 村长仅在村庄中（运粮、赶牛、训练农兵、剿匪等） |

### 政治

| 行动 | 最低层级 | 说明 |
|---|---|---|
| 提议宣战/议和 | 无限制 | 通过影响力发起投票（霸主），不依赖位置 |
| 分封领地 | 无限制 | 国王权限，不依赖位置 |
| 建国 | 城镇/城堡 | 需要拥有至少一个城镇或城堡 |

### 信息

| 行动 | 最低层级 | 说明 |
|---|---|---|
| 查看百科/领主信息 | 无限制 | UI 功能，任何位置可用 |
| 查看大地图局势 | 大陆 | 在大地图上可见 |
| 查看本地价格 | 城镇 | 进入市场后可查看 |

## 三、各聚落层级的独占行动分析

以下标注每个层级**独有的**行动——即只能在该层级执行、其他层级无法替代的行动。

### 大陆（World Map）独占

| 独占行动 | 重要度 |
|---|---|
| 在地图上移动 | 核心 |
| 野外遭遇战 | 核心 |
| 追踪/发现其他队伍 | 中 |
| 攻城发起 | 核心 |
| 加入AI战斗 | 低 |

### 城镇（Town）独占

| 独占行动 | 重要度 |
|---|---|
| 交易商品（完整市场） | 核心 |
| 开设工坊/产业 | 中 |
| 招募佣兵 | 中 |
| 招募同伴（Companion） | 核心 |
| 贩卖战俘 | 中 |
| 参加比武/竞技 | 低 |
| 接行会长任务 | 中 |
| 酒馆社交 | 中 |

**城镇独占 8 项行动，其中 2 项为核心级。**

### 城堡（Castle）独占

| 独占行动 | 重要度 |
|---|---|
| （无完全独占行动） | — |

**城堡没有任何完全独占的行动。** 城堡能做的所有事情，城镇内部的城堡区（城主大厅、地牢）都能做。城堡的独特价值不在于行动种类，而在于：
- **军事定位**：作为纯军事据点，驻军成本更低（士兵半薪）
- **战略位置**：控制通道/要冲，不一定在经济中心
- **防御成本**：驻军约 100~150 人，比城镇更容易攻占

### 村庄（Village）独占

| 独占行动 | 重要度 |
|---|---|
| 招募农兵（基础兵源的唯一来源） | 核心 |
| 接村长任务 | 低 |
| 劫掠（对敌方的破坏行动） | 中 |

**村庄独占 3 项行动，其中 1 项为核心级。**

## 四、对极简化世界结构设计的启示

### 关键发现

1. **城镇是行动最丰富的聚落类型**：独占 8 项行动（交易、同伴、佣兵、工坊、竞技、战俘、行会任务、酒馆），且内部包含城堡的全部功能。城镇是**不可省略**的聚落类型。

2. **城堡没有独占行动**：城堡的所有行动都可以在城镇内完成。城堡的独特价值纯粹是军事/战略层面的（位置、成本、防御难度）。极简化时，城堡**可以考虑与城镇合并**，或简化为"有城防的城镇"/"据点"概念。

3. **村庄独占一个核心行动（招兵）**：如果基础兵源招募被视为核心玩法，则村庄需要以某种形式存在。但村庄可以**不作为独立地图节点**，而是作为城镇的附属属性（城镇拥有 N 个附属村庄 → 决定招兵上限和资源产出）。

4. **大陆/野外是连接一切的粘合层**：移动、遭遇、追踪都发生在这里。极简化时不需要显式建模野外，但节点间的"旅行"概念必须保留。

### 对之前三种世界结构方案的影响

| 发现 | 对方案选择的影响 |
|---|---|
| 城堡无独占行动 | **方案 A/B/C 中的独立城堡节点可能过度设计**。可以考虑将城堡功能合并到城镇中（城镇 = 经济中心 + 军事据点），或将城堡简化为"哨卡/关隘"（纯战略节点，不提供社交/经济行动） |
| 村庄仅独占招兵 | **村庄不必作为独立地图节点**。可以作为城镇的属性（每城镇有附属村庄，影响招兵和资源产出），仅在特定行动时"访问"（如劫掠目标） |
| 城镇独占最多行动 | **城镇应该是地图的核心节点类型**，世界结构设计应以城镇为骨架 |

### 建议的极简化聚落方案

基于以上分析，建议将之前的三类聚落简化为两类：

| 极简类型 | 对应骑砍原版 | 功能 |
|---|---|---|
| **城镇** | 城镇 + 城堡 | 一切核心行动的发生地：交易、招募、社交、政治、驻军、防御 |
| **村庄/哨卡** | 村庄 + 关隘 | 简化节点：资源产出、基础招兵、战略通道。行动种类很少，主要作为地图上的"经过点"和资源来源 |

城镇附属若干村庄（不作为独立节点，而是城镇的属性），村庄被劫掠 → 城镇招兵和资源受损。
