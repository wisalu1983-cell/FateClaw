# 骑砍核心循环概要与评审

> **文档用途**：基于骑马与砍杀（战团 + 霸主）的官方资料，提炼与 AIvilization 架构直接相关的核心机制概要，并附评审结论。本文档是参考资料，设计决策见 `MVP/Design/` 下的各模块文档。
>
> **数据来源**：Mount & Blade Wiki (mountandblade.fandom.com)、StrategyWiki、TaleWorlds 官方开发日志。

## 一、大地图与空间

骑砍的世界是一张持续运转的**大陆地图**，包含三类聚落节点：

| 聚落类型 | 功能 | 数量（战团→霸主） |
|---|---|---|
| **城镇（Town）** | 核心经济中心：交易市场、工坊、酒馆（招募同伴）、竞技场、行会 | 22 → 57 |
| **城堡（Castle）** | 军事据点：驻军、防御工事、攻城战场所 | ~48 → 更多 |
| **村庄（Village）** | 资源产出点：粮食/原材料生产，招募基础兵源，依附于城镇或城堡 | 每座城镇/城堡关联 2+ 个村庄 |

> 霸主的聚落规模约为战团的 2.5 倍。极简化设计不需要还原完整规模，只需保留三类聚落的功能分化逻辑。

聚落之间是**开放的野外区域**：
- 角色在大地图上以**队伍**为单位移动（不是单人）
- 移动速度受队伍规模、地形、马匹、技能影响
- 野外存在**强盗窝点**、流动的**商队**、巡逻的**领主队伍**
- 在大地图上遭遇其他队伍可触发战斗或对话

**与 AIvilization 的映射**：骑砍的大地图 ≈ AIvilization 的"世界空间"，聚落 ≈ 地点节点，野外移动 ≈ Agent 在地点间的位移。

## 二、角色日常行为循环

骑砍中不存在显式的"一天日程表"，角色的行为是**目标驱动的连续行动流**：

```
评估当前状态（钱、兵、声望、关系、队伍状况）
    ↓
选择下一个目标（去哪里？做什么？）
    ↓
在大地图上移动到目标地点
    ↓
到达后执行行动：
  - 在城镇：交易、招兵、接任务、参加比武、拜访领主、宴会社交
  - 在村庄：招募农兵、购买粮食/原材料
  - 在野外：战斗、追击、护送、巡逻
    ↓
获得结果（金钱、声望、关系变化、战利品、情报）
    ↓
回到评估 → 循环
```

**关键特征**：
- 没有固定日程，行为完全由目标和当前状态驱动
- 同一时间世界中所有领主都在独立执行自己的行为循环
- 食物会消耗、士气会波动、伤兵需要恢复——这些"维护需求"自然产生日常行为
- 任务系统是行为多样性的重要来源：村长、行会长、领主各自提供不同类型的委托（护送商队、剿灭强盗、运送物资、收税等），完成任务是获取关系值和金钱的主要途径

**四变量平衡框架**：Wiki 将骑砍的策略核心概括为四个基础变量的平衡管理——

| 变量 | 获取方式 | 消耗/风险 |
|---|---|---|
| **Money（金钱）** | 交易、任务、战利品、领地税 | 军队维持费、装备、招募、贿赂 |
| **Renown（声望）** | 战斗胜利（以少胜多奖励更高）、任务 | 随时间自然衰减，高声望更难获取 |
| **Honor（荣誉）** | 释放被俘领主、拒绝不义任务 | 与军事效率直接冲突（释放敌人=放虎归山） |
| **Relationships（关系）** | 完成委托、战斗协同、政治支持 | 与其他关系冲突（分封让一人满意则其他人不满） |

四者相互牵制：释放敌方领主 → 荣誉 ↑ 但军事优势 ↓；劫掠村庄 → 金钱 ↑ 但关系 ↓ 且荣誉 ↓。这个"四变量平衡"框架是骑砍策略层的灵魂。

## 三、势力与 NPC 自主行为

骑砍世界中有多个势力并存（战团 6 个，霸主 8 个——帝国分裂为北/南/西三部分 + 5 个独立王国），每个势力包含：

| 层级 | 说明 |
|---|---|
| **国王** | 势力领袖，决定宣战/议和、分封领地 |
| **家族（Clan）** | 势力内的基本组织单位，拥有领地和军队 |
| **领主（Lord）** | 家族成员中的活跃角色，各自带领队伍行动 |
| **聚落** | 城镇/城堡/村庄，归属于具体家族 |

**NPC 领主的自主行为模式**：补给、巡逻、防御、进攻、社交、经济。

**领主性格影响行为倾向**（战团有 8 种性格类型，可归为四大倾向）：
- 正直/善良型（Upstanding/Good-natured） → 忠诚守约，偏好巡逻防御
- 好战型（Martial） → 偏好进攻和挑战敌军
- 狡猾型（Cunning） → 追求自身利益最大化
- 卑劣型（Sadistic/Debauched/Pitiless/Bad-tempered） → 偏好劫掠，容易叛变

**势力兴衰机制**：宣战/议和投票、领主叛变、攻城战导致版图变化、势力过弱时领主叛逃。

## 四、社交与政治系统

骑砍的社交系统以**关系值**为核心。关系受任务完成、战斗互动、领地分封、释放/俘虏领主、宴会赴约等因素影响。

**政治层面**：封臣贡献影响力→提议政策/宣战/议和；领主间形成非正式派系；联姻建立家族联盟；说服系统基于魅力+性格匹配度。

**宴会系统**：领主聚集、关系建设、政治站队、求婚的核心场景，是"多 Agent 在同一时空社交"最集中的体现。

---

## 五、经济与资源

围绕**金钱（第纳尔）** 运转。收入来源：交易、任务报酬、战利品、领地税收、工坊产业。支出去向：军队维持费、食物消耗、驻军费用、装备招募。

**核心经济压力**：养兵很贵——队伍越大、兵种越高级，维持费越高。

**聚落繁荣度**：影响税收、物价和招兵质量。受贸易畅通度、巡逻安全性、战争破坏影响。

---

## 六、成长与进阶路径

```
自由冒险者 → 佣兵（Clan Level 1） → 封臣（Clan Level 2） → 建国者（Clan Level 4）
```

每阶段解锁不同行为空间。同伴系统（酒馆招募、4 种角色、上限 4~8 人、可派遣独立队伍）。

## 七、概要评审结论

> 基于骑砍官方 Wiki、TaleWorlds 开发日志交叉验证。

### 逐节评审摘要

| 维度 | 评价 |
|---|---|
| 大地图与空间 | 聚落数量标注有误已修正（战团 22 城镇，霸主 57 城镇），功能描述准确 |
| 角色日常行为循环 | 准确。补充了四变量平衡框架和任务系统 |
| 势力与 NPC 自主行为 | 基本准确。领主性格从 4 类修正为 8 类归四大倾向 |
| 社交与政治系统 | 准确。补充了宴会系统 |
| 经济与资源 | 准确。补充了聚落繁荣度 |
| 成长与进阶路径 | 准确 |

### 总体结论

| 维度 | 评价 |
|---|---|
| **事实准确性** | 良好，无重大事实错误 |
| **结构完整性** | 良好，遗漏已补充（四变量框架、宴会系统、繁荣度、任务系统） |
| **与本项目的对齐度** | 较好，始终以 AIvilization 架构约束为锚点做映射 |

### 评审发现的遗漏机制

| 遗漏机制 | 对本项目的影响 | 重要度 |
|---|---|---|
| 四变量平衡框架 | 数值骨架至少需要 4 个维度 | 高 |
| 宴会系统 | Agent 社交场景载体 | 高 |
| 聚落繁荣度 | 经济系统"健康度指标" | 中 |
| 任务系统 | Agent 行为多样性来源 | 中 |
| Time 作为核心约束 | 影响时间系统设计 | 中 |
